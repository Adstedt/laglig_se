1 · Dual-segment single bar recommended

Single bar shows both metrics in one glance: the lighter segment grows when a user assesses a row, the solid segment grows when they sign it. Sequential by design (you bedöm before you signera), so the segments stack naturally. Mirrors the modal treatment from the Kontroller list.

3 · Single bar — slutmål (signerade)

Testkontroll Pågående

Endast slutmålet — “signerat = klart”

Signerat 18 av 68 · 26 %

44 av 68 bedömda — 26 kvar att signera

Simplest. Shows the only metric that really matters for "är vi klara?". The Bedömda count drops to a sub-label. Tradeoff: hides the in-between progress signal — a user who has assessed 60/68 rows but signed none still looks like they're 0 % done.

5 · Inline pill — kompakt

Most compact. Becomes a single visual element next to the title, fits into smaller surfaces (modals, list summaries) and feels immediately scannable. Tradeoff: needs a hover/click to reveal the two underlying metrics, so it's less informative at first glance.

Design notes

  • Sequential semantics. The audit flow is bedöm → signera. A signed row is also (by definition) assessed. Single-bar variants encode that order via stacked segments; two-bar variants don't.
  • Colour. Emerald reads as “progress / klart” in our existing palette (matches the “Uppfylld” pill on the rows). Track is muted white at 8 % so it sits quietly in the dark UI.
  • Status badge interaction. When status flips to Klar, the bar can hold at 100 % and the colour shifts from emerald to the muted “locked” treatment used elsewhere for completed states.
  • Empty state. 0 / 68 still renders the empty track — the absence of fill is itself the progress signal.
  • Accessibility. Each bar gets role="progressbar" + aria-valuenow / valuemax + a hidden text fallback (“X of Y rader signerade”).
