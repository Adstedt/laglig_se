Today every list page in Laglig draws its own empty state. The visual weight, icon, title voice, and CTA placement all differ. This prototype audits the variance, names a single primitive (the existing <EmptyState>), and shows the five pages re-rendered through it — with a clean split between truly empty (first-run) and filtered empty (no matches).

2 — Three things to fix

The chrome around the empty area (toolbar, breadcrumb, page title) is already consistent. The empty state itself isn't.

01
One anatomy, one voice

Every empty list uses the same icon container (56 × 56, rounded muted), the same title voice ("Inga {entity} ännu"), and renders Safiro 500 on the title. No more "Tom mapp" / "Du har inga X just nu" / missing titles.

02
Body CTAs mirror the toolbar's primary action

If the toolbar has a primary + secondary, the empty state mirrors both. Power users still skip the empty state and use the toolbar — keep both. "Ny X" is primary; "Importera" / "Mall" is secondary.

03
Truly-empty ≠ filtered-empty

Filtered-empty (filters applied, no matches) gets a softer treatment: dashed-border icon, "Inga X matchar dina filter", and a "Rensa filter" primary action. Create-flow CTA stays available but secondary.

API

  • icon — lucide icon, wrapped in EmptyState.Icon
  • title — "Inga {entity} ännu" (Safiro 500, 17px)
  • description — one sentence, action-oriented
  • action — JSX (one or two buttons)
  • variant — 'empty' (default) | 'filtered'

Title-voice rules

  • Truly empty → Inga {entity} ännu
  • Filtered → Inga {entity} matchar dina filter
  • Never: "Tom mapp", "Du har inga X just nu"

CTA rules

  • Truly empty → mirror the toolbar's primary (+ secondary if it exists)
  • Filtered → primary "Rensa filter", secondary preserves the create flow
  • Toolbar buttons stay either way

6 — How the rollout would land

Mechanically small. The primitive already exists; the work is mostly copy + replacing one-off divs with <EmptyState>.

  • 1. Add variant: 'empty' | 'filtered' to EmptyState; introduce the dashed-border icon style for filtered.
  • 2. Codify title-voice rules in a tiny copy doc (or as inline JSDoc on the primitive) so reviewers catch deviations.
  • 3. Replace the bespoke empty markup on the five surfaces with <EmptyState> calls. Each page already has the data needed to choose empty vs filtered (filter state).
  • 4. Audit pass: grep for any remaining "Tom mapp" / "Du har inga X just nu" / titleless paragraphs.

Estimated effort: half a day for the primitive + per-page swap, plus copy review for each Swedish string. Could ride along the next epic-22 consistency story.
