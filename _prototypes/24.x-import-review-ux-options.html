Import-review — UX rework prototype

The current per-row card layout (~120 px tall) doesn't scale to 50–1000-row imports the way users expect from tools like HubSpot, Stripe, or Mailchimp. This prototype shows a compact-table reviewer that fits ~3× more rows on screen and keeps the rich detail (LLM reasoning, alt candidates) one click away in a side drawer.

Why it works

  • 3× more rows visible — for a 50-row import users can see all status tiers without scrolling
  • Familiar pattern — HubSpot contact import, Stripe customer import, Notion CSV upload all use this shape
  • LLM reasoning + alt candidates stay accessible (drawer) but stop dominating the page
  • Bulk select + bulk-action footer matches modern workflows ("review 50 rows, accept the obvious 40, scrutinise the 10 weird ones")
  • Sticky filter chips + commit button stay in view at all scroll depths

Trade-offs

  • Reasoning text isn't visible at a glance — requires click. Mitigation: hover-tooltip on the confidence badge could show first ~80 chars inline
  • Long titles need truncation with title-tooltip (catalog titles can be 100+ chars). Code uses CSS truncation + native title attr
  • The row-as-button vs row-with-actions can confuse — separate the click target (row body opens drawer) from the action buttons (don't propagate)
  • Mobile layout needs a separate stacked-card view (out of scope for this prototype, current layout is also bad on mobile)
Alternates considered

Other patterns we could explore

Not built out — just sketched. Happy to prototype either if you'd prefer one of these over the table.

B. Wizard grouped by status

3 steps: review your 4 high-confidence matches (one-click batch accept), then your 10 medium-confidence matches (paginated review), then your 6 unmatched rows (request adds or reject). TurboTax / FreeAgent vibes — guides the user through the path of least resistance.

C. Two-pane reviewer (Linear-style)

Dense list on the left (one line per row), full detail panel on the right that updates as you arrow-key through. Keyboard-first: j/k to navigate, Enter to accept, x to reject, r to replace. Power-user heaven; less friendly for first-time users.
