Laglig Design Language

A living reference for the Laglig product surface — what we ship today, why, and a small list of brand-polish moves that would make the workspace feel as considered as the marketing site. Built by walking the running app and reading the source.

Foundations · 01

Color tokens

Defined as HSL triplets on :root. Tailwind consumes them via hsl(var(--token)). The palette leans warm — cream backgrounds, earth-dark primary — which differentiates Laglig from the cooler/bluer enterprise SaaS norm. Three "section" tints exist for atmospheric backgrounds.

Foundations · 02

Typography

Two families. Safiro carries the brand — display moments, page H1s, marketing headers. Google Sans Flex handles all UI body copy — labels, tables, form inputs, paragraph text. Reserve Safiro for moments that announce; never use it for content density.

Foundations · 03

Spacing & radii

Tailwind's default 4-pt scale. Radii carry semantics: chip/badge are pill (full), buttons + cards are 6px (md), the page-level Card primitive jumps to 8px (lg). That last gap is a small inconsistency worth knowing about — see Suggestions.

Foundations · 04

Chart palette

Five warm/earth-toned chart colors defined as --chart-1…--chart-5. Sage, moss, terracotta, amber — deliberately not the standard blue/green/red Tailwind defaults. When building dashboards, reach for these tokens (not bg-blue-500 / stroke-emerald-500).

Primitives · 03

FilterChip

Toggle filters — distinct from view tabs. Renders aria-pressed not role="tab". Always wrap in FilterChipGroup for screen-reader scoping.

Statescomponents/ui/filter-chip.tsx
The pressed state currently fills with --primary (warm dark) — visually loud in dense filter rows. A "soft pressed" treatment is a candidate refinement; see Suggestions → Soft pressed FilterChip.

Suggestions · ideas

Worth exploring later

Lower confidence, larger blast radius. Do not ship without prototyping.

Adopt chart tokens app-wide
idea

Audit Hem signal cards + any future dashboards to use --chart-1..5 instead of Tailwind defaults. Currently the warm chart palette exists but is barely consumed.

Unify table column-header treatment
idea

Legal-doc tables use uppercase tracking-wide column headers; main app tables don't. A consolidated <Th> with text-xs uppercase tracking-wide text-muted-foreground would feel more polished on hub pages.

Section-tinted hero cards on Hem
idea

Use --section-warm / --section-sage as the background of signal cards on the Hem dashboard so the page reads more like a brand artifact than a data console. Already defined in tokens — zero new CSS.

Radius unification (md vs lg)
idea

Buttons + most surfaces use 6px (md). Card jumps to 8px (lg). Pick one and document it. Most Linear-class apps unify on 6px or 8px and stay there. Verify with Card against Button visual pairing first.

Density toggle (sm-input filter rows)
idea

Inputs at 40px feel tall paired with 36px secondary buttons (size sm). Adding a 36px size="sm" variant for filter/toolbar contexts would let dense hub pages breathe without sacrificing form-screen comfort.

Dark-mode section tints
idea

Currently --section-warm/sage/cream are light-only. If the empty-state + Hem-card proposals land, dark-mode counterparts (e.g., 5% warm tint) need defining or those surfaces fall back to neutral muted in dark.
