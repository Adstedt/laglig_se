Aligning the five Uppgifter tabs without flattening their personalities

Each tab serves a different job (dashboard, kanban, list, calendar, archive) — that's correct. What's drifting are the shared atoms: status pill, priority pill, overdue indicator, empty state. Same data, five different visual languages. This prototype proposes three small primitives + an empty-state migration so every task atom looks the same wherever it appears.

// components/ui/task-status-badge.tsx
export function TaskStatusBadge({
  label,         // column.name from workspace
  color,         // column.color (hex from Prisma)
  size = 'md',   // 'sm' for calendar-pill density
  showDot = true,
}) {
  return (
    <span
      className={cn(
        'inline-flex items-center gap-1.5 rounded-full border font-medium',
        size === 'sm' ? 'px-2 py-0.5 text-[10.5px]' : 'px-2.5 py-0.5 text-xs'
      )}
      style={{
        backgroundColor: `${color}24`,    // ~14% alpha
        borderColor: `${color}59`,        // ~35% alpha
        color: color,
      }}
    >
      {showDot && <span className="h-2 w-2 rounded-full" style={{ background: color }} />}
      {label}
    </span>
  )
}

§5

Toolbar layout — tabs and filters fight for one row

Today the tabs, search, four filter dropdowns, and (on Lista) the Kolumner button all share one flex-wrap row with justify-between. On wide viewports the two groups float apart with awkward air between them; on Lista the row wraps unpredictably and Kolumner ends up offset right while filters drop to a new line on the left. Five elements total (tabs, search, 4 dropdowns, sometimes Kolumner) is more than a single row should carry.

Kolumner

↑ Kolumner is in UnifiedToolbar's right zone, so justify-between pins it to the right. When the left group wraps onto two visual lines, Kolumner doesn't move with the filters — it floats on its own. Reads as "where did Kolumner come from?"

proposed

Two rows: tabs above, filters (+ Kolumner) below

Tabs are about which view of this page (navigation). Filters are about what subset of this view's data (operation). They're conceptually different and should sit on different rows. Tabs always span left to right with no opposing element to fight; the filter row is left-aligned with Kolumner pinned right (so it follows the filters when they wrap, instead of floating on its own line). Adds about 36px of vertical chrome — modest cost, big alignment payoff.
