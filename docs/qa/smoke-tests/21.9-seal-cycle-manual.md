# Story 21.9 — Fastställ kontroll manual smoke test

**Prerequisite:** PROD-001 from Quinn's QA gate. Run this before flipping status → Done.

## 0. Setup (one-time, ~2 min)

```bash
# From the project root
pnpm install            # if fresh clone
pnpm prisma migrate dev # apply the CONSTRAINT-001 migration + all prior
pnpm tsx scripts/seed-seal-cycle-demo.ts --workspace almasa-havshotell-ab-fvarlj
pnpm dev
```

The seed creates two cycles in the `almasa-havshotell-ab-fvarlj` workspace (Almåsa Havshotell):
- **DEMO — Redo för fastställande** — 3 items signed, 1 open AVVIKELSE, 1 closed AVVIKELSE, 1 OBSERVATION, mixed evidence
- **DEMO — Draft-dokument blockar** — 2 items signed, DRAFT styrdokument as evidence (v0.5: now requires override, not a hard reject)

Log in as the workspace OWNER and navigate to `/laglistor/kontroller`.

## Test 1 — Happy path with override (open AVVIKELSE)

**Goal:** exercise the primary seal flow + override-gate policy.

1. Click **"DEMO — Redo för fastställande"** in the cycle list.
2. **Verify header state (pre-seal):**
   - Status badge shows `Avslutad` (amber).
   - No sealed-banner visible.
   - Progress cluster shows `3 av 3 bedömda` / `3 av 3 signerade`.
   - `Åtgärder` dropdown shows **three** entries: `Återställ till Pågående`, `Fastställ kontroll` (destructive red), and no `Slutför kontroll` (already completed).
3. Click **Åtgärder → Fastställ kontroll**.
4. **Verify dialog state:**
   - Title: `"Fastställ kontrollen?"`
   - Three base paragraphs render (irreversibility, edit lock, file-deletion guard).
   - Amber warning box reads: `"Du har 1 öppna avvikelser. Fastställande med öppna avvikelser kräver en motivering..."`
   - Textarea labelled `"Motivering för fastställande trots öppna avvikelser (obligatoriskt, min 20 tecken)"`.
   - Primary button `"Fastställ kontroll"` is **disabled**.
5. Type 19 chars → **verify** button stays disabled. Type a 20th → enabled.
6. Enter a realistic override: `"Åtgärdsplan för A1 pågår över Q2 och hanteras via task #123."`
7. Click **Fastställ kontroll**.
8. **Verify post-seal state:**
   - Toast: `"Kontrollen är fastställd"`.
   - Dialog closes.
   - Status badge flips to `Fastställd`.
   - **New amber banner appears** with:
     - ShieldCheck icon.
     - `"Fastställd"` label.
     - Truncated hash chip: `abc12345…f1a2` format (first 8 + `…` + last 4).
     - Copy button (icon).
     - Subline: `"Fastställd av [Owner Name] den 24 apr. 2026"`.
   - Åtgärder dropdown is now hidden (cycle is read-only).
9. Click the **copy-to-clipboard** icon.
   - Toast: `"Kontrollsumma kopierad"`.
   - Paste into a text editor → full 64-char hex hash.
10. Navigate to the **Rapport tab**.
   - Emerald banner: `"Fastställd kontroll — seal-hash visas på titelsidan och i sidfoten."`
   - Iframe renders the revisionsrapport with the hash on the cover + footer of every page.
11. Navigate to **`/workspace/activity`**.
   - New row appears: `"[Owner] fastställde kontrollen DEMO — Redo för fastställande"`.
   - Clicking expands the row → verify `new_value.sealHash`, `new_value.sealedAt`, `new_value.overrideReason` (verbatim), `new_value.openAvvikelsesAtSeal: 1`.

**Pass criteria:** all 11 steps observe the expected state. If any Swedish copy differs from above, screenshot + flag.

## Test 2 — DRAFT styrdokument requires override (INTEGRITY-001 v0.5)

**Goal:** confirm the v0.5 softened pre-seal integrity check — DRAFT-state evidence no longer HARD-blocks the seal; it surfaces in a dedicated DRAFT panel and requires an override motivering (mirroring the AVVIKELSE pattern).

> **Policy note:** v0.4 hard-rejected any seal with DRAFT evidence. v0.5 softens this to snapshot-and-accept-with-override at PO's request after Almåsa smoke surfaced that linking DRAFT styrdokument as work-in-progress references is routine. The cryptographic infrastructure (SHA-256, snapshots, manifest, file-deletion guard) is unchanged — only the UX friction is softened. The override reason + acknowledged draft list are locked into both the ActivityLog and the canonical manifest.

1. From `/laglistor/kontroller`, click **"DEMO — Draft-dokument blockar"**.
2. Click **Åtgärder → Fastställ kontroll**.
3. **Verify dialog state:**
   - Three base paragraphs render (irreversibility, edit lock, softened file-deletion copy: *"skyddas mot oavsiktlig radering — du får ett varningsmeddelande om någon försöker ta bort dem"*).
   - **Amber "DraftDocumentsOverridePanel"** renders with:
     - `[UTKAST]` badge (amber) on each draft row.
     - Doc title: `"DEMO styrdokument — Utkast — blockerar seal"`.
     - Context label underneath the title (e.g. `"AFS 2020:1 Arbetsplatsens utformning"`).
   - Textarea labelled `"Motivera varför kontrollen fastställs trots ett utkast-styrdokument (obligatoriskt, min 20 tecken)"`.
   - Primary button `"Fastställ kontroll"` is **disabled** (needs motivering + type-to-confirm).
4. Type 19 chars in the motivering → verify button stays disabled. Type a 20th → enabled (after also typing FASTSTÄLL in the confirm box).
5. Enter a realistic override: `"Utkastet är referensmaterial utanför auditomfattningen. Beskriver framtida rutin, ej tillämpad."`
6. Type `FASTSTÄLL` in the confirm input.
7. Click **Fastställ kontroll**.
8. **Verify post-seal state:**
   - Toast: `"Kontrollen är fastställd"`.
   - Status flips to `Fastställd` with the hash banner.
9. Navigate to **`/workspace/activity`** and expand the `cycle_sealed` row:
   - `new_value.sealHash`, `new_value.sealedAt` present.
   - `new_value.overrideReason` — **the verbatim text you typed**.
   - `new_value.draftDocumentsAtSeal: [{ id: "...", title: "DEMO styrdokument — Utkast — blockerar seal" }]` — the acknowledged draft list.
   - `new_value.openAvvikelsesAtSeal: 0` (no open avvikelser on this cycle).

**Pass criteria:** dialog surfaces the DRAFT panel pre-seal; seal proceeds with a valid override; activity log captures both the override reason AND the acknowledged draft list.

**Edge case — empty override:** click primary without filling the motivering. Toast: `"Fastställande blockeras: 1 styrdokument i utkast-status (DEMO styrdokument — Utkast — blockerar seal). Ange en motivering för att fastställa trots utkast-styrdokument."` Dialog stays open.

**Combined blockers (manual variant):** if you link an *additional* DRAFT doc to one of the DEMO #1 items via the UI, then attempt seal on cycle #1 (which already has an open AVVIKELSE), the dialog should show **both** the `OpenAvvikelserOverridePanel` AND the `DraftDocumentsOverridePanel` stacked, with a single shared textarea labelled `"Motivera varför kontrollen fastställs trots öppen avvikelse och ett utkast-styrdokument (obligatoriskt, min 20 tecken)"`. One override motivering covers both.

## Test 6 — Race guard (SF-2 updateMany)

**Goal:** verify concurrent seal attempts don't create partial state.

Harder to test manually. If you want coverage:
1. Open the cycle in two browser tabs.
2. In both tabs, click Åtgärder → Fastställ kontroll → fill in override → click confirm at roughly the same time (use DevTools to throttle network if needed).
3. **Verify:** one tab succeeds (status flips, banner appears). The other gets toast:
   `"Kunde inte fastställa kontrollen"` with description `"Kontrollens status ändrades under fastställandet. Försök igen."`
4. Cycle has exactly ONE sealed state + ONE activity-log row (no duplicates).

This is automated-test-only territory realistically. The unit test `SF-2 — surfaces SF-2 race error when updateMany returns count 0` covers it.
