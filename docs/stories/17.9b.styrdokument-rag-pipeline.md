# Story 17.9b: Chunk & Embed Authored Styrdokument into the RAG Pipeline

## Status

Draft — **Deferred (behind 17.9)**

> Split from Story 17.9 on 2026-05-22. 17.9 ships the uploaded-file RAG path (`USER_FILE`); this story adds the **authored styrdokument** path (`WorkspaceDocument` → `WORKSPACE_DOCUMENT`) so the agent is aware of a workspace's *full* document corpus (uploaded files **and** typed styrdokument). Deferred because the styrdokument path carries several prerequisites the file path doesn't — captured below so they aren't rediscovered. Reuses 17.9's `chunk-workspace-document.ts` / `sync-workspace-chunks.ts` / `retrieveContext` machinery.

## Story

**As a** system component,
**I want** authored styrdokument (`WorkspaceDocument`) chunked + embedded into the vector pipeline (workspace-scoped, `WORKSPACE_DOCUMENT`),
**so that** the agent can semantically search a workspace's authored policies/styrdokument alongside its uploaded files.

## Acceptance Criteria

1. Styrdokument chunks: `source_type = WORKSPACE_DOCUMENT` (exists, `schema:1797` region — no enum migration), `source_id = WorkspaceDocument.id`, `workspace_id` non-null. Reuse 17.9's `WorkspaceChunkInput` + `syncWorkspaceChunks` + the `sourceTypes` retrieval option (extend the agent's workspace query to `['USER_FILE','WORKSPACE_DOCUMENT']`).
2. **Content source = `content_html`, not `extracted_text`** (the prerequisite that blocked it in 17.9): `WorkspaceDocumentVersion.extracted_text` is **flat plaintext** (`documents.ts:41` strips all tags → no headings to split on). Chunk from `content_html` via `htmlToMarkdown` first (preserves `<h1>/<h2>` for heading-aware section breaks), then the same paragraph-merge as files.
3. **Reliable trigger (NOT fire-and-forget):** route styrdokument re-index through a cron/queue or `waitUntil` — consistent with 17.8/17.9's stated reason that Vercel doesn't reliably run post-response work. (The original 17.9 draft used `void sync(...)`, which contradicts that rule.)
4. **Enumerate which version events trigger re-index** (5 write paths touch content): `saveDocumentVersion` (`documents.ts:537`), `restoreDocumentVersion` (`:664`), `createDraftFromApproved` (`:755`), `importDocxDocument` (`:941`) → re-index; `autosaveDocument` (`:478`) → **do NOT** re-embed (high-frequency in-place mutation = cost bomb; index on explicit save/version events only).
5. **Draft-status gating:** do not embed `DRAFT` / `ARCHIVED` / `SUPERSEDED` documents (`WorkspaceDocumentStatus`, `schema:1909`) — embedding a draft would let the agent surface unapproved policy as authoritative. Decide the indexable status set (likely `APPROVED`/published only).
6. **Lifecycle cleanup:** a `WorkspaceDocument` **delete action does not exist yet** (`prisma.workspaceDocument.delete` is referenced only in a completed story doc, never in `app/actions/documents.ts`). Either create it (with `WORKSPACE_DOCUMENT` chunk `deleteMany`) or wire cleanup wherever deletion lands. (Files used `deleteFile`/`deleteFilesBulk` — styrdokument has no equivalent today.)
7. **Dedupe:** styrdokument has no `content_hash` (that's a 17.8/`WorkspaceFile` field). Define a per-document content hash (e.g. sha256 of `content_html`) stored in chunk `metadata` to gate re-embedding, mirroring 17.9 AC 9.
8. `metadata` carries `{ title, document_type }`; `contextual_header` = title + `WorkspaceDocumentType` (`schema:1898`).

## Dev Notes

- **Model names (anti-hallucination):** the styrdokument models are `WorkspaceDocument` (`schema:1927`, `prisma.workspaceDocument`) and `WorkspaceDocumentVersion` (`:1972`, `prisma.workspaceDocumentVersion`). There is no plain `Document` model. (A `DocumentVersion` model + `DocumentStatus` enum DO exist — `schema:1081` / `:604` — but under the SFS/LegalDocument namespace, unrelated to styrdokument; do not use them here.)
- Reuses everything 17.9 builds; the net-new work is: `content_html`→markdown chunking, the trigger/queue, status gating, the delete action, and the per-doc hash.
- `WorkspaceDocumentVersion.extracted_text` exists (`schema:1981`) but is plaintext — see AC 2.

## Prerequisites

- **Story 17.9 must ship first** (provides `chunk-workspace-document.ts`, `sync-workspace-chunks.ts`, the `sourceTypes` retrieval option).
- A `WorkspaceDocument` delete pathway (AC 6).
