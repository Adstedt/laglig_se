# Story 22.4: Cycle Items + Styrdokument → shadcn `<Table>` migration

## Status

Draft

## Story

**As a** developer working on workspace UI,
**I want** the two remaining custom div-grid tables (Cycle Items, Styrdokument) migrated to the shared shadcn `<Table>` primitive at `components/ui/table.tsx`,
**so that** every tabular surface in the workspace uses the same primitive (a11y semantics, sortable header pattern, future affordances like sort / drag / selection unlocked uniformly), and the audit's "two of six tables aren't tables" gap is closed.

## Acceptance Criteria

### Functional

#### Task group A — Cycle Items

1. **`cycle-items-tab.tsx` migrated to `<Table>` primitive.** `components/features/compliance-audit/cycle-detail/cycle-items-tab.tsx` (line 55 area) is refactored from custom div-rendered rows to `<Table><TableHeader><TableHead>...</TableHead></TableHeader><TableBody><TableRow>...</TableRow></TableBody></Table>` from `components/ui/table.tsx`. Columns preserved as today: `Lag` / `Nuvarande status` / `Bedömning` / `Motivering` / `Ansvarig` / `Signerad`.

2. **All inline editors preserved unchanged.** The existing `ItemBedomningSelect`, `ItemMotiveringEditor`, `ItemSignOffButton` components render INSIDE `<TableCell>` — same components, same props, same handlers. No behaviour change, only the wrapping markup differs.

3. **Expandable drawer behaviour preserved.** `cycle-item-row-drawer.tsx` continues to render outside the `<TableRow>` (it's a separate panel today). The chevron / row-click trigger that opens the drawer is wired correctly inside the new `<TableRow>` structure. Drawer open/close state and content unchanged.

4. **Virtualisation decision documented.** Today the file uses `@tanstack/react-virtual` for rows >100. Two options:
   - **(a)** Keep virtualisation: wrap `<TableBody>` in a virtualizer-aware container. The shadcn `<TableBody>` is just a `<tbody>` — virtualization can sit between it and `<TableRow>`. Use the existing `useVirtualizer` setup; only the inner row markup changes.
   - **(b)** Remove virtualisation: if cycle Items are reliably ≤100 rows in production, drop the virtualizer for simplicity. Verify maximum row count from real workspaces before choosing this path.
   - **Recommendation: (a)** — keep virtualisation. Cycles can have ≥100 items (an entire laglista's worth of compliance items). Removing virtualisation risks UX regression on large cycles. Confirm with PO if (b) is preferred.
   - Document the decision in the Dev Agent Record.

#### Task group B — Styrdokument

5. **`document-table.tsx` migrated to `<Table>` primitive.** `components/features/documents/document-table.tsx` (line 70 area) is refactored from custom grid layout to shadcn `<Table>`. Columns preserved as today: `Titel` / `Dokumentnr` / `Typ` / `Status` / `Version` / `Författare` / `Senast uppdaterad` / `Granskningsdatum` / (action menu).

6. **Sortable headers preserved using `SortableHeader` primitive.** Existing sort behaviour on `Titel`, `Senast uppdaterad`, `Granskningsdatum` (line 70 area) is preserved. Wrap each sortable column header in the existing `components/ui/sortable-header.tsx` primitive (or whatever the existing sort-arrow pattern is — confirm via Glob first). Sort state, sort handlers, ARIA `aria-sort` attribute all preserved.

7. **Per-row "..." overflow menu preserved.** The rightmost cell renders the existing `<DropdownMenu>` with row actions (Redigera / Ta bort / Ladda ner / etc — preserve whatever options exist today). Menu state and handlers unchanged; only the wrapping `<TableCell>` is new.

8. **Type pill, Status pill, "Senast uppdaterad" relative-time formatting all preserved.** These are content within cells; the migration is structural only.

#### Cross-cutting

9. **`<Table>` primitive is the same one used by Laglistor / Tasks / Kontroller list.** No new wrapper, no per-surface variant; the migration consumes `Table`, `TableHeader`, `TableBody`, `TableHead`, `TableRow`, `TableCell` exports from `components/ui/table.tsx` directly.

10. **Audit's count of `<Table>` consumers goes from 4-of-6 to 6-of-6.** After this story:
    - `compliance-detail-table.tsx` (Laglistor) ✓ (already)
    - `cycle-list-table.tsx` (Kontroller) ✓ (already)
    - `cycle-items-tab.tsx` (Cycle Items) ✓ (NEW — Task group A)
    - `cycle-findings-tab.tsx` (Findings) — INTENTIONALLY NOT MIGRATED, see AC 11
    - `list-tab.tsx` (Tasks) ✓ (already, TanStack Table)
    - `document-table.tsx` (Styrdokument) ✓ (NEW — Task group B)

11. **Findings tab explicitly stays as a virtualised list, not a table.** `cycle-findings-tab.tsx` renders findings as expandable cards / rows in a virtualised list. This is intentional — findings are not tabular records (each has variable height, expandable detail, action buttons inline). Migrating findings to `<Table>` would harm the existing UX. **This story does NOT touch `cycle-findings-tab.tsx`.** Document this exception in a comment at the top of the file: `// Intentional non-Table layout: findings render as expandable cards, not tabular rows. See Story 22.4 AC 11.`

  - `getByRole('table')` is the regression guard that confirms the migration landed (custom divs render as `role="grid"` at best, often as nothing).
  - `getByRole('columnheader')` for header assertions.
  - `aria-sort` for sortable column state on Styrdokument.

- **Manual a11y verification (IV4)** is highly recommended — automated `getByRole('table')` confirms the markup but a screen reader run confirms the announce-quality.

### Technical Constraints

- **Zero behaviour change.** All per-row interactions, sort handlers, drawer triggers, menu actions work exactly as today.
- **Visual deltas acknowledged** (flag in PR description):
  - Cycle Items rows may render slightly different (semantic `<table>` rendering vs CSS grid). Tailwind classes on the shadcn `<Table>` primitive may give subtly different padding / borders. Confirm visual parity is acceptable; if not, override per-cell classes.
  - Styrdokument rows: same caveat. Existing custom grid layout has explicit column widths; shadcn `<Table>` uses semantic auto-sizing. Verify column widths look right; if needed, set explicit `className="w-[Xpx]"` on `<TableHead>` to preserve widths.
- **Findings tab is intentionally NOT migrated** (AC 11). Document the exception with a comment.
- **Out of scope for 22.4 (handled by 22.3):** header / toolbar layout. 22.4 touches only the table body.
- **Out of scope for 22.4 (handled by 22.1):** Badge / pill rendering inside cells.
- **Coordination:** ensure 22.1–22.3 have landed (or are in advanced review) before starting 22.4. Cycle detail components are touched by all four stories.
- **No new dependencies.** All work uses existing shadcn / Tailwind / TanStack Virtual / cva infrastructure.
- **PR landing strategy:** Task group A and Task group B as separate commits / sub-PRs. They share no code; landing independently limits revert blast radius.
