# Story 19.1b: Promote a Chat Attachment to Filer ("Spara i Filer")

<!-- Source: split from Story 19.1 during PO validation (2026-05-23). 19.1 keeps chat attachments conversational-only (excluded from Filer + RAG, category-keyed). This story adds the opt-in to promote one into a curated workspace file. -->

## Status

Draft

> 📌 **Why this exists:** [[19.1]] makes chat attachments *conversational context* by default — uploaded with `category = 'CHAT_ATTACHMENT'`, extracted (so the agent can read them in-turn), but **excluded from Filer and the RAG index**. Sometimes a user realises a chat upload is worth keeping. This story adds the deliberate opt-in: **"Vill du lägga till filen i dina filer?"** → saves it to Filer + ingests it into RAG. 19.1 already left the **promotion seam** (the exclusion is keyed purely on `category`), so this is a thin, additive story.

## Story

**As a** workspace member who attached a useful file in chat,
**I want** a one-click "Spara i Filer" action on that attachment,
**so that** it becomes a permanent, searchable workspace file (visible in Filer and findable by the agent later) — instead of staying a one-off conversational attachment.

## Dependencies

- **Story 19.1** — **REQUIRED (must be Done).** Provides `FileCategory.CHAT_ATTACHMENT`, the upload+extraction path, the persisted attachment chips (`ChatMessage.metadata.attachments`), and the category-keyed RAG/Filer exclusion this story reverses. [[19.1]] AC 15-17.

## Context

A chat attachment after 19.1 is a `WorkspaceFile` with `category = 'CHAT_ATTACHMENT'`: extracted (`extracted_text` populated), **not** embedded (skipped at `extract-files/route.ts:197`), **not** in Filer (excluded in `getWorkspaceFiles:744`). Promotion is therefore cheap: flip the category to a curated value and run the embed it skipped. No re-upload, no re-extraction.

## Acceptance Criteria

1. **Affordance.** Each chat-attachment chip (the persisted history chip from 19.1, rendered in `chat-message.tsx`) gains a quiet **"Spara i Filer"** action (icon + tooltip, e.g. `FolderPlus`). It is **not** a modal interrupt on attach, and it appears only for chips whose file still has `category = 'CHAT_ATTACHMENT'` (already-promoted files show a non-actionable "Sparad i Filer" state).
2. **Promote action.** A server action `promoteChatAttachment(fileId, category?)` (new, in `app/actions/files.ts` or `agent-reads.ts`'s neighbourhood) — workspace-scoped via `withWorkspace` — that, for a file whose current `category = 'CHAT_ATTACHMENT'`:
   - flips `category` to the chosen value (default `OVRIGT`; the existing `updateFile` action at `app/actions/files.ts:859` already handles the category write — reuse or extend it), **and**
   - triggers `syncWorkspaceChunks(fileId, 'USER_FILE', workspaceId, file.extracted_text, { filename, category, content_hash })` to embed it (the embed 19.1 skipped). If `extracted_text` is empty (extraction not yet complete), the action still flips the category and lets the normal `extract-files` cron embed it on its next pass (no error).
3. **Idempotent / guarded.** Calling promote on a file that is already non-`CHAT_ATTACHMENT` is a no-op success (returns the current state). A file from another workspace returns a Swedish `wrapToolError`-style `{ success: false }` — never cross-tenant.
4. **Result is visible.** After promotion: the file appears in **Filer** (`getWorkspaceFiles` no longer excludes it — its category is now curated) and is **searchable** via `search_workspace_files` (17.9c) once embedded. The chip flips to the "Sparad i Filer" state with a link to the file in Filer.
5. **Optional category pick (nice-to-have).** The action MAY offer a small category picker (`POLICY | AVTAL | BEVIS | CERTIFIKAT | OVRIGT`) before promoting; if deferred, default `OVRIGT` (user can re-categorise in Filer afterwards). Keep v1 minimal — default `OVRIGT` is acceptable.
6. **Swedish copy.** All user-facing strings Swedish: action "Spara i Filer", success toast "Filen sparades i dina filer.", promoted-state chip "Sparad i Filer".

## Tasks / Subtasks

- [ ] **Task 1: `promoteChatAttachment` server action (AC 2, 3)** — workspace-scoped; guard current category; category flip (reuse `updateFile:859`) + `syncWorkspaceChunks` embed; idempotent no-op when already curated; `extracted_text`-empty fallback (let the cron embed).
- [ ] **Task 2: Chip affordance (AC 1, 4, 6)** — in `chat-message.tsx`, add the "Spara i Filer" action to the chat-attachment chip; gate on `category === 'CHAT_ATTACHMENT'`; on success flip to "Sparad i Filer" + Filer link; Swedish toast.
- [ ] **Task 3: Optional category picker (AC 5)** — small inline picker before promote, or default `OVRIGT`. Implementer's choice for v1.
- [ ] **Task 4: Tests** — `promoteChatAttachment`: category flip + embed called; already-curated → no-op; cross-workspace → failure; `extracted_text` empty → flips category, no embed, no error. Chip: action shown only for `CHAT_ATTACHMENT`; hidden/"Sparad" otherwise.

## Dev Notes

- **Promotion seam (from 19.1):** the RAG/Filer exclusion is keyed solely on `category` ([[19.1]] AC 15-17), so flipping the category *automatically* re-includes the file in `getWorkspaceFiles` (`files.ts:744`); only the embed must be triggered explicitly (the cron skipped it at `extract-files/route.ts:197` precisely because the category was `CHAT_ATTACHMENT`).
- **Reuse the backfill pattern:** the embed call is the same `syncWorkspaceChunks(fileId, 'USER_FILE', …)` used by the file-RAG pipeline — text is already extracted, so it's a single embed.
- **`updateFile`** (`app/actions/files.ts:859`, category write at `:885`) already handles a category change with workspace scoping — extend it or call it from `promoteChatAttachment`.
- **No schema change, no migration** — `category` is an existing column; this only changes a value + triggers an existing embed path.
