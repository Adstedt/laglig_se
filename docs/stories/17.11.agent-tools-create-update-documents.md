# Story 17.11: Agent Tools — Create & Update Documents

## Status

Draft

## Story

**As an** AI compliance agent,
**I want** to create new documents and edit existing ones,
**so that** I can help users draft policies, risk assessments, and action plans based on their compliance context.

## Story Context

**Existing System Integration:**

- Integrates with: Agent tools registry from Story 17.10, `Document`/`DocumentVersion` models (created in Story 17.1), two-phase confirmation pattern from existing write tools (`create-task.ts`, `update-compliance-status.ts`), system prompt
- Technology: Vercel AI SDK tool definitions (`tool` + `zodSchema` from `ai`, `z` from `zod/v4`), markdown ↔ Tiptap JSON conversion
- Follows pattern: Existing write tools use two-phase confirmation: `execute=false` → calls `wrapWriteToolResponse()` returning `WriteToolResponse<T>`, user approves, `execute=true` → persist and return `wrapToolResponse()`
- Touch points: `lib/agent/tools/index.ts`, `app/actions/documents.ts` (created in Story 17.1), `lib/agent/system-prompt.md`, `components/features/ai-chat/chat-message.tsx` (TOOL_CONFIG)

## Acceptance Criteria

**create_document Tool:**

1. New tool: `create_document`
2. Input: `title` (string), `document_type` (DocumentType enum), `content` (markdown string), optional `template_id`, optional `linked_list_item_id`, optional `linked_task_id`
3. Agent provides content as **markdown** — service layer converts markdown → Tiptap JSON via `markdownToTiptap()` utility
4. Follows two-phase confirmation: `execute=false` returns preview of the document to be created via `wrapWriteToolResponse()`, `execute=true` persists and returns via `wrapToolResponse()`
5. Creates `Document` + `DocumentVersion` v1 with `source: 'AGENT'`
6. If `linked_list_item_id` or `linked_task_id` provided, creates corresponding link records
7. Returns: document ID, title, version number

**update_document Tool:**

8. New tool: `update_document`
9. Input: `document_id` (string), `section_heading` (string — heading text to locate), `updated_content` (markdown for that section), `change_summary` (string)
10. Service layer: reads current `content_json`, locates section by heading text, replaces section content (everything between the target heading and the next heading of same or higher level), saves as new `DocumentVersion` with `source: 'AGENT'`
11. Follows two-phase confirmation pattern (same `execute=false`/`execute=true` pattern as other write tools)
12. Returns: document ID, new version number, change summary

**Guards:**

13. Both tools enforce: document must be in DRAFT or IN_REVIEW status — cannot edit APPROVED/SUPERSEDED/ARCHIVED documents. Agent informed via `wrapToolError()` with guidance to tell user to create a new version first.
14. Both tools scoped to current workspace

**Conversion Utilities:**

15. `markdownToTiptap(markdown: string): TiptapJSON` — converts markdown to Tiptap JSON (headings, paragraphs, bold/italic, lists, tables, links)
16. `tiptapToMarkdown(json: TiptapJSON): string` — converts Tiptap JSON to markdown (for agent to read existing content in a format it handles well)
17. Both utilities handle common markdown elements: `#`-`###` headings, `**bold**`, `*italic*`, `- bullet`, `1. numbered`, `| tables |`, `[links](url)`

**Audit:**

18. ActivityLog entries created for all agent document operations with `entity_type: 'document'` and user context noting agent authorship. New action labels (`document_created`, `document_updated`) added to `lib/constants/activity-labels.ts`.

**Prompt:**

19. Agent system prompt updated with guidance on when/how to create vs. update documents
20. TOOL_CONFIG in `components/features/ai-chat/chat-message.tsx` updated with entries for both tools (each entry requires `label`, `doneLabel`, and `icon` fields matching existing TOOL_CONFIG shape)

## Dev Notes

**Two-phase confirmation pattern — actual implementation (from `create-task.ts` and `update-compliance-status.ts`):**

All write tools use the shared helper functions from `lib/agent/tools/utils.ts`:
- `wrapWriteToolResponse(tool, action, params, preview, startTime)` → returns `WriteToolResponse<T>` with `confirmation_required: true`
- `wrapToolResponse(tool, data, startTime)` → returns successful `ToolResponse<T>`
- `wrapToolError(tool, message, guidance, startTime)` → returns `ToolError`

The `WriteToolResponse<T>` type (from `lib/agent/tools/types.ts`) has fields: `confirmation_required: true`, `action`, `params`, `preview`, `_meta`. The TOOL_CONFIG's `sidebarHint: 'open'` (set automatically by `wrapWriteToolResponse`) causes the sidebar to auto-open with the preview.

```typescript
// Correct tool factory pattern (matches existing tools exactly):
import { tool, zodSchema } from 'ai'
import { z } from 'zod/v4'
import { wrapWriteToolResponse, wrapToolResponse, wrapToolError } from './utils'

const createDocumentSchema = z.object({
  title: z.string().describe('Document title'),
  document_type: z.enum(['POLICY', 'RISK_ASSESSMENT', 'ACTION_PLAN', 'PROCEDURE', 'INSTRUCTION', 'CHECKLIST', 'REPORT', 'OTHER']).describe('Document type'),
  content: z.string().describe('Document content in markdown format'),
  template_id: z.string().optional().describe('Optional template ID'),
  linked_list_item_id: z.string().optional().describe('Optional law list item ID to link'),
  linked_task_id: z.string().optional().describe('Optional task ID to link'),
  execute: z.boolean().optional().default(false).describe('false = return proposal for confirmation, true = execute the action'),
})

export function createCreateDocumentTool(workspaceId: string, userId: string) {
  return tool({
    description: 'Skapa ett nytt efterlevnadsdokument...',
    inputSchema: zodSchema(createDocumentSchema),
    execute: async ({ title, document_type, content, execute, ...rest }) => {
      const startTime = Date.now()
      const tiptapJson = markdownToTiptap(content)

      if (!execute) {
        return wrapWriteToolResponse(
          'create_document',
          'create_document',
          { title, document_type, content_preview: content.slice(0, 500) },
          `Skapa dokument: "${title}" (${document_type})`,
          startTime
        )
      }

      try {
        const doc = await createDocumentFromAgent(workspaceId, userId, { title, document_type, tiptapJson, ...rest })
        return wrapToolResponse('create_document', { document_id: doc.id, title, version: 1 }, startTime)
      } catch (err) {
        const message = err instanceof Error ? err.message : String(err)
        return wrapToolError('create_document', `Kunde inte skapa dokumentet: ${message}`, 'Ett tekniskt fel uppstod. Försök igen om en stund.', startTime)
      }
    },
  })
}
```

**Section boundary detection:**

```typescript
function findSectionBoundaries(content: TiptapNode[], headingText: string): { start: number, end: number, level: number } | null {
  for (let i = 0; i < content.length; i++) {
    const node = content[i]
    if (node.type === 'heading' && getTextContent(node).toLowerCase() === headingText.toLowerCase()) {
      const level = node.attrs.level
      // Find end: next heading of same or higher level, or end of doc
      let end = content.length
      for (let j = i + 1; j < content.length; j++) {
        if (content[j].type === 'heading' && content[j].attrs.level <= level) {
          end = j
          break
        }
      }
      return { start: i, end, level }
    }
  }
  return null
}
```
