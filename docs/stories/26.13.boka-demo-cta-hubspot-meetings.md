# Story 26.13: Boka demo CTA + HubSpot Meetings integration

## Status

Draft

## Story

**As a** prospective customer evaluating Laglig.se who wants a human walkthrough before committing,
**I want** a clearly visible "Boka demo" path on the marketing pages that lets me book a time directly in a calendar,
**so that** I can get a sales-assisted demo without emailing back and forth — while self-serve visitors keep the org-number check as the fastest path to value.

### Concept decision (ratified in discussion 2026-07-23)

- The inline org-number check **stays the sole primary CTA** in marketing heroes. It is the product-led differentiator ("skriv ert orgnummer, se era regler på 30 sek") and must NOT be collapsed behind a generic "Prova gratis" button.
- "Boka demo" ships as the **secondary, sales-assisted path**: persistent in the navbar, quiet link in heroes — never an equal-weight competing button next to the org check.
- Demo bookings land on a dedicated `/boka-demo` page with the **HubSpot Meetings scheduler embedded** (linkable from ads/emails/auditor channel, keeps UTM attribution, auto-creates the contact + meeting in the company HubSpot account). This **supersedes Epic 26 Story 26.10 AC 3** (`/demo` with Cal.com embed) — PO to ratify the epic edit.

## Acceptance Criteria

1. **`/boka-demo` page ships** under `app/(marketing)/boka-demo/` using the existing marketing chrome (NavbarV3 + FooterV3 via the same shell pattern as other marketing pages): short Safiro headline + 2–3 sentence supporting copy (natural Swedish per the copy voice rules), the HubSpot Meetings scheduler embed, and a visible fallback mailto contact link for visitors whose browsers block the embed (no `/kontakt` route exists — verified 2026-07-23).
2. **HubSpot Meetings embed** loads its script **only on `/boka-demo`** (no global script; protects Core Web Vitals on all other marketing pages). The meeting link URL comes from an env var (e.g. `NEXT_PUBLIC_HUBSPOT_MEETINGS_URL`) — no hardcoded HubSpot account slugs in the repo. Verify the current embed snippet/API against HubSpot's official docs (Context7/web) at implementation time — do not code it from memory.
3. **Cookie consent respected:** the HubSpot embed sets third-party cookies, so it must follow the same consent pattern that gates GA4 (`lib/track-event.ts` pattern / existing cookie banner). If consent is declined, show a static card with the fallback contact link + a direct (non-embedded) link out to the HubSpot meeting page instead of silently loading the embed.
4. **Navbar:** `navbar-v3.tsx` gains a "Boka demo" outline/ghost button next to the existing "Kom igång" filled button (desktop right-side cluster AND the mobile sheet). "Kom igång" stays the visually primary action. Link goes to `/boka-demo` with UTM + click event.
5. **Heroes:** the secondary line under the org check gains a "boka en demo" link in BOTH hero variants:
   - `components/marketing/sections/hero-org-check.tsx` — extend the existing "Vill ni hellre börja direkt? … i 15 dagar" line to also offer the demo link (one sentence, two quiet options; org check remains sole primary).
   - `components/features/landing-v3/hero-v3.tsx` (homepage) — equivalent quiet demo link adjacent to/below the `OrgCheckForm`.
6. **Org-check → demo context handoff:** if the visitor has run the org check, the demo link/booking carries the company context. Minimum: `/boka-demo` accepts `?org=&company=` and forwards supported prefill params to the HubSpot scheduler (HubSpot meeting links accept `firstname`/`lastname`/`email` prefill; company/org fields only if the booking form has matching fields — verify against HubSpot docs and configure the booking form accordingly). The org-check result modal's demo pathway (if trivially reachable) may pass these; otherwise document the handoff as a follow-up.
7. **Analytics + attribution:** a `marketing_demo_cta_click` event fires with `{ kind, slug, placement }` from every demo CTA (navbar, hero, any footer/band usage), mirroring `marketing_cta_click`. All demo links funnel through `buildUtmUrl` — extend `CtaPlacement` in `lib/marketing/utm.ts` with `'navbar'` (unit-tested). Booked-meeting attribution then lives in HubSpot (contact source) + GA4.
8. **Quality gates (epic conventions):** `/boka-demo` ships full metadata via the existing marketing metadata pattern, Lighthouse SEO ≥90, no CLS regression from the embed (reserve embed height), and the page is added to the sitemap.
9. **Epic bookkeeping:** Epic 26 doc updated — 26.10 AC 3 marked superseded by 26.13, 26.13 added to the story list (PO ratifies).

## Dev Notes

### Previous story insights

- 26.4–26.6 established the epic's quality conventions: Lighthouse SEO ≥90 on prod build, OG image verified as real PNG 1200×630, full regression command before review. [Source: docs/stories/completed/26.4.first-validation-batch.md; 26.6 Completion Notes]
- Copy rule: natural Swedish, read aloud as an HR user; no stiff derived nouns. "Boka demo" / "boka en demo" are fine; avoid constructions like "demobokning". [Source: memory feedback_swedish_copy_naturalness]

### Existing conversion architecture (verified in code 2026-07-23)

- **Hero org check (marketing pages):** `components/marketing/sections/hero-org-check.tsx` — renders `OrgCheckForm` (modal result) + secondary trial text link; events `marketing_org_check_submit` / `marketing_cta_click` with `{kind, slug, placement:'hero'}`.
- **Homepage hero:** `components/features/landing-v3/hero-v3.tsx` renders `OrgCheckForm` (eyebrow "Vilka regler gäller er? · 30 sek", `resultMode="modal"`, ~400px column). Mid-page re-conversion: `try-now-section.tsx` + `components/marketing/sections/org-check-cta.tsx`.
- **Org check → signup handoff:** `components/features/landing-v3/org-check-form.tsx` builds `/signup?org={orgNumber}` from the preview — the same pattern to reuse for `/boka-demo?org=&company=`.
- **CTA plumbing:** `components/marketing/sections/cta-block.tsx` is the canonical instrumented CTA (`buildUtmUrl` + `trackEvent('marketing_cta_click')`). `lib/marketing/utm.ts` exports `CtaPlacement = 'hero' | 'mid-page' | 'footer-strip'` — extend with `'navbar'`; `buildUtmUrl` is pure + unit-testable and never clobbers existing utm params.
- **Navbar:** `components/features/landing-v3/navbar-v3.tsx` — right cluster is currently `Logga in` (link) + `Kom igång` (Button → `/signup`), mirrored in the mobile sheet (~lines 520–630). No shared `components/shared/navigation/navbar.tsx` exists (epic doc's path is stale — navbar-v3 is the marketing navbar).
- **Marketing layout:** `app/(marketing)/layout.tsx` only forces light theme; page templates render NavbarV3 + FooterV3 themselves — `/boka-demo` must do the same (follow an existing page under `app/(marketing)/` as the pattern).
- **No HubSpot code exists in the repo today** (grep-verified) — this story introduces the first integration. Keep it embed-only (no HubSpot tracking script site-wide, no API calls).

### Technical constraints

- Next.js 16 App Router, React 19, TS strict, Tailwind 3. [Source: docs/prd/epic-26-marketing-pages-seo-content-engine.md#technology-stack]
- Embed script must be page-scoped (`/boka-demo` only) — epic's Core Web Vitals bar applies to all marketing pages. Reserve iframe height to avoid CLS.
- Consent: HubSpot embed sets cookies → gate behind the existing consent store in `lib/consent/` — `readConsent()` in `lib/consent/storage.ts`, `ConsentCategories` in `lib/consent/types.ts` (storage key `laglig_consent_v1`, versioned, 365-day TTL); GA4 gating pattern in `lib/track-event.ts` + `lib/consent/gtag.ts`. Reuse this store — do not invent a parallel one. The embed maps to the marketing/third-party category.
- Per CLAUDE.md: use Context7 to fetch current HubSpot Meetings embed docs before writing the embed code.

### Project structure notes

- New route: `app/(marketing)/boka-demo/page.tsx` (+ client embed component, suggest `components/marketing/sections/hubspot-meetings-embed.tsx`). This is a route page, not MDX — export metadata directly (or via `generateMarketingMetadata()` in `lib/marketing/get-page-metadata.ts` if it fits a non-MDX page).
- **Sitemap is NOT automatic for this page:** `scripts/generate-sitemaps.ts` (postbuild) auto-registers only MDX pages by walking `content/marketing/{funktioner,branscher,omraden}`. `/boka-demo` must be added to the script's static-pages list explicitly (AC 8).
- Env var `NEXT_PUBLIC_HUBSPOT_MEETINGS_URL` — document in `.env.example`; must be set in Vercel (Preview + Production).
- Epic doc conflict: Story 26.10 AC 3 says `/demo` + Cal.com. This story replaces that (HubSpot is the company's actual CRM/booking stack). 26.10 keeps `/kom-igang` + `/kundcase`.
