# Story 28.9: Laglistor Grouped Mode — Selection Fixed

**Epic:** 28 — Unified DataTable Core (`docs/prd/epic-28-unified-datatable-core.md`)
**Status:** Done (2026-07-08)

## What changed

**The grouped-mode selection bug is fixed** (the epic's stated goal for
this story): the flat tables gained optional CONTROLLED selection
(`selectedItemIds` + `onSelectionChange`); when provided they suppress
their internal bulk bar. Both grouped wrappers now pass their ONE lifted
`Set<string>` + setter through the section components into every nested
table — cross-section select-all and the shared bulk-bar count are correct
by construction. The dead `(groupId, itemIds, isSelected)` plumbing that
child sections ignored is deleted.

Touched: `document-list-table.tsx`, `compliance-detail-table.tsx`
(controlled-selection props), `group-table-section.tsx`,
`compliance-group-section.tsx` (forwarding), both grouped wrappers
(direct Set handler).

## Architectural judgment (deviation from the plan, documented)

The plan targeted replacing the wrappers with the core's
`GroupedDataTable`. In practice the laglistor sections embed the FULL
flat-table feature set (6 editors, bulk actions, remove dialog, load-more)
— not bare column configs — so `GroupedDataTable` (built for
personalregister-style sections) is the wrong tool here. The wrappers'
own mechanics (cross-group drag with header-prioritized collision,
within-group reorder, DragOverlay) are production-proven and now drive
core-rendered sortable rows via dnd 'external'. Replacing them would have
been rewrite-risk without user-visible gain.

**Residual (candidate for a later cleanup story):** the two wrappers
remain ~95% mutually identical (~1,000 lines). Deduplicating them into one
domain wrapper is hygiene-only and deliberately deferred.
