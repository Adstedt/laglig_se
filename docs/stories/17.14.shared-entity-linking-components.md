# Story 17.14: Shared Entity Linking Components

## Status

Draft

## Story

**As a** developer,
**I want** shared, reusable components for entity picking and linked-items display,
**so that** all linking surfaces (documents, files, law list items, tasks) use consistent UI patterns and new entity types can be linked without reimplementing the same picker/display logic.

## Story Context

**Existing System Integration:**

- Refactors: 6+ existing linking components across `components/features/tasks/`, `components/features/documents/`, `components/features/document-list/`, `components/features/files/`
- Technology: React, shadcn/ui (CommandDialog, Checkbox, Badge), cmdk
- Follows pattern: Existing `FilePickerModal` / `DocumentPickerModal` / `LinkedLawsBox` — all use CommandDialog with similar props and display logic
- Touch points: Task modal, legal document modal, document editor settings panel, file management

**Motivation:**

Currently 6+ components independently implement the same three UI patterns:
1. **Picker dialog** — CommandDialog + search + checkbox multi-select + available/already-linked groups + confirm
2. **Linked items list** — icon + title + metadata badges + hover X unlink button + loading/empty states
3. **Two-level cascading picker** — select category → select item within category → back navigation

Each reimplements loading states, selection logic, search debounce, toast feedback, and unlink confirmation from scratch. This story extracts shared primitives to eliminate duplication and ensure consistency.

## Acceptance Criteria

**Shared Picker Dialog:**

1. Generic `EntityPickerDialog<T>` component that accepts: `open`, `onOpenChange`, `onSelect` (returns selected IDs), `excludeIds`, `allowMultiple`, a data-fetching function, and a render function for each item row
2. Handles: search with 300ms debounce, loading spinner, empty state ("Inga resultat hittades"), "Tillgängliga" vs "Redan länkade" groups, checkbox multi-select, confirm/cancel footer with selection count
3. Uses `CommandDialog` from `@/components/ui/command` (cmdk) for keyboard navigation
4. Existing `FilePickerModal` and `DocumentPickerModal` refactored to use `EntityPickerDialog` — no visual or behavioral changes

**Shared Linked Items List:**

5. Generic `LinkedItemsList<T>` component that accepts: items array, a render function for each row, an `onUnlink` callback, loading state, empty message, and optional `readOnly` mode
6. Handles: loading spinner, empty state text, per-item hover X unlink button with spinner, group-hover opacity pattern
7. Existing linked-item displays in `LinkedDocumentsBox`, `DocumentsTab`, and the tasks/list-items sections of `DocumentLinksSection` refactored to use `LinkedItemsList` — no visual or behavioral changes

**Shared Cascading Picker Dialog:**

8. `CascadingPickerDialog<TGroup, TItem>` component for two-level selection: first level shows groups (with item count), second level shows items within selected group, with "← Tillbaka" back navigation
9. Accepts: group-fetching function, item-fetching function (given group ID), render functions for group and item rows, `onSelect` callback, `excludeIds`
10. Existing two-level pickers in `LinkedLawsBox` and `DocumentLinksSection` (law list item picker) refactored to use `CascadingPickerDialog` — no visual or behavioral changes. (`LawLinkSelector` is a Popover-based form selector with a different pattern — excluded from this story.)

**Integration Requirements:**

11. All existing linking behavior unchanged — this is a pure refactoring with zero user-facing changes
12. All existing tests continue to pass without modification (or with minimal mock updates)
13. New shared components have their own unit tests

**Quality Requirements:**

14. No new dependencies added
15. Shared components located in `components/shared/entity-linking/`

## Dev Notes

**Existing component interfaces (for reference when building shared components):**

Both `FilePickerModal` and `DocumentPickerModal` share this core interface:
```typescript
interface PickerProps {
  open: boolean
  onOpenChange: (_open: boolean) => void
  onSelect: (_ids: string[]) => void
  excludeIds?: string[]
  allowMultiple?: boolean
}
```

The shared `EntityPickerDialog` should generalize this to:
```typescript
interface EntityPickerDialogProps<T extends { id: string }> {
  open: boolean
  onOpenChange: (_open: boolean) => void
  onSelect: (_ids: string[]) => void
  fetchItems: (_search?: string) => Promise<T[]>
  renderItem: (_item: T) => React.ReactNode
  excludeIds?: string[]
  allowMultiple?: boolean
  searchPlaceholder?: string
  confirmLabel?: string
  emptyMessage?: string
}
```

**Linked items display — shared interface:**
```typescript
interface LinkedItemsListProps<T extends { id: string }> {
  items: T[]
  renderItem: (_item: T) => React.ReactNode
  onUnlink: (_id: string) => Promise<void>
  isLoading?: boolean
  emptyMessage?: string
  readOnly?: boolean
}
```

**Cascading picker — shared interface:**
```typescript
interface CascadingPickerDialogProps<
  TGroup extends { id: string; name: string; itemCount: number },
  TItem extends { id: string }
> {
  open: boolean
  onOpenChange: (_open: boolean) => void
  onSelect: (_itemId: string) => void
  fetchGroups: () => Promise<TGroup[]>
  fetchItems: (_groupId: string, _search?: string) => Promise<TItem[]>
  renderItem: (_item: TItem) => React.ReactNode
  excludeIds?: string[]
  searchPlaceholder?: string
  groupsHeading?: string
  itemsHeading?: string
}
```

**Key constraints:**

- `LinkedLawsBox` has **optimistic updates** with grouped display — do NOT refactor the display portion, only the picker dialog
- `FilePickerModal` has an extra "Ladda upp ny" button in the footer — the `EntityPickerDialog` should support an optional footer slot or extra actions prop
- `LawLinkSelector` uses a **Popover** (not CommandDialog) — it's a form-integrated selector, not a modal picker. Leave it as-is for this story; it could be addressed in a future story if needed
- All existing tests must continue to pass. Mock adjustments are acceptable but behavioral changes are not.
