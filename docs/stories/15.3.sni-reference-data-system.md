# Story 15.3: SNI Reference Data System

## Status

Approved

## Story

**As a** system enriching company data from BolagsAPI,
**I want** an internal SNI reference database that resolves raw SNI codes into their full meaning, hierarchy, and industry context,
**so that** the platform understands what a company actually does — enabling smarter law matching, richer agent context, and a user-friendly SNI selector in onboarding.

## Acceptance Criteria

### Data Preparation

1. A conversion script transforms `data/SNI-data/sni-2025-struktur.xlsx` into `data/SNI-data/sni-2025-struktur.json` with a flat array of normalized entries
2. Each JSON entry contains: `code`, `level` (1-5), `section` (letter A-U), `titleSv`, `titleEn`, `parentCode` (nullable)
3. The `code` field is derived from the appropriate Excel column per level: `Avdelning` (L1), `Huvudgrupp` (L2), `Grupp` (L3), `Undergrupp` (L4), `Detaljgrupp` (L5)
4. The `parentCode` field is derived by walking the hierarchy (e.g., `01.110` → parent `01.11`, `01.11` → parent `01.1`)

### Database Model

5. Prisma model `SniCode` with fields: `id`, `code` (unique), `level` (1-5), `section` (letter A-U), `title_sv`, `title_en`, `parent_code` (nullable, self-reference)
6. Indexes on `code`, `level`, `section`, `parent_code`
7. Migration runs cleanly

### Data Seeding

8. Seed script loads all 1,882 SNI 2025 codes from `data/SNI-data/sni-2025-struktur.json`
9. Full 5-level hierarchy preserved: Avdelning (letter) → Huvudgrupp (2-digit) → Grupp (XX.X) → Undergrupp (XX.XX) → Detaljgrupp (XX.XXX)
10. Both Swedish and English titles stored
11. Seed is idempotent — can be re-run without duplicating data

### API Endpoints

12. `GET /api/sni/search?q={term}&limit={n}` — search by Swedish title or code, returns top matches (default limit 10)
13. Search is case-insensitive and handles partial matches (e.g., "bygg" matches "Byggverksamhet")
14. `GET /api/sni/{code}` — returns single SNI code with full hierarchy path (ancestors up to section)
15. Both endpoints return: `{ code, level, section, titleSv, titleEn, parentCode }`
16. Search results sorted by relevance (exact code match first, then title match)

### SNI Autocomplete Component

17. Reusable `SniCodeSelect` component at `components/features/onboarding/sni-code-select.tsx`
18. Input field with dropdown that searches on type (debounced 300ms)
19. Dropdown shows: SNI code + Swedish title (e.g., "62.010 — Dataprogrammering")
20. Selecting an item sets both `sniCode` and `industryLabel` on the form
21. Shows hierarchy breadcrumb on selection (e.g., "J > 62 > 62.0 > 62.01 > 62.010")
22. Replaces plain text input in `company-info-step.tsx`
23. Keyboard accessible — arrow keys to navigate, Enter to select, Escape to close

### Editability & Override

24. User can change the pre-selected SNI code via the autocomplete (overrides BolagsAPI value)
25. The `industry_label` field remains an editable text input below the SNI selector — auto-populated from the selected SNI title but freely editable for edge cases (e.g., niche business not fully captured by SNI description)
26. Both `sniCode` and `industryLabel` are persisted independently to `CompanyProfile`, so a user override is never lost

### SNI Code Format Normalization

27. A utility function `normalizeSniCode(raw: string): string` converts BolagsAPI format (`62010`) to dotted format (`62.010`)
28. When BolagsAPI auto-fill (Story 15.2) returns an SNI code, it is normalized before pre-selecting in the `SniCodeSelect`

### Testing

29. At least 10 unit tests covering:
    - Search: partial match, exact code match, no results, Swedish characters (å, ä, ö)
    - Lookup: valid code returns hierarchy, invalid code returns 404
    - Seed: correct count (1,882), hierarchy integrity (every child has valid parent)
    - Component: renders, searches, selects, keyboard navigation, industry label editable after selection
    - Normalization: `62010` → `62.010`, `01` → `01`, section letters unchanged

### Source Data: SNI 2025 Structure (`data/SNI-data/sni-2025-struktur.xlsx`)

**File**: Official SCB (Statistics Sweden) SNI 2025 classification structure.
**Format**: Excel with 1 sheet ("SNI 2025"), 1,882 data rows (+ 1 header).

**Columns** (CSV headers):
```
Nivå, Avdelning, Huvudgrupp, Grupp, Undergrupp, Detaljgrupp, Aktivitetsart, Title EN
```

**Column mapping by level:**

| Level | Swedish Name  | Code Column    | Example Code | Example Title SV                            |
|-------|--------------|----------------|-------------|---------------------------------------------|
| 1     | Avdelning    | `Avdelning`    | `A`         | JORDBRUK, SKOGSBRUK OCH FISKE             |
| 2     | Huvudgrupp   | `Huvudgrupp`   | `01`        | Jordbruk och jakt samt stödverksamhet...   |
| 3     | Grupp        | `Grupp`        | `01.1`      | Odling av ett- och tvååriga växter          |
| 4     | Undergrupp   | `Undergrupp`   | `01.11`     | Odling av spannmål, baljväxter...          |
| 5     | Detaljgrupp  | `Detaljgrupp`  | `01.110`    | Odling av spannmål, baljväxter...          |

**Distribution**: 22 sections + 87 main groups + 287 groups + 651 sub-groups + 835 detail groups = 1,882 total.

**Parent derivation logic**: Each level's parent is the code at level-1. For example:
- `01.110` (L5) → parent `01.11` (L4) → parent `01.1` (L3) → parent `01` (L2) → parent `A` (L1) → parent null

**Section inheritance**: Only L1 rows have the `Avdelning` column populated. L2-L5 rows inherit the section from the most recent L1 row above them in row order.


### SNI Code Format Normalization

BolagsAPI returns SNI codes in compact format without dots (e.g., `62010`). The official SNI 2025 structure uses dotted notation (e.g., `62.010`). The normalization rule is simple: **for all-digit strings longer than 2 characters, insert a dot after position 2**. Examples:
- 5-digit codes: `62010` → `62.010`, `41000` → `41.000`
- 4-digit codes: `0111` → `01.11`, `4100` → `41.00`
- 3-digit codes: `011` → `01.1`
- 2-digit codes: `62` → `62` (no change)
- 1-character codes: `J` → `J` (section letters, no change)
- Already-dotted codes: `62.010` → `62.010` (passthrough)

**Note**: BolagsAPI typically returns the most specific (Level 5, Detaljgrupp) code as 5 digits. The simple dot-insertion rule handles all levels correctly.

### API Route Pattern

Both endpoints follow the REST API pattern for reference data (no auth required — SNI codes are public data):

```typescript
// app/api/sni/search/route.ts
import { NextResponse } from 'next/server'
import { z } from 'zod'
import { prisma } from '@/lib/prisma'

const SearchSchema = z.object({
  q: z.string().min(1).max(100),
  limit: z.coerce.number().int().min(1).max(50).default(10),
})

export async function GET(request: Request) {
  const { searchParams } = new URL(request.url)
  const parsed = SearchSchema.safeParse(Object.fromEntries(searchParams))
  if (!parsed.success) return NextResponse.json({ error: 'Invalid query' }, { status: 400 })

  const { q, limit } = parsed.data
  // ... search logic
}
```

### Search Performance

With 1,882 rows, a simple `contains` (case-insensitive) query on `title_sv` and `startsWith` on `code` is fast enough without full-text search. PostgreSQL `ILIKE` on a small table is <10ms. No caching needed for this dataset size.
