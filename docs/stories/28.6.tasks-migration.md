# Story 28.6: Tasks — Selection, Reorder & the Bulk-Bar Shell

**Epic:** 28 — Unified DataTable Core (`docs/prd/epic-28-unified-datatable-core.md`)
**Status:** Done (2026-07-08)

## Core capabilities added (with tasks as first consumer)

- **Column drag-reorder**: the table header wraps non-pinned columns in the
  shared `DraggableColumnHeader` whenever `columnState.order` +
  `onOrderChange` are provided; pinning comes from `meta.dt.pinned` (the
  legacy PINNED_COLUMN_IDS magic-string sets are gone).
- **`DataTableBulkBar` shell** (`data-table/bulk-action-bar.tsx`): count,
  clear, `role="toolbar"` semantics and loading spinner — domain actions
  are children. The laglistor bar adopts it in 28.8.

## Migrations

- **`list-tab.tsx`** (1,211 → ~900 lines, all domain): Set-based selection
  (core-injected checkbox column replaces the hand-rolled one), Zustand
  store adapters (plain-value setters wrapped as updater-resolvers), client
  sorting, 56px virtualization, `cardBelow: 800`. Inline editors (status /
  priority / due-date / assignee) and their optimistic-update + rollback
  orchestration unchanged. Card face: status + priority as interactive
  badges, description with null-skip, assignee editor + name, due date,
  comments count + created in the footer.
  - AC: the non-functional dragHandle placeholder column is REMOVED.
  - AC: the inverted resize bounds are FIXED (dueDate 140/160→160 min 120;
    priority 100/130→130 min 100).
- **`all-work-tab.tsx`** (466 → ~430 lines): read-only archive on the core;
  its byte-identical local SortableHeader copy deleted; `getRowClassName`
  carries the is_done opacity + overdue left-border accents; CSV export
  untouched.
- **`bulk-action-bar.tsx`** (tasks): adopted the shell — owns only the
  move/assign/priority/delete actions now.
