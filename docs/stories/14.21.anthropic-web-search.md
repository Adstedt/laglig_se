## Technical Notes

- **Integration approach:** Add `web_search: anthropic.tools.webSearch_20250305({...})` to the `tools` object passed to `streamText`. Only apply when `AI_CHAT_MODEL === 'anthropic'`; the OpenAI path remains unchanged.
- **Existing pattern reference:** `createAgentTools()` returns a tools object; `web_search` merges in as an additional key for Anthropic runs only.
- **Source events vs tool-result events:** Anthropic web_search emits `source` stream events (distinct from `tool-result` that our own tools emit). The `messageMetadata` callback currently only handles `'tool-result'` via `extractSourcesFromToolResult`. Add a new branch for `part.type === 'source'` that:
  - Reads `part.source.url`, `part.source.title`
  - Generates a stable key (e.g., `web:${url}`) for deduplication
  - Merges into `citationSources` (first-write-wins, consistent with existing behavior)
- **Optional tiering:** Consider varying `maxUses` by `contextType` (3 for `change`/`task`, 2 for `law`/`global`). If tiered with Story 14.20, both configs can share the same context map.

## Risk and Compatibility Check

**Minimal Risk Assessment:**

- **Primary Risk:** Model over-relies on `web_search` when `search_laws` would suffice (wasted cost + slower), OR cites secondary commentary in a way that reads as authoritative law.
- **Mitigation:** System prompt explicitly orders sources (primary â†’ agency â†’ commentary). `maxUses: 3` cap. Whitelist excludes all unvetted domains. Commentary sites are kept at the bottom of the hierarchy.
- **Rollback:** Remove the `web_search` key from the `tools` object and the `source`-event handler in `messageMetadata`. Revert system prompt. Single-PR revertable.

**Compatibility Verification:**

- [ ] No breaking changes to APIs
- [ ] No database changes
- [ ] UI reuses existing citation pill components (no new primitives)
- [ ] Performance: additional tool-use rounds bounded by `maxUses: 3` and existing `stopWhen: stepCountIs(10)`
