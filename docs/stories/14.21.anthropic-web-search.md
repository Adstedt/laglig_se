# Story 14.21: Anthropic Web Search with Swedish Legal Domain Whitelist

**Epic:** 14 (AI Chat)
**Status:** Ready for Review
**Size:** Single session (~4–5 hours)

## Story

**As a** Laglig.se user asking about content outside our curated law database (court rulings, propositions, agency guidance, kollektivavtal commentary, recent news of upcoming changes),
**I want** Lexa to fetch authoritative Swedish legal sources from the web and cite them inline with clickable pills that open a sidebar detail,
**so that** I get grounded, verifiable answers instead of "I don't have that information" or a confident hallucination.

## Acceptance Criteria

### Tool registration

1. `anthropic.tools.webSearch_20250305()` is registered on the Anthropic model path (`AI_CHAT_MODEL === 'anthropic'`) in the `tools` object passed to `streamText`, with:
   - `allowedDomains` → `SWEDISH_LEGAL_WEB_DOMAINS` (see Domain Whitelist below)
   - `maxUses: 3`
   - `userLocation: { type: 'approximate', country: 'SE', timezone: 'Europe/Stockholm' }`
2. `SWEDISH_LEGAL_WEB_DOMAINS` is exported as a named, typed constant (not env-var) from a dedicated file `lib/agent/web-search-config.ts`, so the whitelist is reviewable at a glance and importable by tests.
3. The `web_search` tool is only spread into `tools` when the model provider is `'anthropic'`; the OpenAI path remains unchanged.

### System prompt

4. The system prompt in `lib/agent/system-prompt.md` is updated with sourcing hierarchy guidance inside `<tool_guidance>`:
   > "Föredra `search_laws` för primär lagtext från vår kurerade databas. Använd `web_search` enbart för (a) innehåll som inte finns i databasen — domstolsavgöranden, propositioner, myndighetsvägledning, kollektivavtalskommentarer — eller (b) nyligen publicerade ändringar. Behandla advokatbyråbloggar och branschkommentarer som *tolkning*, inte som lag. Citera alltid webbkällor med [Källa: {sidtitel}] — samma format som för databaskällor."

### Source capture

5. `SourceInfo` in `lib/ai/citations.ts` is extended with `url?: string | null` (default `null`). Existing DB sources are unaffected — they continue to use `slug` + `anchorId`.
6. `ChatMessageMetadata` is unchanged structurally — `citationSources` already carries `Record<string, SourceInfo>`.
7. The `messageMetadata` callback in `app/api/chat/route.ts` is extended with a new branch for `part.type === 'source'` (Anthropic web search emits `source` stream events distinct from `tool-result`). The handler:
   - Reads `part.source.url` and `part.source.title`
   - Generates a stable key: `web:${url}`
   - Creates a `SourceInfo` entry: `{ documentNumber: domain, title, snippet: null, slug: null, path: null, anchorId: null, url }`
   - Merges into `citationSources` (first-write-wins, consistent with existing behavior)
   - A type assertion or type guard may be needed if `TextStreamPart<ToolSet>` doesn't include `'source'` as a discriminant — the dev should cast `part as { type: 'source'; source: { url: string; title: string } }` behind a `part.type === 'source'` guard.

### Citation rendering — inline pill

8. `resolveSource()` in `lib/ai/citations.ts` is extended with a web-source fallback path. After existing SFS-based matching fails, it scans the source map for entries keyed with `web:` prefix and matches by title (case-insensitive `startsWith` on the label against the source's `title`). Returns the first match.
9. `CitationPillInline` in `components/features/ai-chat/citation-pill.tsx` detects web sources (resolved `SourceInfo` has `url` and no `slug`):
   - **Hover card**: Shows the page title and domain (extracted from URL), with a small external-link icon.
   - **Click**: Opens the sidebar detail panel via `chatDetail.openDetail()` with a `{ type: 'citation' }` detail item that includes the `url` in its data payload.
   - **Pill appearance**: Renders with the same pill styling as DB citations, but with a `Globe` icon (from lucide-react) instead of the default. `Globe` is the firm choice — `ExternalLink` is already used for the "Visa i lagläsaren" link inside the hover card (`citation-pill.tsx:15`) and reusing it on the pill itself would blur the distinction.

### Citation rendering — sidebar detail

10. `CitationDetail` in `components/features/ai-chat/details/citation-detail.tsx` is extended to handle web sources:
    - When `data.url` is present: shows the page title, domain, and a "Besök källa" button/link that opens the URL in a new tab (`target="_blank"`, `rel="noopener noreferrer"`).
    - When `data.url` is absent: existing behavior (snippet + "Visa i lagläsaren" link).
11. The `CitationDetailData` type in `lib/ai/chat-detail-context.tsx` is extended with `url?: string`.

### Tool display

12. `TOOL_CONFIG` in `components/features/ai-chat/chat-message.tsx` has an entry for `web_search`:
    - `label: 'Söker på webben'`
    - `doneLabel: 'Sökte på webben'`

### Integration

13. Existing tools (`search_laws`, `get_document_details`, `get_change_details`, etc.) continue to work unchanged.

### Data Model — Extended `SourceInfo`

```typescript
export interface SourceInfo {
  documentNumber: string       // SFS number for DB sources, domain for web sources
  title: string | null
  snippet: string | null
  slug: string | null          // Internal navigation — null for web sources
  path: string | null          // Chunk path — null for web sources
  anchorId: string | null      // Deep-link anchor — null for web sources
  url?: string | null          // NEW: external URL — null/undefined for DB sources
}
```

The `url` field is the discriminator: `url` present + `slug` absent = web source; `slug` present + `url` absent = DB source.

### API — Anthropic Web Search Tool

The Vercel AI SDK exposes Anthropic's web search via:

```typescript
import { anthropic } from '@ai-sdk/anthropic'

anthropic.tools.webSearch_20250305({
  allowedDomains: string[],       // Whitelist
  maxUses: number,                // Max searches per response
  userLocation: {
    type: 'approximate',
    country: string,              // ISO 3166-1 alpha-2
    timezone: string,             // IANA timezone
  },
})
```

Returns a tool object that can be spread into the `tools` param of `streamText()`.

**Version note:** The project uses `@ai-sdk/anthropic` ^3.0.23. The tool factory name `webSearch_20250305` and the `source` stream event shape come from the Vercel AI SDK docs and the Anthropic API — verify against the installed version at implementation time. Check `node_modules/@ai-sdk/anthropic/dist/index.d.ts` for the exact export name if TypeScript can't resolve `anthropic.tools.webSearch_20250305`.

**Stream events:** When the model invokes web search and uses results in its response, the AI SDK emits `source` stream parts with shape `{ type: 'source', source: { url: string, title: string } }`. These arrive in the `messageMetadata` callback alongside `tool-result` parts.

### Citation Rendering — How It Works for Web Sources

The existing citation pipeline is reused end-to-end:

1. **Model output:** `"Arbetsdomstolen har fastslagit att...[Källa: AD 2024 nr 12 - Arbetsdomstolen]"`
2. **`rehypeCitationPills`** (unchanged): transforms `[Källa: AD 2024 nr 12 - Arbetsdomstolen]` → `<cite>AD 2024 nr 12 - Arbetsdomstolen</cite>`
3. **`CitationPillInline`**: calls `resolveSource("AD 2024 nr 12 - Arbetsdomstolen", sourceMap)` — existing SFS matching fails, web-source fallback matches by title → returns `SourceInfo` with `url`
4. **Render**: Globe icon pill, hover card shows title + domain, click → sidebar detail
5. **`CitationDetail`**: "Besök källa" external link

**Unresolved web citation fallback:** If `resolveSource` fails to match a web citation label, the pill renders as plain muted text (existing behavior at `citation-pill.tsx:140–146`). This is acceptable for v1. A future improvement could try to extract a domain from the label text and show it as a muted domain badge, but this is not in scope.

**Alternative approach (not used in v1):** Handle `SourceUIPart` message parts directly in the `ChatMessage` rendering loop. This would render web citations as separate elements below the text rather than inline. Deferred — inline pills are more consistent with the existing UX.

### Technical Constraints

- **TypeScript strict mode** — `noImplicitAny`, `strictNullChecks`, `exactOptionalPropertyTypes` are on. The `url` field on `SourceInfo` must be optional (`url?: string | null`) to avoid breaking existing call sites that don't set it [Source: architecture/17-coding-standards.md#172].
- **No new npm dependencies** — `@ai-sdk/anthropic` is already in the dependency tree; `Globe` is already available from lucide-react.
- **shadcn/ui mandatory** — the "Besök källa" button in `CitationDetail` must use the shadcn `Button` component [Source: architecture/17-coding-standards.md#173].
- **`source` stream part typing** — `TextStreamPart<ToolSet>` may not include `'source'` as a union member. Use a string comparison guard (`part.type === 'source'`) with a targeted type assertion. This is safe because the AI SDK emits this part type for Anthropic web search — the types may simply lag behind the runtime.

## Risk and Compatibility Check

**Minimal Risk Assessment:**

- **Primary Risk:** Model over-relies on `web_search` when `search_laws` would suffice (wasted cost + slower), OR cites secondary commentary in a way that reads as authoritative law.
- **Mitigation:** System prompt explicitly orders sources (database → primary → agency → commentary). `maxUses: 3` cap. Whitelist excludes all unvetted domains.
- **Secondary Risk:** `resolveSource` title matching is imprecise — web citations may render as unresolved pills if the model's `[Källa: ...]` label doesn't match the `source` event title closely enough.
- **Mitigation:** The match uses a case-insensitive prefix comparison (first 20 chars). In practice, the model tends to repeat the exact page title from the search result. Unresolved pills degrade gracefully (render as plain muted text, no crash).
- **Rollback:** Remove `web-search-config.ts`, revert the `tools` spread in the chat route, revert the `source`-event handler in `messageMetadata`, revert `SourceInfo` extension, revert `CitationPillInline`/`CitationDetail` changes, revert system prompt. Single-PR revertable.

**Compatibility Verification:**

- [ ] No breaking changes to APIs
- [ ] No database changes
- [ ] UI extends existing citation pill + sidebar detail components (no new primitives)
- [ ] Performance: additional tool-use rounds bounded by `maxUses: 3` and existing `stopWhen: stepCountIs(10)`
