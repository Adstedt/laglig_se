# Story 22.1: Badge `tone` primitive + Priority enum alignment

## Status

Draft

## Story

**As a** developer working on workspace UI,
**I want** a single `Badge` primitive with semantic `tone` and `variant` props backed by a shared token map, and a single `PriorityEditor` shared by Laglistor and Uppgifter,
**so that** every status, severity, and priority value renders identically across surfaces, and adding a new tabular surface composes from existing tokens rather than reinventing inline pill classes.

## Acceptance Criteria

### Functional

1. **`Badge` primitive extended.** `components/ui/badge.tsx` gains a `tone` prop (`"neutral" | "info" | "success" | "warning" | "danger"`, default `"neutral"`) and a `variant` prop (`"soft" | "solid" | "outline"`, default `"soft"`) via a `cva` config. Existing shadcn variants (`default`, `secondary`, `destructive`, `outline`) remain available for non-status uses but become deprecated for status / priority / severity (JSDoc comment in the file steers new code to `tone`).
   - Class strings per cell pulled from `_prototypes/ui-alignment-prototype.html` §1 matrix:
     - `soft` (default): `inline-flex items-center rounded-full px-2.5 py-0.5 text-xs font-medium bg-{tone}-500/15 text-{tone}-300` for dark theme; `bg-{tone}-100 text-{tone}-700` light analogue retained in token map for future use.
     - `solid`: `bg-{tone}-500 text-white` (warning uses `text-amber-950` for contrast).
     - `outline`: `border border-{tone}-700 text-{tone}-300` + transparent background.
   - All variants share: `rounded-full`, `px-2.5 py-0.5`, `text-xs`, `font-medium`. **No `whitespace-nowrap`** (it caused past drift between Laglistor and Cycle pills — let the consumer decide).

2. **Token module created.** `lib/ui/badge-tones.ts` exports:
   - `type Tone = 'neutral' | 'info' | 'success' | 'warning' | 'danger'`
   - `type Variant = 'soft' | 'solid' | 'outline'`
   - `BADGE_TONES: Record<Tone, Record<Variant, string>>` — single source of truth; consumed by Badge's `cva`.
   - `getStatusBadgeProps(domain, value): { tone: Tone; variant: Variant; label: string }` for the four status domains: `'compliance-status' | 'cycle-status' | 'document-status' | 'finding-severity' | 'finding-type'`. Returns Swedish labels.
   - `getPriorityBadgeProps(value: 'HIGH' | 'MEDIUM' | 'LOW'): { tone, variant, label }` — returns `{ tone: 'danger' | 'warning' | 'neutral', variant: 'soft', label: 'Hög' | 'Medel' | 'Låg' }` per Option A from the prototype (rose / amber / slate ramp via the danger / warning / neutral tokens).

3. **Compliance status migration.** `components/features/document-list/table-cell-editors/compliance-status-editor.tsx` (line 26 area) replaces its inline `span` render with `<Badge {...getStatusBadgeProps('compliance-status', value)} />`. Domain mapping:
   - `EJ_PABORJAD → tone: 'neutral', variant: 'soft'`
   - `PAGAENDE / DELVIS_UPPFYLLD → tone: 'info', variant: 'soft'`
   - `UPPFYLLD → tone: 'success', variant: 'soft'`
   - `EJ_UPPFYLLD → tone: 'danger', variant: 'soft'`
   - `EJ_TILLAMPLIG → tone: 'neutral', variant: 'outline'` (visually distinct from "not started" by the outline treatment)

4. **Cycle status migration.** `components/features/compliance-audit/cycle-detail/cycle-status-badge.tsx` (line 11 area) replaces the inline `STATUS_VARIANTS` map with `<Badge {...getStatusBadgeProps('cycle-status', value)} />`. Mapping:
   - `PLANERAD → tone: 'neutral', variant: 'soft'`
   - `PAGAENDE → tone: 'info', variant: 'soft'`
   - `AVSLUTAD → tone: 'success', variant: 'soft'`
   - `SEALED → tone: 'success', variant: 'solid'` (solid signals "locked / final")
   - `ARKIVERAD → tone: 'neutral', variant: 'outline'`
   - The inline `Pågående` pill in `cycle-detail-header.tsx` and the row pill on `cycle-list-table.tsx` both consume `<CycleStatusBadge>` — verify no duplicate inline pill render leaks through after migration.

5. **Document status migration.** `components/features/documents/document-status-badge.tsx` (line 32 area) — already wraps `Badge`; refactor to pass `tone` instead of custom className overrides. Mapping:
   - `DRAFT → tone: 'neutral', variant: 'soft'`
   - `IN_REVIEW → tone: 'info', variant: 'soft'`
   - `APPROVED → tone: 'success', variant: 'soft'`
   - `SUPERSEDED → tone: 'neutral', variant: 'outline'`
   - `ARCHIVED → tone: 'neutral', variant: 'outline'`
   - **Acknowledged visual delta:** "Utkast" (currently solid `bg-secondary`) becomes soft-neutral; "Under granskning" (currently solid `bg-primary`) becomes soft-info. This intentionally aligns Styrdokument with the rest of the app per the audit DoD. Flag in PR description.

6. **Finding type, severity, state migration.** `components/features/compliance-audit/cycle-detail/cycle-findings-tab.tsx` row pills (currently `Avvikelse` / `Observation` / `Förbättringsförslag` for type; `Kritisk` / `Större` / `Mindre` for severity; `Öppen` / `Stängd` for state) consume the helper. Mapping:
   - **Type** (`getStatusBadgeProps('finding-type', …)`): `AVVIKELSE → danger+soft`, `OBSERVATION → warning+soft`, `FORBATTRING → info+soft`.
   - **Severity** (`getStatusBadgeProps('finding-severity', …)`): `KRITISK → danger+solid`, `STORRE → danger+soft`, `MINDRE → warning+soft`.
   - **State**: `Öppen → warning+outline`, `Stängd → success+outline` (outline reads as "secondary signal" per prototype convention; state isn't the row's primary identifier).

7. **Priority editor unification.** `components/features/document-list/table-cell-editors/priority-editor.tsx` (line 31 area) replaces its inline span render with `<Badge {...getPriorityBadgeProps(value)} />`. **Investigation step:** confirm whether Uppgifter's `components/features/tasks/task-workspace/list-tab.tsx` actually imports this file or has a parallel implementation (the audit found "Hög" rendering as `bg-orange-100 text-orange-700` on Uppgifter vs `bg-rose-100 text-rose-700` on Laglistor — the colour drift suggests divergent code paths). If parallel: delete the duplicate, redirect imports to the canonical editor, verify side-by-side that "Hög" and "Medel" render identically on both surfaces. Document the finding (and the deletion if applicable) in the story's Dev Agent Record. Acceptance: a single `PriorityEditor` is imported by both Laglistor and Uppgifter; grep `priority-editor` and `PriorityEditor` returns one definition site.

8. **Priority labels are Swedish.** All occurrences of `"Medium"` (English) as a priority value in user-facing UI are replaced with `"Medel"`. Search the codebase for the literal `"Medium"` in priority-rendering contexts (likely in task-related files); does NOT touch translations of the word in non-priority contexts (e.g., free-text descriptions). The DB enum value remains `MEDIUM` — only the display label changes. Per Option A from `_prototypes/ui-alignment-prototype.html` §2.

### Component Specifications

- **shadcn/ui is mandatory for all UI primitives** [Source: architecture/17-coding-standards.md#17.3]. Story 22.1 extends an existing shadcn primitive (`Badge`) — does not replace it. Existing variant API (`default | secondary | destructive | outline`) remains for non-status uses; the new `tone × variant` axis is additive.
- **Custom domain components must compose shadcn primitives, not raw HTML.** [Source: architecture/17-coding-standards.md#17.3]. `compliance-status-editor.tsx`, `cycle-status-badge.tsx`, `document-status-badge.tsx`, and `priority-editor.tsx` are domain components that must wrap `<Badge>` rather than render `<span>` with hand-rolled Tailwind classes.
- **Existing `cn` className helper** at `lib/utils/cn.ts` [Source: architecture/12-unified-project-structure.md#12.2] is used by all shadcn primitives and should be used to merge variant classes.

### Tech Stack & Versions

- **TypeScript 5.5+** with strict mode (`strict`, `noImplicitAny`, `strictNullChecks`, `exactOptionalPropertyTypes`) [Source: architecture/17-coding-standards.md#17.2; architecture/3-tech-stack.md#3.1]. The new `Tone`/`Variant` types in `badge-tones.ts` must use `as const` arrays + indexed-type pattern per the existing convention.
- **React 19** (bundled with Next.js 16) [Source: architecture/3-tech-stack.md#3.1]. Badge is a pure render component — no client-only APIs needed (`'use client'` not required for this primitive).
- **Tailwind CSS 3.4+** with shadcn/ui (Radix UI + Tailwind) [Source: architecture/3-tech-stack.md#3.1]. Class strings in `BADGE_TONES` must be valid Tailwind utility classes recognised by the project's existing config.
- **`cva` (class-variance-authority)** is the convention for shadcn variant configuration. The existing `components/ui/badge.tsx` uses `cva` already; extend the same `cva` call rather than wrapping it.
- **Lucide Icons 0.365+** [Source: architecture/3-tech-stack.md#3.1] — not directly used in this story but referenced because Badge consumers may pass icons; the prop API should not preclude a leading icon slot in future stories.

### Technical Constraints

- **Zero behaviour change.** No new server actions, no API changes, no data shape changes. Only rendered pill class strings differ. Editor `onChange` handlers stay identical. Editors keep the same trigger surface and dropdown composition.
- **Visual delta acknowledged on Styrdokument** (AC 5). "Utkast" and "Under granskning" pills become softer (move from solid shadcn variants to soft-neutral / soft-info). Intentional alignment per epic DoD; flag in PR description.
- **Dark theme is the active theme.** All token classes target dark first (`bg-{tone}-500/15 text-{tone}-300`); light analogues retained in `BADGE_TONES` map for future use but no surface today renders in light mode.
- **No new dependencies.** All work uses existing `cva`, Tailwind, and shadcn primitives.
- **Out of scope for 22.1 (handled by 22.2):** filter chips on `cycle-list-table.tsx` (Aktiva / Slutförda / …) and `cycle-findings-tab.tsx` (type / state filters). Touching those here would create a merge conflict with 22.2.
- **Coordination with active Epic 21:** if Epic 21 stories add new pill renders to `cycle-items-tab.tsx` or `cycle-findings-tab.tsx` while 22.1 is in flight, those should target the new `<Badge tone>` API directly. Flag in PR review.
- **TypeScript strict mode:** `noUncheckedIndexedAccess` is enabled [Source: architecture/17-coding-standards.md#17.2]. Indexing into `BADGE_TONES[tone][variant]` will return `string | undefined` — handle with a non-null assertion only if the type guarantees the cell exists, otherwise add a runtime fallback.
