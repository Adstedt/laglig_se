# Story 17.12: Document Linking to Tasks & Law List Items

## Status

Draft

## Story

**As a** workspace member,
**I want** to link documents to specific tasks and law list items,
**so that** I can trace which compliance documents satisfy which legal obligations.

## Story Context

**Existing System Integration:**

- Integrates with: `WorkspaceDocumentTaskLink` and `WorkspaceDocumentListItemLink` models from Story 17.1, task modal (`components/features/tasks/task-modal/`), legal document modal (`components/features/document-list/legal-document-modal/`), document editor from Story 17.2
- Technology: React, shadcn/ui, Prisma server actions
- Follows pattern: Existing file linking pattern — `FileTaskLink`/`FileListItemLink` with `linkFileToTask()`, `FilePickerModal` component
- Touch points: Task modal right panel, legal document modal tabs, document editor metadata, server actions

## Acceptance Criteria

**Linking from Task Modal:**

1. "Länka dokument" action available in task modal right panel (alongside existing evidence/files section)
2. Opens a document picker modal showing workspace documents — searchable by title, filterable by document type
3. Linked documents shown in task modal: title, type badge, status badge, version number, link to open document
4. Unlink action available (removes link, does not delete document)

**Linking from Legal Document Modal:**

5. "Dokument" tab added to `activity-tabs.tsx` in legal document modal (alongside existing Alla/Kommentarer/Uppgifter/Bevis/Historik tabs)
6. Same document picker functionality as task modal
7. Shows all documents linked to this law list item
8. Unlink action available

**Linking from Document Editor:**

9. Document detail/editor page shows "Länkade till" section listing all linked tasks and list items
10. Each linked item shows: type (task/lag), title, clickable link to open the task modal or law document modal
11. "Länka till uppgift" and "Länka till lagkrav" actions available from document page

**Auto-link on Creation:**

12. When creating a document from within a task context, offer to auto-link to that task
13. When creating a document from within a law list item context, offer to auto-link to that item

**Data Model:**

14. `WorkspaceDocumentTaskLink` and `WorkspaceDocumentListItemLink` used for all linking (separate from `FileTaskLink`/`FileListItemLink`)
15. A document can be linked to multiple tasks and multiple list items
16. A task or list item can have multiple linked documents

**Audit:**

17. Link creation and removal logged to ActivityLog

**Integration Requirements:**

18. Existing file/evidence linking in task modal and legal document modal unchanged
19. Documents section is additive — appears alongside existing evidence section

**Quality Requirements:**

20. Document picker loads within 1 second with up to 200 documents

**Server action pattern for linking:**

```typescript
// Add to existing app/actions/documents.ts (already has 'use server', prisma, withWorkspace imports)

export async function linkDocumentToTask(
  documentId: string,
  taskId: string
): Promise<ActionResult> {
  return await withWorkspace(async ({ workspaceId, userId }) => {
    // Verify document and task belong to workspace
    await prisma.workspaceDocumentTaskLink.create({
      data: { document_id: documentId, task_id: taskId, linked_by: userId },
    })
    // Log to ActivityLog
    return { success: true }
  })
}
```

**Performance Guidance (AC 20):**

The document picker must load within 1 second with up to 200 documents. Use client-side filtering over the full list — `getWorkspaceDocuments()` already supports fetching all workspace documents. Pre-fetch documents when the picker modal opens, filter client-side via cmdk's built-in search matching. For workspaces exceeding 200 documents in the future, consider server-side search with debounced queries.
