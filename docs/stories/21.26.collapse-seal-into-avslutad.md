# Story 21.26: Collapse SEAL into AVSLUTAD as terminal active state

> **Followed by Story 21.27 (2026-04-28)** — ARKIVERAD also collapsed into AVSLUTAD on the same "less ceremony" thesis. The lifecycle is now 3-state (`PLANERAD → PAGAENDE → AVSLUTAD`). This spec's references to ARKIVERAD as the post-AVSLUTAD retention state no longer apply; preserved as historical record.

## Status

Ready for Review (2026-04-28) — followed by 21.27

## Story

**As a** product owner,
**I want** the cycle lifecycle simplified from PLANERAD → PAGAENDE → AVSLUTAD → SEALED → ARKIVERAD to PLANERAD → PAGAENDE → AVSLUTAD → ARKIVERAD,
**so that** SMB self-audit users have one completion moment instead of two ceremonial steps, and engineering doesn't maintain a cryptographic seal-hash flow that no Nordic regulator requires.

## Context

Story 21.9 shipped a SEAL state with cryptographic ceremony (SHA-256 manifest hash, evidence snapshot table, irreversible lock with override panels). Post-shipping PO review concluded the two-step split is bureaucracy without payoff for Laglig's customer base:

- Nordic regulators (AML, AFS 2001:1, ISO certifying bodies, GDPR Art 30, Bokföringslagen) require documented audit records, named auditor, dated review, retention. None require cryptographic tamper-evidence.
- The split forces the same SMB user (typical: lead auditor = OWNER) to perform two ceremonial steps for one mental moment.
- The seal dialog's irreversibility friction (override panels + type-to-confirm gate) doesn't apply to AVSLUTAD because revert-to-PAGAENDE is the escape hatch.
- The cryptographic integrity layer (manifest, hash, evidence snapshot) doesn't appear in any customer workflow; it's marketing technology that never landed in the actual narrative.
