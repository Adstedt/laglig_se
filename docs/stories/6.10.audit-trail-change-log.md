# Story 6.10: Implement Audit Trail & Change Log

## Status

Draft

## Story

**As a** compliance team member,
**I want** a complete audit trail of all compliance activities — visible in modal history tabs and a global activity log page,
**so that** I can demonstrate accountability during audits and track who changed what and when.

## Acceptance Criteria

**List Item History (Legal Document Modal → Historik tab):**

1. Log entries for:
   - Compliance status changes (old → new)
   - Responsible person changes
   - Business context edits
   - Compliance actions edits
   - Priority changes
   - Task created/completed/deleted (linked tasks)
   - Evidence uploaded/deleted
   - Comments added/edited/deleted
2. Entry format: User avatar + name + Action description + old→new values + Timestamp (relative, with full date on hover)

**Task History (Task Modal → Historik tab):**

3. Already implemented in Story 6.6 — no changes needed. The existing `HistoryTab` component in `components/features/tasks/task-modal/history-tab.tsx` and `logActivity()` calls in `app/actions/task-modal.ts` provide full task audit trail coverage for: status, assignee, due date, priority, description, labels, evidence, law links, and comments.

**Global Activity Log:**

4. Activity log page at `/workspace/activity` (accessible from workspace navigation)
5. Filterable by: user (dropdown of workspace members), action type (multi-select), entity type (list_item/task/comment/evidence), date range (date picker)
6. Exportable as CSV for audit purposes (max 10,000 entries per export)
7. Retention: 2 years minimum (manual cleanup or cron)
8. Paginated with cursor-based "Load more" (50 items per page)
9. Each entry shows: timestamp, user avatar + name, action description (Swedish), entity type badge, old→new values

**Activity Logging Service:**

10. Shared `logActivity()` utility extracted from `task-modal.ts` into `lib/services/activity-logger.ts` — used by both task and list item server actions
11. All list item mutations in `legal-document-modal.ts` instrumented with activity logging (currently none of them log activities)

### Existing ActivityLog Infrastructure (DO NOT recreate)

The `ActivityLog` Prisma model already exists and is actively used by task modal actions:

```prisma
model ActivityLog {
  id           String   @id @default(uuid())
  workspace_id String
  user_id      String
  entity_type  String   // 'list_item', 'task', 'comment', 'evidence'
  entity_id    String
  action       String   // 'created', 'updated', 'deleted', 'status_changed', etc.
  old_value    Json?
  new_value    Json?
  created_at   DateTime @default(now())

  workspace Workspace @relation(fields: [workspace_id], references: [id], onDelete: Cascade)
  user      User      @relation(fields: [user_id], references: [id])

  @@index([workspace_id, created_at])
  @@index([entity_type, entity_id])
  @@index([user_id])
  @@map("activity_logs")
}
```

### Existing `logActivity()` Helper (Extract to Shared)

Currently a private function in `app/actions/task-modal.ts` (lines 207-227):

```typescript
async function logActivity(
  workspaceId: string,
  userId: string,
  entityType: string,
  entityId: string,
  action: string,
  oldValue?: unknown,
  newValue?: unknown
) {
  await prisma.activityLog.create({
    data: {
      workspace_id: workspaceId,
      user_id: userId,
      entity_type: entityType,
      entity_id: entityId,
      action,
      old_value: oldValue ? JSON.parse(JSON.stringify(oldValue)) : null,
      new_value: newValue ? JSON.parse(JSON.stringify(newValue)) : null,
    },
  })
}
```

Extract to `lib/services/activity-logger.ts` and import in both `task-modal.ts` and `legal-document-modal.ts`.

[Source: app/actions/task-modal.ts#logActivity]

### Technical Constraints

- **Performance:** Use existing indexes `[workspace_id, created_at]` and `[entity_type, entity_id]` for queries. Do not add full-text search on `old_value`/`new_value` JSON fields.
- **Retention:** 2 years minimum. Implement as a note/reminder for now; the actual cron cleanup can be done in Story 5.11 or a follow-up.
- **CSV export cap:** 10,000 entries max per export to prevent timeout on Vercel serverless (10-minute max).
- **date-fns:** Use `sv` locale for all Swedish date formatting (`formatDistanceToNow` with `addSuffix: true`).

[Source: docs/architecture/3-tech-stack.md#date-fns]
