# Story 14.17: Agent Action Plans — Multi-Step Sidebar Workflows

## Status

Backlog (post-MVP)

## Story

**As a** compliance officer reviewing an amendment assessment,
**I want** the agent to propose a checklist of relevant follow-up actions in the sidebar,
**So that** I can efficiently complete the full compliance workflow without prompting the agent for each step.

## Context

Today the sidebar shows a single detail view (e.g., an assessment). In practice, responding to an amendment involves multiple steps: reviewing the assessment, creating a task, checking external guidance, updating compliance status, notifying a colleague. The agent should be able to propose these as a suggested action plan.

## Requirements

### Action Plan Structure
- The agent returns a typed list of suggested actions alongside its primary response
- Each action has: label, type (assessment, task, status-update, external-link, custom), pre-filled parameters, and an optional rationale
- Actions can be completed in any order, skipped, or dismissed — not a linear wizard
- Completed actions show a checkmark; the plan persists until explicitly dismissed

### Agent Intelligence
- The agent decides which actions to suggest based on the amendment's severity, scope, and the user's compliance context
- Minor editorial changes → minimal plan (review + dismiss)
- Major structural changes → full plan (review + task + status update + notify)
- The action plan is a tool response, not hardcoded logic

### Sidebar UX
- The sidebar header shows the plan title and progress (e.g., "2 av 4 klara")
- Each action is a card that expands into its detail view (reusing existing detail components)
- Inline completion: e.g., "Skapa uppgift" opens the task form inside the action card
- External links open in a new tab with a return-to-plan affordance

### Scope Boundary
- Initial implementation targets the amendment assessment flow only
- Generalize to other workflows (onboarding, periodic review) only after validating the pattern with users

## Acceptance Criteria

1. Agent can return an action plan with 1–5 typed actions in a single tool response
2. Sidebar renders the plan as a checklist with expandable action cards
3. Actions can be completed, skipped, or dismissed independently
4. Completing an action (e.g., creating a task) updates the checklist state in real-time
5. Plan state is ephemeral (per-conversation, not persisted)
6. The agent varies suggested actions based on amendment severity and user context

## Design Notes

- Keep the plan lightweight — it should feel like a helpful suggestion, not a project management tool
- Reuse existing detail components (assessment-detail, write-preview-task, etc.) inside action cards
- Consider a `suggest_action_plan` tool that the agent calls after completing an assessment
