# Story 21.13: ActivityLog integration + `/workspace/activity` surface

## Status

Done

## Story

**As a** workspace admin or auditor viewing the global activity feed,
**I want** compliance-audit-cycle events (cycle + item mutations from Stories 21.2 and 21.5) to render alongside kravpunkter and task events with Swedish sentences, category badges, deep links, and entity-type filtering,
**so that** the timeline of compliance activity is unified and traceable without a separate UI — and so that later stories (21.6 completion, 21.7 findings, 21.9 seal) can slot their own action strings into the same infrastructure with a single-file edit.

   | Action | Sentence parts |
   |---|---|
   | `cycle_created` | `[user] skapade kontrollen [primary]` (primary = cycle) |
   | `cycle_metadata_updated` | `[user] uppdaterade metadata på kontrollen [primary]` — if `oldValue.name !== newValue.name`, also append `: namnet ändrades från [old] till [new]`. Other field changes NOT individually rendered (expanded-row view handles the full diff per existing convention). |
   | `cycle_soft_deleted` | `[user] tog bort kontrollen [primary]` |
