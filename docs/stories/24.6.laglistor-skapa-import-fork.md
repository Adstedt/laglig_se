# Story 24.6: `/laglistor/skapa` — add Generate / Import path-choice fork

## Status

Draft

## Story

**As an** existing workspace member who wants to create a second (or third) law list — perhaps for a sibling entity or to migrate mid-contract from a competitor tool,
**I want** the in-app `/laglistor/skapa` page to offer the same Generate-or-Import choice that new users see during onboarding (Epic 25 B.1),
**so that** I can bring an existing curated list into Laglig without going through the post-signup modal — which I no longer trigger because my workspace isn't fresh.

This is the **wiring story** — no new logic. It composes the existing Generate flow (already shipped) with the import flow (Stories 24.2 → 24.4) into a unified path-choice surface on `/laglistor/skapa`.

## Acceptance Criteria

### Path-choice on `/laglistor/skapa`

1. **Modify `app/(workspace)/laglistor/skapa/page.tsx`** to add a path-choice step at the top of the flow:
   - Renders **before** the existing Generate-flow steps when no path is yet selected
   - Two cards in a 2-column grid (matching prototype frame ① fidelity from Epic 25):
     - **Generera ny lista** (recommended chip, primary background) → routes to existing Generate flow
     - **Importera befintlig** → routes to import upload step
   - Subtitle: "Skapa en ny laglista — generera från er profil eller importera en befintlig."
   - "Tillbaka till mina laglistor" link in header

2. **Path state in URL.** Use `?path=generate|import` query parameter to track which path the user is on. No path = path-choice step renders. With path = the corresponding flow renders. Allows browser back-button to return to path-choice.

3. **Generate path** — when `?path=generate`:
   - Renders the existing Generate-flow step(s) unchanged. (Whatever currently lives at `/laglistor/skapa` — typically a form to name the list + optional template selection + AI customisation.)
   - Existing tests pass unchanged.

4. **Import path** — when `?path=import`:
   - Renders `<ImportUploadStep>` (the component built in Story 24.2)
   - Submit → `createImport` + `parseImportFile` → `runMatching` (fire-and-forget; user redirected to granska page where they wait for completion)
   - Routes to `/laglistor/skapa/[importId]/granska` on success

### Reuse vs duplicate

5. **`<ImportUploadStep>` component** must be reused from Story 24.2 unchanged — same component, same submit behaviour. No styling fork.

6. **`commitImport`'s redirect target** is the same regardless of entry point: `/laglistor/{newLawListId}`. The user lands in their normal workspace's law list view; no special "imported from" badge or banner needed.

### Header / breadcrumb adjustments

7. **Breadcrumb** on `/laglistor/skapa` updates based on `path` param:
   - No path: `Mina laglistor → Skapa ny lista`
   - `?path=generate`: `Mina laglistor → Skapa ny lista → Generera`
   - `?path=import`: `Mina laglistor → Skapa ny lista → Importera`

   **Implementation note:** the `/laglistor` page already uses the Epic 22 `<PageHeader>` primitive (verified via `app/(workspace)/laglistor/page.tsx` line 25). `<PageHeader>` exposes a `breadcrumbs` slot — pass an array of `{label, href}` items. If `/laglistor/skapa/page.tsx` does not yet use `<PageHeader>`, this story migrates it (small additive change).

8. **Page title** (browser tab) likewise updates: `Skapa ny lista | Laglig.se` / `Generera ny lista | Laglig.se` / `Importera lista | Laglig.se`.

### Empty-state hint on `/laglistor`

9. **Update `/laglistor` empty state copy** (when user has zero law lists) to mention import:
   - Was: "Du har inga laglistor än. Skapa din första lista." with a single CTA
   - Now: "Du har inga laglistor än. Skapa en ny eller importera en befintlig." with the existing primary CTA + a small text-link "eller importera en lista" routing to `/laglistor/skapa?path=import`

   **Implementation note:** the empty state likely renders inside `components/features/document-list/document-list-page-content.tsx` (loaded from `app/(workspace)/laglistor/page.tsx` line 20). Read that file first to confirm where the empty state branch is. Search for "inga" or "skapa din första" string fragments. The empty state may also be inside `LawListPrimaryAction` (line 23) — both candidates worth checking.

### Coordination with Epic 25

10. **Epic 25 B.1's import card** routes to `/laglistor/skapa?path=import` — same surface as 24.6 (modal closes; user lands on the create-list page in the import path). This is a **convention, not code change**: Story B.1 already routes there per its acceptance criteria. This story just verifies that the route accepts and handles that param correctly.

    **Smoke verification step (manual):** once Epic 25 B.1 is also live, open the first-run modal as a fresh user, click the Importera card, confirm the URL is `/laglistor/skapa?path=import` and the import upload step renders. Document result in story Completion Notes. If B.1 has not yet shipped at the time 24.6 is QA'd, this verification is deferred — the route should still accept the param and render correctly when navigated to directly.

11. **`<ImportUploadStep>`** is the same component used by both surfaces (B.1's modal-internal import step AND 24.6's full-page step). When inside the modal it renders inside a Dialog; when on `/laglistor/skapa?path=import` it renders inside the page layout. The component does not own its container chrome — the parent does.

### Compatibility & out-of-scope guards

15. **Existing Generate flow is unchanged in behaviour and tests.** Whatever the current `/laglistor/skapa` page does (template picking, AI customisation, etc.) renders identically when `?path=generate` is present.

16. **No first-run modal interaction.** This story does not modify Epic 25's modal; the modal handles its own first-run entry point. `/laglistor/skapa` is the in-app/non-onboarding surface.

17. **No new server actions.** This story is pure routing + composition.

18. **No schema changes.**

19. **Stories 24.2 (upload), 24.3 (matching), 24.4 (review)** must be live for the import path to work end-to-end. If any of those stories haven't shipped at the time 24.6 is merged, the import card should remain disabled with a "Kommer snart" badge — but ideally 24.6 is sequenced last in Epic 24 so all dependencies are present.

## Dev Notes

**Source artefacts:**
- Epic PRD: `docs/prd/epic-24-import-existing-law-list.md` — see "Story 24.6" — minimal wiring story
- Story 24.2 — `<ImportUploadStep>` component
- Story 24.4 — `/laglistor/skapa/[importId]/granska` route
- Epic 25 B.1 — sibling surface that lands in the same place via the modal's import card

**Existing patterns to follow:**
- **URL-param-driven step state** — see `/laglistor/kontroller/skapa` (cycle creation wizard) for the pattern of using URL params to track wizard step. Mirror that here.
- **Generate flow** — currently on `/laglistor/skapa` (this story's modification target). Read it first to understand what should render under `?path=generate`.
- **Path-choice cards visual** — `_prototypes/onboarding-tutorial-modal.html` frame ① is the visual reference. Adapt for full-page (no modal chrome).

**Key decisions baked in:**
- **URL state, not React state.** Using `?path=...` for step tracking ensures browser back works and the path is shareable. React-only state would lose the back-button affordance.
- **Path-choice is rendered above the existing Generate flow, not as a separate route.** Keeps `/laglistor/skapa` as the canonical create-list URL. Both paths still complete on the same URL prefix.
- **No special handling for "imported from" lists.** Once committed, an imported list is indistinguishable from a generated list (same `LawList` schema). The fact that it came from import is in the `committed_law_list.import_committed_from` back-reference + activity log. Users don't need to see it in the UI unless we add an admin / debug surface.
- **Empty-state mention of import is intentional.** Surfaces the import path to existing workspaces that haven't tried it yet — workspace owners see the option without having to find the create-list page.

**Out of scope explicitly:**
- Onboarding modal changes (Epic 25's territory)
- Re-import support (delete + re-upload is the only "redo")
- Differentiation of imported lists in the `/laglistor` index view
