# Story 17.13: Document Browser & Manager UI

## Status

Ready for Review

## Story

**As a** workspace member,
**I want** a dedicated document management page to browse, filter, and manage all workspace documents,
**so that** I can find and organize compliance documentation efficiently.

## Story Context

**Existing System Integration:**

- Integrates with: `WorkspaceDocument` model (from Story 17.1), all document server actions in existing `app/actions/documents.ts` (~1000 lines, 13 exported actions from Stories 17.1-17.7), workspace sidebar navigation (`components/layout/left-sidebar.tsx` — already has "Dokument" entry at `/workspace/documents` from Story 17.2)
- Technology: React, shadcn/ui Table + Checkbox + Popover, Tailwind CSS
- Follows pattern: Existing file browser layout (`components/features/files/file-list-view.tsx` — sortable table with checkboxes), activity filter URL-sync pattern (`components/features/activity/activity-filters.tsx` — `useSearchParams` for bookmarkable filters)
- Touch points: New page at `app/(workspace)/workspace/documents/page.tsx`, reuses `CreateDocumentDialog` (17.2/17.7), `ImportDocumentDialog` (17.5), `DocumentStatusBadge` (17.4), `getReviewDateStatus` (17.4)

**Model naming — `WorkspaceDocument*` prefix required:**
- `WorkspaceDocument` (Prisma: `prisma.workspaceDocument`)
- `WorkspaceDocumentType` enum, `WorkspaceDocumentStatus` enum

**What already exists from previous stories (do NOT re-implement):**

- `getWorkspaceDocuments()` in `app/actions/documents.ts` (line 301) — already supports `type`, `status`, `search` filters + cursor-based pagination with `take + 1` pattern. Currently hardcoded to `orderBy: { updated_at: 'desc' }`. Does NOT support dynamic `sortBy`/`sortOrder` or `createdBy` filter — these need to be added if required.
- `DocumentStatusBadge` at `components/features/documents/document-status-badge.tsx` — reusable status badge with `STATUS_CONFIG`
- `getReviewDateStatus()` at `lib/utils/review-date-status.ts` — returns `'overdue' | 'upcoming' | 'normal' | null`
- `CreateDocumentDialog` at `components/features/documents/create-document-dialog.tsx` — with template selection (17.7)
- `ImportDocumentDialog` at `components/features/documents/import-document-dialog.tsx` — batch .docx import (17.5)
- `DOCUMENT_TYPE_LABELS` — defined in multiple components (create-document-dialog, document-settings-panel)
- Sidebar "Dokument" entry already points to `/workspace/documents` (line 117-119 of left-sidebar.tsx)

## Acceptance Criteria

**Document Browser Page:**

1. Document browser page accessible at `/workspace/documents`
2. Table/list view showing columns: title, document number, type badge, status badge, version ("v{N}"), author name, last updated date, review date
3. Rows clickable — navigates to document editor (`/workspace/documents/{id}/edit`)

**Filtering & Sorting:**

4. Filter by document type (multi-select dropdown from `WorkspaceDocumentType` enum)
5. Filter by status (multi-select dropdown from `WorkspaceDocumentStatus` enum)
6. Search by title (text search, debounced input)
7. Sortable columns: title, last updated, created date, review date
8. Default sort: last updated descending (already the default in `getWorkspaceDocuments`)

**Actions:**

9. "Nytt dokument" primary action button (opens `CreateDocumentDialog` from 17.2/17.7)
10. "Importera" button for .docx import (opens `ImportDocumentDialog` from 17.5)
11. Row-level actions (dropdown): Öppna, Exportera som Word, Exportera som PDF, Arkivera

**Review Date Indicators:**

12. Review date column: overdue dates in red text, upcoming (within 30 days) in amber, future in normal — use `getReviewDateStatus()` from `lib/utils/review-date-status.ts`
13. Optional: count of documents with overdue reviews shown as badge at top of page

**Empty & Loading States:**

14. Empty state: illustration/message with "Skapa ditt första dokument" CTA button
15. Loading skeleton while documents load
16. "Ladda fler" button for cursor-based pagination (loads next page)

**Integration Requirements:**

17. Existing file browser ("Mina filer") unchanged — remains at `/documents` as separate nav entry
18. Sidebar "Dokument" nav entry links to this page — already configured, verify only

**Quality Requirements:**

19. Page loads within 2 seconds for workspaces with up to 500 documents
20. Filters respond within 500ms

export const getWorkspaceDocumentsSchema = z.object({
  type: z.nativeEnum(WorkspaceDocumentType).optional(),
  status: z.nativeEnum(WorkspaceDocumentStatus).optional(),
  search: z.string().max(255).optional(),
  cursor: z.string().uuid().optional(),
  take: z.number().int().min(1).max(100).default(25),
})
