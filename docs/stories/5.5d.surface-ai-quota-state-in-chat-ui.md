# Story 5.5d: Surface AI Quota State in Chat UI

## Status

Draft

## Story

**As a** workspace user on a metered tier (SOLO / TEAM),
**I want** the chat UI to surface my AI token usage state — a soft warning when I approach the cap and an actionable upgrade flow when I cross it,
**so that** I'm never silently blocked from drafting, can take action before service interruption, and so a customer-facing failure ("Lexa won't write drafts") cannot recur from an unsurfaced HTTP 402.

## Acceptance Criteria

1. **Soft-warn banner appears above the chat input** when the most recent successful chat response carried the `X-AI-Usage-Warning` header. Banner text is Swedish, surfaces the usage % and approxQueriesRemaining, and includes an "Uppgradera" CTA linking to `/settings?tab=billing` (SOLO/TEAM) or a "Kontakta säljarna" mailto/secondary path (ENTERPRISE never sees this banner because Enterprise returns no warning per `lib/usage/check.ts:67` — verify and assert in tests).

2. **Banner persists within the conversation session** until dismissed: once shown, it stays visible across subsequent chat turns in the same session — even on turns whose response does NOT carry the warning header — so the user can act on it without it disappearing on the next message.

3. **Banner is dismissible per session.** Dismiss state persists via `sessionStorage` keyed by workspace id (`laglig:ai-quota-warning-dismissed:{workspaceId}`). New browser session → banner re-appears if usage is still ≥80%. Mirror the `TrialCountdownBanner` dismiss pattern exactly (try/catch sessionStorage, default to NOT dismissed on read failure).

4. **Banner copy is data-driven:** parsed from the JSON payload of `X-AI-Usage-Warning` (shape: `{ usage: number, limit: number, approxQueriesRemaining: number }` per `lib/usage/check.ts:38-42`). Format numbers with `toLocaleString('sv-SE')`. Example: `"Du har använt 2 430 000 av 3 000 000 tokens denna månad (81%) — ungefär 19 AI-frågor kvar. Uppgradera för att fortsätta utan avbrott."` The exact phrasing should be finalized during implementation; final copy is captured in the dev record.

5. **Hard-cap 402 renders a blocking, actionable modal** (NOT a generic error toast). Triggered when the chat returns HTTP 402 with `code === 'AI_TOKEN_QUOTA_EXCEEDED'`. Modal shows:
   - Title: `"AI-tokenkvot uppnådd"` (matches the route response body)
   - The Swedish `message` field from the 402 body verbatim (already includes usage / limit / percentage)
   - A `tier`-aware primary CTA: SOLO + TEAM → "Uppgradera planen" linking to `/settings?tab=billing`; ENTERPRISE → "Kontakta säljarna" linking to `mailto:sales@laglig.se` (or the existing enterprise contact path — verify against current billing UI)
   - A secondary line stating when the quota resets — sourced from `workspace.current_period_end` (fetched via a server action; format with `new Intl.DateTimeFormat('sv-SE', { dateStyle: 'long' })`)
   - Modal is dismissible but does NOT have a per-session-suppress mechanism (the user must take action OR the quota must reset)

6. **Chat input is disabled while the 402 modal is open** and a disabled-state hint appears: `"Skicka är pausat tills du uppgraderar eller kvoten återställs"`. Existing `Stoppa generering` / send disable logic is preserved (the new disable composes additively).

7. **Tier-aware copy throughout.** Both the soft-warn banner and the 402 modal show different CTAs per tier. SOLO sees an "Uppgradera till Team" framing; TEAM sees an "Lägg till fler platser eller uppgradera till Enterprise" framing; ENTERPRISE — does NOT see the soft-warn banner (unlimited tier returns no warning). Source tier from the 402 body's `tier` field for the modal, and from a server-action fetch for the banner. Reuse `lib/billing/tier-display.ts` copy primitives where applicable; do NOT hardcode tier names.

8. **The 402 body is parsed defensively.** The AI SDK's `onError` callback (`lib/hooks/use-chat-interface.ts:177-193`) currently inspects `err.message` as a string. To get the structured 402 body, extend the `DefaultChatTransport`'s `fetch` option with a wrapper that intercepts non-OK responses and stashes the parsed JSON body in a ref / state slot the hook exposes (`quotaError: { code, message, usage, limit, hardCap, tier } | null`). The hook MUST still surface a generic error for non-quota 4xx/5xx — only `AI_TOKEN_QUOTA_EXCEEDED` triggers the upgrade modal.

9. **Soft-warn header is parsed via the same transport-fetch wrapper.** On every successful chat response, the wrapper reads `response.headers.get('X-AI-Usage-Warning')`, JSON-parses it when present, and exposes it via the hook as `quotaWarning: { usage, limit, approxQueriesRemaining } | null`. State persists in the hook until the chat is unmounted (combined with sessionStorage dismiss state → AC 2).

10. **Telemetry events.** Emit `track('ai_chat_quota_warning_shown', { workspaceId, usage, limit })` on first banner render of a session. Emit `track('ai_chat_quota_warning_dismissed', { workspaceId, usage, limit })` on dismiss. Emit `track('ai_chat_quota_402_shown', { workspaceId, tier })` on 402 modal render. Emit `track('ai_chat_quota_upgrade_cta_clicked', { workspaceId, surface: 'banner' | 'modal' })` on CTA click. Use the existing `track()` import pattern from `lib/hooks/use-chat-interface.ts` — do NOT introduce a new analytics layer.

11. **All chat surfaces inherit the banner + modal.** This includes: the dashboard chat (`components/features/dashboard/hem-chat.tsx`), the law-list-item Lexa chat (mounted under `chat-modal.tsx` from the laglistor drawer), the task chat (`components/features/tasks/task-modal/ai-chat-panel.tsx`), the workspace document chat, and the assess-change chat. Because all surfaces consume `useChatInterface`, AC 1-10 implemented at the hook layer + a shared banner/modal component should naturally inherit. Verify visually on each surface.

12. **Accessibility:**
   - Soft-warn banner uses `role="status"` + `aria-live="polite"` (does NOT steal focus; mirror `TrialCountdownBanner`)
   - 402 modal uses `role="dialog"` + `aria-modal="true"` + initial focus on the primary CTA (use the existing shadcn Dialog primitive)
   - Dismiss button is keyboard-focusable with `aria-label="Stäng varning"` / `aria-label="Stäng AI-tokenkvot-dialog"`
   - All CTAs are real `<Link>` / `<a>` elements where they navigate, real `<button>` where they don't — no div+onClick

13. **No new external dependencies.** All UI uses existing shadcn primitives (Alert, Dialog, Button, Progress) and existing icons (`lucide-react`). All copy uses existing `lib/billing/tier-display.ts` helpers where applicable.

14. **Test coverage:** unit tests for the transport-fetch wrapper (header parse + 402 body parse + error fallthrough), component tests for the banner (sessionStorage dismiss + show conditions + tier-aware copy) and modal (render conditions + CTA wiring), and an E2E that drives a workspace past the soft-warn threshold via `scripts/set-nordviken-counter.ts` and asserts banner appears in the chat UI. Hard-cap E2E covered by an existing 5.5c test (`tests/e2e/chat-quota-hard-cap.spec.ts`) — extend it to assert the new modal renders instead of the generic error.

## Dev Notes

### Triggering-incident summary (2026-06-02)

Verbatim from `memory/project_quota_gate_failure_mode.md`:

> When a workspace hits its SOLO-tier hard cap (3M soft / 6M hard per `lib/usage/limits.ts`), the gate at `lib/usage/check.ts:69-81` throws `TokenQuotaExceededError` *synchronously before `streamText()` fires*. The route catches it at `app/api/chat/route.ts:158-173` and returns HTTP 402. There is no soft-throttle, no mid-stream abort, no tool-call refusal — it's a clean request-entry block.
>
> **Why it looked like a `draft_styrdokument` bug on 2026-06-02:** the draft tool is the heaviest single-call token consumer in the registry (generates full Tiptap JSON inline, ~1000-2500 tokens of *tool input*). When the workspace was near the cap, draft requests were the calls that pushed it over. Once over, every subsequent chat message — draft or not — got 402. But because the user's active intent was drafting at that moment, the UI symptom looked tool-specific.

### Exact contract — `X-AI-Usage-Warning` header

Set in `app/api/chat/route.ts:697-699`:

```ts
...(quotaWarningHeader
  ? { headers: { 'X-AI-Usage-Warning': quotaWarningHeader } }
  : {}),
```

Where `quotaWarningHeader = JSON.stringify(quota.warning)` and `quota.warning` is typed as `TokenQuotaWarning` from `lib/usage/check.ts:38-42`:

```ts
export interface TokenQuotaWarning {
  usage: number
  limit: number
  approxQueriesRemaining: number
}
```

**The header is set on the SUCCESS response (the 200 stream).** Soft-warn does NOT block the request. Header is set per turn — the soft-warn fires every turn ≥80% until hard cap. The client is responsible for de-duping ("show banner once per session, persist until dismiss or end of session").

### Transport wrapper pattern (AC 8, 9)

`DefaultChatTransport` from `@ai-sdk/react` accepts a `fetch` option. The current hook at `lib/hooks/use-chat-interface.ts:125-139` does NOT use it. The dev should:

1. Wrap the global `fetch` in a function that intercepts the response and stashes `X-AI-Usage-Warning` / 402-body data via refs/setState BEFORE returning the cloned response to the SDK
2. The cloned response MUST be returned untouched (do not consume the body for the success case) — the SDK reads the stream
3. The 402 case: the SDK's `onError` will fire after; the hook should branch on `quotaError !== null` to render the modal instead of the generic error

Recommended approach: keep the wrapper LOCAL to `useChatInterface` and pass it into the transport — do NOT monkey-patch `window.fetch`. Cleanest pattern:

```ts
const fetchWithQuotaInterception = useCallback(
  async (input: RequestInfo | URL, init?: RequestInit) => {
    const res = await fetch(input, init)
    if (res.status === 402) {
      const cloned = res.clone()
      try {
        const body = await cloned.json()
        if (body?.code === 'AI_TOKEN_QUOTA_EXCEEDED') {
          setQuotaError(body)
        }
      } catch {
        /* fall through — non-JSON 402 is a different problem */
      }
    } else if (res.ok) {
      const header = res.headers.get('X-AI-Usage-Warning')
      if (header) {
        try {
          setQuotaWarning(JSON.parse(header))
        } catch { /* malformed header — log + ignore */ }
      }
    }
    return res
  },
  []
)

const transport = useMemo(
  () => new DefaultChatTransport({
    api: '/api/chat',
    body: { /* existing */ },
    fetch: fetchWithQuotaInterception,
  }),
  [fetchWithQuotaInterception, contextType, contextId, lawListItemId, initialContext]
)
```
