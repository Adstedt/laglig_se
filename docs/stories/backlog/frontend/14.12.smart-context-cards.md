# Story 14.12: Smart Context Cards — Tool-Backed Responses

## Status

Draft

## Story

**As a** compliance manager viewing the Hem dashboard,
**I want** context card clicks to trigger agent tools that return structured, actionable output,
**so that** I get immediate, data-rich answers instead of generic conversational responses.

## Acceptance Criteria

1. Each context card's `prompt` is rewritten to instruct the agent to use specific tools and return structured output (tables, lists, actionable items) instead of generic prose.
2. The **Efterlevnad** card prompt instructs the agent to call `get_company_context` and present a compliance summary table grouped by status (non-compliant first, then partial, then compliant) with law name, status, and gap summary.
3. The **Ändringar** card retains the existing ChangePicker behavior when amendments > 0. When amendments = 0, the prompt instructs the agent to confirm no pending changes and suggest reviewing recent amendments.
4. The **Förfallna** card prompt instructs the agent to list overdue tasks with task name, due date, days overdue, and assignee — sorted by days overdue descending.
5. The **Denna vecka** card prompt instructs the agent to list this week's tasks with task name, priority, due date, and status — grouped by day.
6. All card prompts include a fallback instruction: if the tool call fails or returns empty data, the agent should provide a helpful text response explaining the situation.
7. Existing card UI (icons, colors, progress bar, pulse animation, loading skeletons) remains unchanged.
8. The `CardConfig` interface is extended with an optional `toolHint` field documenting which tools the prompt is designed to invoke (for developer reference only — not sent to the agent).

## Dev Notes

### Current Implementation (Story 14.11)

The Hem dashboard has 4 context cards in `components/features/dashboard/context-cards.tsx`. Each card has a `prompt` field — a Swedish text string sent as a user message to the chat when clicked. The agent responds conversationally.

**Current prompts:**
- Compliance: `"Visa en översikt av min efterlevnad"`
- Amendments: `"Visa mina väntande ändringar"`
- Overdue: `"Vilka uppgifter är förfallna?"`
- This week: `"Vad behöver jag göra denna vecka?"`

**Click handler** (lines 181-191): The `amendments` card has special behavior — if `pendingAmendments > 0`, it calls `onChangeClick()` (opens ChangePicker dialog) instead of sending the prompt.

### Available Agent Tools

The agent has these tools available (from `lib/agent/tools/index.ts`):

| Tool | Purpose | Relevant to Card |
|------|---------|-----------------|
| `get_company_context` | Company profile + compliance posture + law list stats | Efterlevnad |
| `search_laws` | Search legal documents by query | — |
| `get_document_details` | Full document content | — |
| `get_change_details` | Change event details | Ändringar |
| `create_task` | Create a new task | — |
| `update_compliance_status` | Update law compliance status | — |

The agent doesn't need explicit tool calls in the prompt — the system prompt already describes all available tools. The key is crafting prompts that **imply** tool use through the specificity of the request.

### Prompt Design Approach

Instead of vague prompts like "Visa en översikt", the upgraded prompts should be specific about the desired output format. The LLM will naturally use the right tools when the request is specific enough.

**Example upgrade:**
```
// Before (Tier 1)
"Visa en översikt av min efterlevnad"

// After (Tier 2)
"Ge mig en sammanställning av min efterlevnadsstatus. Visa en tabell med lagnamn, status och senaste granskning. Gruppera efter status — visa icke-uppfyllda först. Om det finns luckor, sammanfatta dem kort."
```
