# Story 21.15: Kontroll ↔ Task linkage — manual linking UI + cycle-detail Uppgifter tab

## Status

Backlog (frontend) — deprioritised 2026-04-24 behind Story 21.9 (Seal) and Story 21.12 (background PDF generation). Both deferred stories are strictly higher-leverage than manual cycle↔task linking: 21.9 makes the audit trail legally defensible (cycles are currently mutable forever), 21.12 makes the revisionsrapport portable outside the app. 21.15 is task-UX polish that does not move Laglig's compliance-platform differentiation. Resurface only if post-21.9/21.12 customer feedback specifically asks for cycle-scoped prep/cross-finding task linking. If resurfaced, consider a minimal cut: Uppgifter tab + "Skapa ny uppgift" flow only; drop the symmetric task-modal cycle picker + "Länka befintlig uppgift" to avoid doubling authoring surfaces.

## Story

**As a** KMA-samordnare preparing or running a Lagefterlevnadskontroll,
**I want** to manually link existing tasks to a kontroll (and see all its linked tasks on the cycle detail page),
**so that** preparatory and cross-finding tasks — not only finding-spawned corrective actions — can be associated with the audit, and the cycle surface gives a complete view of everything the team is working on for this kontroll.

## Acceptance Criteria

### Functional

1. **Task modal `Länkade kontroller` card gains "+ Lägg till länk" affordance.** The card landed in Story 21.8 AC 9 and currently only renders when `linkedCycles.length >= 1`. Story 21.15 extends it to ALWAYS render when the user has `tasks:edit` (empty state supported):
   - **Empty state:** when `linkedCycles.length === 0`, card renders with Swedish copy `'Ingen kopplad kontroll'` + `'+ Lägg till länk'` button.
   - **Populated state:** card renders existing rows per 21.8 AC 9 + a smaller `'+ Lägg till länk'` secondary button at the card footer.
   - Read-only mode (per existing `complianceReadOnly` threading): button hidden, empty state reads `'Inga kopplade kontroller'` (plural) without the add affordance.

2. **Cycle picker dialog.** Clicking "+ Lägg till länk" opens a shadcn `Dialog` with a searchable workspace-scoped cycle list. Matches the existing `law-link-selector.tsx` pattern used for linking laws to tasks. Features:
   - Search input with debounced fuzzy filtering on cycle name.
   - Cycle rows show: name, status badge, item count, lead auditor. **Shape note:** `TaskDetails.linkedCycles` (the card's read) carries `{id, name, status, itemCount?}` per `app/actions/task-modal.ts:95-106` — it does NOT include `lead_auditor`, so the picker fetches a richer shape via its own read action `listLinkableCyclesForTask(taskId)` (Task 3.2).
   - Status filter chips: All / Planerad / Pågående / Avslutad (Sealed + Arkiverad hidden by default — linking to a sealed/archived cycle is nonsense, but available via a "Visa alla" toggle). **Why Planerad is visible by default** (corrects an internal contradiction in the v0.1 draft): preparatory-task linking to PLANERAD cycles is the primary manual-link use case per AC 3's server-action allow-list. The picker's default view MUST match that — hiding PLANERAD behind "Visa alla" would force users to hunt for the toggle to enable the intended flow.
   - Multi-select supported (checkbox per row) — user can link to multiple cycles in one action.
   - Already-linked cycles render with a visible "Redan kopplad" tag and unselectable checkbox.
   - Confirm button: `'Länka {N} kontroll(er)'` where N updates live.

3. **Server action `linkTaskToCycles(taskId, cycleIds[])`.** Under `app/actions/compliance-cycle-task-link.ts` (NEW file). Guards:
   - `withWorkspace(cb, 'tasks:edit')` — same permission gate as task mutations.
   - Validate task belongs to workspace.
   - Validate every cycle id belongs to the same workspace AND is NOT soft-deleted AND is NOT in status `SEALED` / `ARKIVERAD` (linking to a sealed cycle is nonsense — the cycle is immutable by then). `PLANERAD` IS allowed (preparatory tasks are the primary use case).
   - `prisma.complianceCycleTaskLink.createMany({ data: [...], skipDuplicates: true })` — the `@@unique([task_id, cycle_id])` constraint on the table (Story 21.8 AC 12b) prevents double-linking; `skipDuplicates` avoids throw on race.
   - `logActivity` one row per new link: `task_linked_to_cycle` with entity_type `task`, entity_id `taskId`, payload `{ cycle_id, cycle_name }`.
   - `revalidatePath('/tasks')` + `revalidatePath('/laglistor/kontroller/' + cycleId)` per cycle.
   - Return the refreshed `TaskDetails.linkedCycles` shape.

4. **Server action `unlinkTaskFromCycle(taskId, cycleId)`.** Same guards as AC 3. Idempotent (deleting an already-deleted link is a no-op). Single-link unlink (not bulk — the UI surface is a per-row "Ta bort" confirm on the card). Activity log: `task_unlinked_from_cycle` with matching payload. Revalidates the same paths.

5. **Unlink affordance on the card.** Each linked-cycle row in the card gains a small `X` button (or dropdown with "Ta bort koppling"). Confirmation dialog: `'Ta bort kopplingen till kontrollen "{cycleName}"? Uppgiften själv påverkas inte.'`. On confirm, calls `unlinkTaskFromCycle`.
   - **Special case: cannot remove the spawn-link.** If `task.complianceFinding?.cycle.id === cycleId` (the cycle is the one whose finding spawned this task), the unlink button is disabled with tooltip: `'Kopplingen skapades när åtgärdsuppgiften genererades från avvikelsen. För att ta bort kopplingen, radera uppgiften eller ta bort avvikelsens koppling till uppgiften i kontrollen.'` Why: unlinking the spawn-cycle while `compliance_finding_id` still points to the finding would desync the two tables. Future story could add proper severance; for now, block at the UI layer.

6. **Cycle detail page — new `Uppgifter` tab.** `/laglistor/kontroller/[cycleId]` gains a new tab between `Findings` and `Rapport`:
   - Label: `Uppgifter`.
   - Count badge: total linked tasks for this cycle.
   - Empty state: `'Inga kopplade uppgifter. Klicka "+ Lägg till uppgift" för att länka befintliga eller skapa nya.'`
   - Populated state: table/list of linked tasks with columns: Titel, Status (column name + color), Ansvarig (avatar + name), Förfallodatum, Prioritet, Ursprung (`Från avvikelse: ...` if `compliance_finding_id` is set; `Manuellt kopplad` otherwise).
   - Click task row → opens the existing task-modal (`task-modal/index.tsx`) overlay; task mutations within the modal reflect back on the cycle page's task list via SWR revalidation.
   - Virtualisation threshold 50 (matches the findings tab pattern from Story 21.7).
   - Filter chips: All / Öppna (not completed) / Klara / Mina (assignee = current user).

7. **"+ Lägg till uppgift" button on the Uppgifter tab.** Dropdown with two options:
   - **"Länka befintlig uppgift"** → opens a task picker dialog (mirror of AC 2's cycle picker, but inverted — lists workspace tasks filtered by search, with already-linked tasks flagged). Multi-select supported. Calls `linkTaskToCycles` reversed — `linkCyclesToTask(cycleId, taskIds[])` (new server action with symmetric guards). OR reuse `linkTaskToCycles` by iterating on the UI side (one round-trip per task) — decide at implementation time based on volume expectations. Default recommendation: add the symmetric action to avoid N round-trips for batch-link.
   - **"Skapa ny uppgift"** → opens the existing `create-task-modal.tsx` with a pre-populated hidden field `linkToCycleId: cycleId`. On save, the task is created AND the link row is inserted in the same tx (`createTask` gets a new optional `linkToCycleId` parameter; if set, adds a `tx.complianceCycleTaskLink.create` inside its existing transaction).

8. **Read action `getCycleLinkedTasks(cycleId, filters)`.** New read in `app/actions/compliance-audit-cycle.ts` (or a new `compliance-cycle-task-link.ts`) returning the cycle's linked tasks with filter support. Same OR-permission gate as other cycle reads (`activity:view || tasks:edit`). Returns `TaskWithRelations[]` (or a narrower shape tailored to the tab — decide at implementation). Supports filters: `status: 'open' | 'done' | 'all'`, `assigneeId: string | null`, `origin: 'spawned' | 'manual' | 'all'` (derived from `compliance_finding_id IS NULL / NOT NULL`).

9. **Activity logger extensions (three-file carry-forward).**
   - `lib/activity/action-constants.ts`: append `'task_linked_to_cycle'`, `'task_unlinked_from_cycle'` under a new `// Compliance-audit cycle ↔ task linkage (Epic 21 — Story 21.15)` section.
   - `lib/activity/categories.ts`: both map to `'kopplingar'` (linkage events).
   - `lib/activity/format-activity.ts`: two new sentence cases:
     - `task_linked_to_cycle`: `[user] kopplade uppgiften [primary] till kontrollen [emphasis: cycle_name]` (primary is the task; emphasised cycle name from `newValue.cycle_name` payload).
     - `task_unlinked_from_cycle`: `[user] tog bort kopplingen mellan uppgiften [primary] och kontrollen [emphasis: cycle_name]`.

10. **Permission gating + read-only propagation.** The tab respects `complianceReadOnly`: when true, "+ Lägg till uppgift" button hidden, task rows still clickable (modal opens in read-only for cycle-scope edits, but task itself remains editable per the existing `tasks:edit` rules — tasks can be worked after a cycle is sealed, per 21.8 AC IV3). The in-modal "+ Lägg till länk" and unlink affordances are hidden when the user lacks `tasks:edit`.

### Why this is a separate story from 21.8

- 21.8 ships the spawn automation + data model + minimal display (read-only card when populated). It's the MVP corrective-action loop.
- 21.15 ships manual authoring + reciprocal cycle-page surface. This is user-agency functionality that depends on — but isn't essential to — the spawn loop.
- Splitting gives 21.8 a smaller, testable surface; 21.15 inherits a landed schema and focuses on UX surfaces.

### Technical references

- `TaskListItemLink` pattern at `prisma/schema.prisma:1125-1145` — symmetric M:N join table (cite verified against live schema on 2026-04-23). Mirror for `linkTaskToCycles` / `unlinkTaskFromCycle` server-action shapes + the "already linked" disabled state in the picker. The sibling `ComplianceCycleTaskLink` model at `prisma/schema.prisma:1147-1160` (landed in 21.8 AC 12b) is the actual table 21.15 mutates — 21.15 does NOT add schema.
- `law-link-selector.tsx` — existing cycle-picker-analog for linking laws to tasks. Reuse the UX vocabulary (search + checkbox rows + count chip).

- Story 21.8 AC 9 card render contract — extend without breaking the read-only fallback.

### Out of scope for 21.15

- **Linking tasks ACROSS workspaces.** Not supported (nor should it be — task workspace isolation is a security invariant).
- **Linking findings to tasks independently of the spawn path.** A finding is either associated with a task via `corrective_action_task_id` (spawn or late-add in 21.8) or not at all. 21.15 only adds cycle-level linking.
- **Preparatory-task templates.** A future story could let KMA-samordnare create per-cycle task templates (e.g., "every internal audit gets these 5 preparatory tasks auto-linked"). Out of scope here.
- **Cross-cycle consolidation.** A task linked to 3 different cycles (e.g., a yearly standing task) is supported by the data model but has no dedicated UX — it just shows 3 rows in the card.
