# Story 14.7d: File Search & Compliance History Tools

## Status

Draft

## Story

**As a** compliance partner agent,
**I want** tools to search the company's uploaded files and review their compliance history for specific laws,
**so that** I can cross-reference legal requirements against the company's internal documents and understand what compliance actions they have already taken.

## Context & Dependencies

This story was split from 14.7b (Company Context Tools) on 2026-03-09 during integration testing. Both tools were deferred because their underlying data pipelines are not yet in place — building the tools now would produce empty results.

**Key existing infrastructure:**

- **Tool infrastructure** (Story 14.7a — Done): Shared types (`ToolResponse`, `ToolError`, `ToolMeta`), utility helpers (`wrapToolResponse`, `wrapToolError`), factory function pattern `createAgentTools(workspaceId)`, chat route wired with `tools` + `stopWhen: stepCountIs(5)`.
- **Retrieval pipeline** (Story 14.8 — Done): `retrieveContext(query, workspaceId, options)` in `lib/agent/retrieval.ts` — provides the backend for `search_company_files`. Supports `sourceType: 'USER_FILE'` filter. Workspace scoping enforced. [Source: lib/agent/retrieval.ts]
- **File upload infrastructure** (existing): `WorkspaceFile` model, Supabase storage bucket, full CRUD in `app/actions/files.ts` (~1700 lines). Files can be uploaded, linked to tasks and law list items, organized in folders. [Source: app/actions/files.ts, prisma/schema.prisma:1017-1087]
- **ContentChunk table** (Story 14.2 — Done): 228,778 chunks with pgvector HNSW index, workspace scoping. `SourceType` enum includes `USER_FILE`. `lib/agent/context-assembly.ts` already handles USER_FILE chunks (line 142). [Source: prisma/schema.prisma]
- **Compliance models** (existing): `LawListItem` has `compliance_status`, `compliance_actions`, `business_context`, `last_change_acknowledged_at`. `TaskListItemLink` joins Task ↔ LawListItem. `ComplianceStatus` enum: `EJ_PABORJAD`, `PAGAENDE`, `UPPFYLLD`, `EJ_UPPFYLLD`, `EJ_TILLAMPLIG`. [Source: prisma/schema.prisma]

**Hard dependency (blocks `search_company_files`):** A file chunking + embedding pipeline that takes uploaded `WorkspaceFile` records → extracts text (PDF, Office, etc.) → chunks → creates `ContentChunk` with `source_type='USER_FILE'` + `workspace_id` → generates embeddings. This pipeline does not exist yet.

**Soft dependency (limits `get_compliance_history` value):** Compliance workflow UI where users can set compliance status, write `business_context` notes, populate `compliance_actions`, and link tasks to law list items. Without this UI, the tool returns empty/null data. The tool is technically buildable now but provides no value until users have a way to populate the data.

**Deferred model:** `ChangeAssessment` (Story 14.10 — not yet built). The `get_compliance_history` tool returns a stub for assessment data.

**Depends on:** 14.7a (tool infrastructure), 14.6 (File Knowledge Extraction & Embedding — blocks `search_company_files`), compliance workflow UI (TBD — limits `get_compliance_history` value)

**Depended on by:** 14.10 (assessment flow uses `get_compliance_history`)

## Acceptance Criteria

### File Search Tool

1. `search_company_files` — semantic search across workspace file chunks. Parameters: `query` (string, required), `limit` (optional, default 5). Calls `retrieveContext()` with `sourceType: 'USER_FILE'`. Returns: matching file passages with filename, relevanceScore, snippet. Workspace scoping enforced — never returns another workspace's file chunks.
2. Tool description includes privacy note: these are the user's private files, always scoped to current workspace
3. Graceful empty result: if no file chunks exist, return helpful guidance in Swedish suggesting the user upload documents first

### Compliance History Tool

4. `get_compliance_history` — past actions for a specific law in this workspace. Parameters: `documentId` (string, required). Returns: `LawListItem` compliance status, `compliance_actions`, `business_context`, `last_change_acknowledged_at`, related tasks via `TaskListItemLink` join
5. Return deferred `ChangeAssessment` field as: `{ assessments: [], _note: "ChangeAssessment-modellen skapas i Story 14.10" }`
6. Handle case where law is not in any of the workspace's law lists — return actionable Swedish guidance

### Quality Standards (inherited from 14.7a)

7. Tool descriptions written as comprehensive docstrings — purpose, when to use, what it returns, edge cases
8. Response shapes follow `ToolResponse<T>` with `_meta: { tool, executionTimeMs, resultCount }` pattern
9. Error responses include specific, actionable guidance in Swedish
10. Tools registered in the `createAgentTools` factory function in `lib/agent/tools/index.ts`
11. At least 5 unit tests across both tools

## Dev Notes

### Blocking Prerequisites

**For `search_company_files`:** The file chunking pipeline must be built first. Current state:
- File upload works: `WorkspaceFile` + Supabase storage in `app/actions/files.ts`
- `USER_FILE` is in the `SourceType` enum
- `context-assembly.ts` handles USER_FILE chunks at read time (line 142)
- **Missing:** Pipeline to chunk uploaded files into `ContentChunk` records and embed them
- Without this pipeline, `retrieveContext(..., { sourceType: 'USER_FILE' })` returns zero results

**For `get_compliance_history`:** All Prisma models and fields exist, but data is empty:
- `compliance_actions` and `business_context` on `LawListItem` are write-through-UI fields — no UI exists to populate them
- `TaskListItemLink` requires tasks to be linked to law list items — linking UI is minimal
- `ChangeAssessment` model doesn't exist yet (Story 14.10)
- Tool is buildable but returns empty/null for every field

### Key Prisma Models (Verified)

**LawListItem** — `document_id`, `compliance_status` (ComplianceStatus enum), `business_context`, `compliance_actions`, `notes`, `priority`, `last_change_acknowledged_at` [Source: prisma/schema.prisma]

**TaskListItemLink** — `task_id`, `law_list_item_id`, unique constraint on pair. Joins Task ↔ LawListItem. [Source: prisma/schema.prisma:963-977]

**ContentChunk** — `source_type` (SourceType enum incl. USER_FILE), `source_id`, `workspace_id`, `path`, `contextual_header`, `content`, `embedding` (vector), `token_count`, `metadata` [Source: prisma/schema.prisma]

**WorkspaceFile** — `workspace_id`, `name`, `mime_type`, `size_bytes`, `storage_path`, folder hierarchy, linked to Task via `FileTaskLink`, linked to LawListItem via `FileListItemLink` [Source: prisma/schema.prisma:1017-1053]
