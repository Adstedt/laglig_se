# Story 14.7b: Company Context Tools

## Status

Draft

## Story

**As a** compliance partner agent,
**I want** tools to access the company's profile, search their uploaded files, and review their compliance history,
**so that** I can give company-specific guidance grounded in the user's actual context.

## Context & Dependencies

This is the second of three stories split from the original 14.7 (Agent Tool Definitions). It adds three read-only tools that surface workspace-specific data: the company profile, uploaded file content, and compliance action history. These tools transform the agent from a generic legal Q&A bot into a company-aware compliance partner.

**Key existing infrastructure:**

- **Tool infrastructure** (Story 14.7a): Shared types (`ToolResponse`, `ToolError`, `ToolMeta`), utility helpers (`wrapToolResponse`, `wrapToolError`), factory function pattern `createAgentTools(workspaceId)`, chat route already wired with `tools` + `maxSteps: 5`. This story adds tools to the existing registry.
- **Retrieval pipeline** (Story 14.8): `retrieveContext(query, workspaceId, options)` in `lib/agent/retrieval.ts` — provides the backend for `search_company_files`. Returns chunks with relevance scores after two-stage vector+rerank pipeline. [Source: docs/stories/14.8.rag-retrieval-pipeline.md]
- **Existing server actions**: Company profile (`getCompanyProfile()` in `app/actions/company-profile.ts`), change events (`getUnacknowledgedChangeCount()` in `app/actions/change-events.ts`) — `get_company_context` wraps these existing functions. [Source: app/actions/]
- **ContentChunk table** (Story 14.2 — Done): 228,778 chunks with pgvector HNSW index, workspace scoping. User file chunks stored with `sourceType: 'USER_FILE'` and `workspaceId` set. [Source: prisma/schema.prisma]

**Depends on:** 14.7a (tool infrastructure, types, factory pattern, chat route wiring), 14.8 (retrieval pipeline for `search_company_files`)

**Soft dependency:** 14.4 (WorkspaceProfile — not yet built). The `get_company_context` tool uses existing `Workspace` model fields (`org_number`, `company_legal_name`, `sni_code`) and `getCompanyProfile()` action. When 14.4 adds the full `WorkspaceProfile`, the tool response is enriched — no code changes needed, just richer data.

**Deferred models:** `ChangeAssessment` (Story 14.10 — not yet built). The `get_compliance_history` tool returns a stub note for assessment data. It will be enriched when 14.10 creates the model.

**Depended on by:** 14.9 (system prompt references company context tools), 14.10 (assessment flow uses `get_compliance_history`)

## Acceptance Criteria

### Company Context Tools

1. `get_company_context` — company profile + compliance posture. Parameters: none (workspace resolved from session context via factory closure). Returns: workspace fields (`companyLegalName`, `orgNumber`, `sniCode`), tracked law list names with item counts, compliance status distribution, recent unacknowledged change count
2. `search_company_files` — semantic search across workspace file chunks. Parameters: `query` (string, required), `limit` (optional, default 5). Calls `retrieveContext()` from Story 14.8 with `sourceType: 'USER_FILE'`. Returns: matching file passages with filename, similarity score. Workspace scoping is enforced — never returns another workspace's file chunks.
3. `get_compliance_history` — past actions for a specific law in this workspace. Parameters: `documentId` (string, required). Returns: related tasks (from `TaskListItemLink`), compliance status on `LawListItem`, `complianceActions` field, `lastChangeAcknowledgedAt`. (Note: `ChangeAssessment` data deferred to Story 14.10 — return empty array with explanatory note)

### Quality Standards (inherited from 14.7a)

4. Tool descriptions written as comprehensive docstrings following the same rigor as 14.7a — purpose, when to use, what it returns, edge cases
5. Response shapes follow `ToolResponse<T>` with `_meta: { tool, executionTimeMs, resultCount }` pattern
6. Error responses include specific, actionable guidance in Swedish
7. Tools registered in the `createAgentTools` factory function — added to the existing tool registry in `lib/agent/tools/index.ts`
8. At least 5 unit tests covering: response shapes, mock server action integration, deferred assessment stub, workspace scoping enforcement, error paths

### Soft Dependency on WorkspaceProfile (14.4)

The `get_company_context` tool works today using existing `Workspace` model fields. When Story 14.4 adds the full `WorkspaceProfile` model (industry, employee count, certifications, activity flags, compliance maturity), the tool's response gets richer automatically — just query the new relation. No breaking changes, just more data.

Design the tool response to include a `profileComplete` boolean so the agent can detect incomplete profiles and nudge the user: "Jag ser att er profil saknar branschinformation — vill ni berätta mer om er verksamhet?"
