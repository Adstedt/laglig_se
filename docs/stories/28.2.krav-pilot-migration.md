# Story 28.2: Pilot — krav-table on the DataTable Core

**Epic:** 28 — Unified DataTable Core (`docs/prd/epic-28-unified-datatable-core.md`)
**Status:** Done (2026-07-08)

## What changed

`components/features/krav/krav-table.tsx` (488 → 354 lines) is now column
definitions + adapter mapping on `components/ui/data-table`. All table
mechanics deleted from the file: raw `<table>` markup, inline `useVirtualizer`,
sticky-header wiring, the INTERACTIVE_COLUMNS stopPropagation set (superseded
by the core's interactive-element guard), and the bespoke "Ladda fler" button.

**Public interface unchanged** — `krav-page-content.tsx` needed zero changes.
Its URL sort state maps onto a `SortingAdapter` (`manual: true`), its cursor
pagination onto the `button` LoadMoreStrategy, and skeleton/error/empty
states stay parent-owned as before.

Card slots authored on the column meta — `/krav` is the app's **first
narrow-container card surface**: kravpunkt text as title, FulfilledToggle
live on the card face (`interactive: true` — it's the primary action),
Regelverk/Laglista/Ansvarig/Bevis as labeled meta rows (AssigneeEditor
popover stays interactive), relative timestamp as footer.

## Core changes made by this story (pre-freeze API additions)

- `VirtualizationConfig.maxHeight` widened to accept CSS strings (`'70vh'`).
- `CardSortDropdown` bug fix: `column.getCanSort()` requires an `accessorFn`,
  which display columns (id-only — the norm for manualSorting servers) never
  have. With `manual: true` the explicit `enableSorting: true` flag is now the
  source of truth for the dropdown's column list.

## Verified in browser (real workspace data, /krav)

- URL sort: header click → `?sort=law_name&dir=asc`; second click → desc
  (parent omits `dir` at its default — pre-existing convention); **survives
  full reload**; sort indicator renders on the active header.
- Row click → `LegalDocumentModal` opens with the right law; Esc closes.
- Interactive guard: AssigneeEditor popover opens without triggering row
  navigation.
- Narrow container (480px viewport): table swaps to 22 cards; card sort
  dropdown lists exactly the 4 sortable columns by `meta.dt.label`; selecting
  one drives the same URL adapter (`?dir=asc`). Restore to 1440px → table
  returns, state intact. No console errors, no render loops.
- 29 unit tests green; tsc + eslint clean.

## Polish round (2026-07-08, live user feedback)

Core additions (pre-freeze, each driven by a real krav need):
- **Fill column** (`meta.dt.fill`): leftover container width goes to the
  primary column instead of the trailing spacer — kravpunkt text breathes at
  every tier, no horizontal scroll in shed states.
- **Column shedding live** (`meta.dt.hideBelow`, was reserved): table-view
  only; krav sheds Uppdaterad → Bevis → Laglista → Ansvarig as the container
  narrows. Three-tier strategy: full table → shed table → cards.
- **First-column edge inset** (pl-6) as a core convention (legacy krav/laglistor
  pattern, regressed in the initial migration).
- **Header overflow clipping** (overflow-hidden + nowrap on th) — misdeclared
  widths clip instead of bleeding into the neighbor column.
- **8px-quantized ResizeObserver** — the 300ms sidebar animation was
  re-rendering the table per pixel (user-reported choppiness).
- **Card renderCard null-skip**: a slot renderer returning null removes the
  whole card row (no label + em-dash noise for non-applicable fields).
- Card layout: items-center meta rows, w-20 label gutter, snug title leading.

krav-side:
- Resize enabled (localStorage-persisted, `laglig:krav:columns:v1`),
  min/max bounds on all resizable columns; Uppfylld fixed at 112px (header
  label + arrow no longer overlap under table-fixed).
- `cardBelow: 800` — chat-maximized shows cards (user decision).
- Card face: labeled status toggle ("Uppfylld"/"Ej uppfylld"), Ansvarig
  avatar + name ("Ej tilldelad" when unassigned), Bevis row hidden when not
  applicable.
- Toolbar single-row under squeeze: chip count badges hide below 56rem
  container (`data-chip-count` hook added to the FilterChip primitive,
  container-queries plugin now in real use), search flexes 120–288px.
