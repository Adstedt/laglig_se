# Story 22.3: `PageHeader` + `TableToolbar` primitives + six-surface migration

## Status

Draft

## Story

**As a** developer working on workspace UI,
**I want** a `PageHeader` primitive (with named slots for breadcrumbs / title / badge / meta / stats / primaryAction) and a `TableToolbar` primitive (with named slots for views / search / filters / rightSlot), and every workspace tabular surface migrated to use them,
**so that** the same visual hierarchy (title → meta → divider → toolbar) appears on every surface, the primary-action button always lands in the same place, and new surfaces compose from these primitives rather than re-deciding header layout in every page file.

## Acceptance Criteria

### Functional

1. **`PageHeader` primitive created.** New file `components/ui/page-header.tsx` exports `<PageHeader>` and a `<PageHeader.Meta>` subcomponent. Props on `<PageHeader>`:
   - `breadcrumbs?: ReactNode` (typically a shadcn `<Breadcrumb>` chain; rendered above the title)
   - `title: string` (renders as `<h1 class="text-2xl font-semibold tracking-tight">`)
   - `badge?: ReactNode` (renders inline next to title — typically a `<Badge tone>` from Story 22.1)
   - `subtitle?: string` (renders as `<p class="text-sm text-muted-foreground">`)
   - `meta?: ReactNode` (typically `<PageHeader.Meta items>`)
   - `stats?: Array<{ label: string; value: string | ReactNode }>` (right-aligned stat columns next to primaryAction; separated from primaryAction by a `<div class="h-8 w-px bg-border">` divider)
   - `primaryAction?: ReactNode` (top-right primary CTA — typically a `<Button>`)
   - `secondaryActions?: ReactNode` (typically a `<DropdownMenu>` with secondary CTAs; rendered to the LEFT of primaryAction)
   - `className?: string`
   - **Canonical layout:** breadcrumbs → flex-row[ title+badge | (subtitle below) | (stats + secondaryActions + primaryAction right) ] → meta below title row → bottom border `<Separator>`.
   - Class strings sourced from `_prototypes/ui-alignment-prototype.html` §4 "Reference layout".

2. **`PageHeader.Meta` subcomponent.** Renders `<div class="mt-1.5 flex flex-wrap items-center gap-x-3 gap-y-1 text-xs text-muted-foreground">` containing dot-separated items. Each item is either a string OR `{ icon?: ReactNode; label: string }` (icon renders left of label). Separators use `<span class="text-border">·</span>`. Replaces the ad-hoc `·`-joined plain text on cycle-detail today.

3. **`TableToolbar` primitive created.** New file `components/ui/table-toolbar.tsx` exports `<TableToolbar>`. Named slot props:
   - `views?: ReactNode` (typically shadcn `<Tabs>` for view-switching — left-aligned)
   - `search?: ReactNode` (typically a `<SearchInput>` — right-side group)
   - `filters?: ReactNode` (typically a `<FilterChipGroup>` from Story 22.2 OR shadcn `<DropdownMenu>` filter dropdowns — right-side group)
   - `rightSlot?: ReactNode` (right-most slot, typically a primary-CTA-for-this-tab — e.g., "Lägg till finding", "Lägg till dokument")
   - `className?: string`
   - **Canonical layout:** `<div class="flex flex-wrap items-center justify-between gap-3">` with `views` on the left and `search + filters + rightSlot` flowing on the right. Responsive wrap on narrow viewports.
   - When `views` is omitted, search+filters+rightSlot left-align (no empty left slot).

4. **Migrate `/laglistor`** (`app/(workspace)/laglistor/page.tsx` and consumed components). Wrap header region in `<PageHeader title="Mina listor" subtitle="Hantera dina listor och håll koll på relevanta rättsfall." primaryAction={<Button>+ Lägg till dokument</Button>}>`. Move `Skapa kontroll` and `Generera om laglista` into a `secondaryActions` overflow `<DropdownMenu>` (per the prototype's "Mina laglistor — after" mock). Wrap the toolbar region in `<TableToolbar views={<Tabs>Mina listor / Ändringar</Tabs>} search={<SearchInput placeholder="Sök dokument..."/>} filters={<>...existing FilterBar children...</>} />`. **Visual delta acknowledged:** "Lägg till dokument" moves from the filter strip to the page-header primary slot; flag in PR description.

5. **Migrate `/laglistor/kontroller`** (`app/(workspace)/laglistor/kontroller/page.tsx` + `cycle-list-table.tsx`). Wrap header in `<PageHeader title="Uppgifter" subtitle="Hantera och spåra efterlevnadsuppgifter för din organisation." primaryAction={<Button>+ Ny uppgift</Button>}>`. Wrap toolbar in `<TableToolbar views={<Tabs>Sammanfattning/Aktiva/Lista/Kalender/Alla uppgifter</Tabs>} search={<SearchInput placeholder="Sök uppgifter..."/>} filters={<FilterPopover>Filter (N)</FilterPopover>} rightSlot={<KolumnerDropdown/>} />`. **Decision flag:** consolidate the four right-aligned filter dropdowns (`Status`, `Prioritet`, `Förfallodatum`, `Ansvarig`) into a single `Filter (N)` popover, OR keep them as four discrete dropdowns inside `filters` slot. **Recommended:** single `Filter (N)` popover for visual parity with Laglistor; flag in PR description. If too disruptive to TanStack table column-filter wiring, ship discrete dropdowns and add a follow-up.

9. **Migrate `/workspace/styrdokument`** (`app/(workspace)/workspace/styrdokument/page.tsx` + `document-browser-page.tsx`). Wrap header in `<PageHeader title="Styrdokument" subtitle="Policyer, rutiner och instruktioner med versionshistorik" secondaryActions={<Button variant="outline">Importera</Button>} primaryAction={<Button>+ Nytt dokument</Button>}>`. Wrap toolbar in `<TableToolbar views={<Tabs>Aktiva/Arkiverade</Tabs>} search={<SearchInput placeholder="Sök dokument..."/>} filters={<>Dokumenttyp dropdown + Status dropdown</>} />`. **Visual delta acknowledged:** active filter dropdowns no longer get their own bg-fill highlight (consistency with other surfaces); flag in PR description.

10. **Action verb standardisation.** Across all migrated surfaces:
    - **`Ny X` / `Nytt X` (descriptive verb)** for top-level primary creates: `Ny uppgift` (Tasks), `Nytt dokument` (Styrdokument), `Skapa kontroll` (Kontroller list — stays "Skapa" because that's the established label; not changing for this story to avoid copy churn — flag for future consistency story).
    - **`Lägg till X` (locative verb)** for additive actions inside an existing context: `Lägg till dokument` (inside a laglista), `Lägg till finding` (inside a cycle).
    - Document the convention in a short comment at the top of `components/ui/page-header.tsx`.

11. **Slot order is enforced by the primitive, not the call site.** `<PageHeader>` always renders breadcrumbs → title row → meta in that order regardless of prop order. `<TableToolbar>` always renders views left and search+filters+rightSlot right. Devs cannot accidentally re-order by re-arranging props.
