# Story 28.10: Laglistor Cards — One Card Implementation

**Epic:** 28 — Unified DataTable Core (`docs/prd/epic-28-unified-datatable-core.md`)
**Status:** Done (2026-07-08)

## What changed

- viewMode `'card'` now renders the SAME core-backed tables forced into the
  card renderer (`forceCardView` → `view: { force: 'card' }`), threaded
  through the flat tables and the grouped wrapper/sections. Grouped card
  mode keeps its collapsible sections with card lists inside.
- Product decisions from the epic recorded and implemented: **cards show
  `complianceStatus`** (same column defs as the table — the hand-made
  card's legacy-`status` divergence is retired), and the container-width
  rule only affects table-configured modes (force wins).
- Deleted: `document-list-card.tsx`, `document-list-grid.tsx`,
  `grouped-document-list.tsx` (+ barrel exports) — **the product now has
  exactly one card implementation** (the core's).
- Card mode also gains what the legacy cards never had: inline editors
  (status/priority interactive badges, editor meta rows), selection, and
  load-more — because it's the same DataTable.
