# Story 17.10: Agent Tools — Search, Read & List Workspace Documents

## Status

Draft

## Story

**As an** AI compliance agent,
**I want** to search, read, and list workspace documents,
**so that** I can provide advice informed by the organization's actual policies, procedures, and risk assessments.

## Story Context

**Existing System Integration:**

- Integrates with: Agent tools registry (`lib/agent/tools/index.ts`), RAG pipeline with `WORKSPACE_DOCUMENT` support (Story 17.9), `Document` and `WorkspaceFile` models, system prompt (`lib/agent/system-prompt.ts`)
- Technology: Vercel AI SDK tool definitions, pgvector retrieval, Prisma
- Follows pattern: Existing tool patterns — `search-laws.ts`, `get-document-details.ts`, `get-company-context.ts` — same input/output structure, same `createAgentTools()` factory
- Touch points: `lib/agent/tools/index.ts` (register new tools), `lib/agent/system-prompt.md` (update prompt), `TOOL_CONFIG` in `components/features/ai-chat/chat-message.tsx` (Swedish labels)

## Acceptance Criteria

**search_workspace_documents Tool:**

1. New tool: `search_workspace_documents`
2. Input: `query` (string, required), optional `document_type` filter (DocumentType enum), optional `status` filter (DocumentStatus enum)
3. Uses `retrieveContext()` with `sourceType: 'WORKSPACE_DOCUMENT'` and `workspaceId` filter (note: `sourceType` is singular, matching the `retrieveContext()` signature used in `search-laws.ts`)
4. Returns: matched snippets with document title, document type, status, version number, relevance score, citation key
5. Maximum 10 results
6. Citation key format: `wdoc:{document_id}` (intentionally distinct from legal doc citations which use `doc:{id}` — note: Epic 17.10 AC5 specifies `doc:` but this story uses `wdoc:` to avoid collisions with `LegalDocument` citation keys)

**get_workspace_document Tool:**

7. New tool: `get_workspace_document`
8. Input: `document_id` (string, required)
9. Returns: title, document_type, status, version_number, document_number, extracted_text (full content), review_date, approved_by, approved_at, linked tasks (id + title), linked list items (id + document title + document number)
10. For large documents, content truncated to first 20,000 characters with a note: "[Truncated — document has {N} characters total]"
11. Validates document belongs to current workspace

**list_workspace_documents Tool:**

12. New tool: `list_workspace_documents`
13. Input: optional `document_type`, optional `status`, optional `linked_list_item_id`, optional `linked_task_id`
14. Returns: list of documents with title, document_type, status, version_count, last_updated, created_by name
15. Maximum 25 results, ordered by last updated
16. Workspace-scoped (enforce `workspace_id`)

**Registration & Prompt:**

17. All three tools registered in `createAgentTools(workspaceId, userId)` alongside existing 9 tools
18. System prompt updated to inform agent about workspace document capabilities — when to use these tools, what kind of information they provide
19. `TOOL_CONFIG` in chat UI updated with Swedish labels and icons for the three new tools

**Integration Requirements:**

20. Existing 9 agent tools unchanged
21. Tool results compatible with existing citation extraction in `app/api/chat/route.ts`

**Quality Requirements:**

22. Semantic search returns relevant results within 2 seconds
23. Document read returns within 500ms

Key patterns from `search-laws.ts` to follow:
- Import `tool` and `zodSchema` from `'ai'`
- Import `z` from `'zod/v4'` (not `'zod'`)
- Use `inputSchema: zodSchema(schemaObj)` — NOT `parameters: z.object(...)`
- Use `wrapToolResponse` and `wrapToolError` from `'./utils'`
- `retrieveContext(query, workspaceId, { sourceType: 'WORKSPACE_DOCUMENT' })` — `sourceType` is singular
