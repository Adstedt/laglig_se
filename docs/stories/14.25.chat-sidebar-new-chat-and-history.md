## Story

**As a** workspace user who uses the AI chat from the right sidebar on law, föreskrift, EU, and modal pages,
**I want** to start a new chat and browse my prior conversations directly from the chat sidebar header,
**so that** I can save the conversation I just had, pick up where I left off earlier, and start a fresh thread without having to navigate to the Hem page.

## Acceptance Criteria

**Functional Requirements**

1. The chat sidebar header (`components/features/ai-chat/chat-sidebar.tsx`) renders two new icon buttons to the left of the existing close (`X`) button: a **History** button (`History` icon from `lucide-react`) and a **New chat** button (`SquarePen` icon from `lucide-react`), in that visual order.
2. Each new button has a `Tooltip` (using the same `Tooltip` / `TooltipContent` / `TooltipTrigger` primitives used in `components/features/dashboard/hem-chat.tsx:340–391`) with Swedish labels **"Tidigare konversationer"** (History) and **"Ny konversation"** (New chat). The buttons expose matching `aria-label` values.
3. Clicking **New chat** archives the current active chat and clears the sidebar's visible messages, matching the behavior of `handleNewConversation` in `hem-chat.tsx:147–157`: call `archiveConversation()` then `clearHistory()` from `useChatInterface`, and show the Swedish error toast `'Kunde inte spara konversationen. Försök igen.'` on failure. If `messages.length === 0`, the handler skips the archive/clear calls and is effectively a no-op.
4. Clicking **History** opens the existing `ConversationHistory` slide-over panel by calling `setChatHistoryOpen(true)` from `useLayoutStore` (the same panel already mounted in `components/layout/workspace-shell.tsx:112–132`). **No new panel is created.**
5. Selecting a conversation from that panel loads it into the sidebar: the sidebar listens for the existing `laglig:load-conversation` CustomEvent dispatched by `workspace-shell.tsx:122–126` and, on receipt, archives any currently active messages (matching `hem-chat.tsx:162–164`), calls `loadConversation(conversationId)`, converts the DB messages to `UIMessage` shape, and calls `replaceMessages(uiMessages)`. The conversion logic must match `hem-chat.tsx:167–178` exactly (mapping `role: 'USER' → 'user' | 'assistant'`, preserving `id`, `createdAt`, `metadata` when present, and wrapping `content` in `parts: [{ type: 'text', text: content }]`).
6. When no messages are present (empty state at `chat-sidebar.tsx:91–92`), the new buttons remain visible and clickable. Clicking **New chat** in the empty state is a no-op per AC 3. Clicking **History** still opens the panel per AC 4.
7. The buttons are rendered only when `isOpen` is `true` (i.e., they live inside the existing `<aside>`).

**Integration Requirements**

8. The existing close button (`X`), `useChatInterface({ contextType: 'global' })` wiring, `EmptyState`, `ChatMessageList`, `ChatInput`, error display, and analytics tracking (`track('ai_chat_opened', …)`) in `chat-sidebar.tsx` continue to work unchanged.
9. The implementation reuses the **Story 14.11** primitives from `app/actions/ai-chat.ts` (`archiveConversation`, `loadConversation`) and the `clearHistory` / `replaceMessages` returned by `useChatInterface` — no new server actions, no changes to `app/api/chat/route.ts`, no Prisma schema changes.
10. The `ConversationHistory` panel at `workspace-shell.tsx:119–131` is not modified. The sidebar opts in to the same pub/sub pattern already used by `HemChat`.
11. Because `contextType` remains `'global'`, the history panel shows the same conversation list the user sees on `/hem` — conversations are not duplicated or segregated by surface.

**Quality Requirements**

12. Add unit tests for `ChatSidebar` covering: (a) rendering of the two new buttons with correct `aria-label`s, (b) clicking New chat calls `archiveConversation` then `clearHistory` when messages exist, (c) clicking New chat is a no-op when `messages.length === 0`, (d) clicking History calls `setChatHistoryOpen(true)`, (e) a dispatched `laglig:load-conversation` event triggers `loadConversation` and `replaceMessages` with the converted payload. Follow the patterns in `tests/unit/components/features/dashboard/hem-chat.test.tsx`.
13. No regression in the existing chat-sidebar test file (if present). Run `pnpm test` and `npx tsc --noEmit` — both must pass.
14. No visual regressions to the rest of the chat sidebar (header height, spacing, messages area, input). The two new buttons fit within the existing `py-3` header and use `h-8 w-8 p-0 variant="ghost"` to match the existing close button.

### Canonical snippets to copy

**New-chat handler (from `hem-chat.tsx:147–157`):**

```ts
const handleNewConversation = useCallback(async () => {
  try {
    if (hasMessages) {
      await archiveConversation()
      await clearHistory()
    }
  } catch {
    toast.error('Kunde inte spara konversationen. Försök igen.')
  }
}, [hasMessages, clearHistory])
```

The sidebar does **not** need Hem's `setUserEngaged(false)` call — that flag is specific to Hem's home-vs-conversation layout switch, which the sidebar does not have.

**Load-conversation handler (from `hem-chat.tsx:159–183`):**

```ts
const handleLoadConversation = useCallback(
  async (conversationId: string) => {
    if (hasMessages) {
      await archiveConversation()
    }
    const result = await loadConversation(conversationId)
    if (result.success && result.data) {
      const uiMessages = result.data.map((msg) => ({
        id: msg.id,
        role: (msg.role === 'USER' ? 'user' : 'assistant') as 'user' | 'assistant',
        parts: [{ type: 'text' as const, text: msg.content }],
        createdAt: msg.createdAt,
        ...(msg.metadata ? { metadata: msg.metadata } : {}),
      }))
      replaceMessages(uiMessages)
    }
  },
  [hasMessages, replaceMessages]
)
```

**Event subscription (from `hem-chat.tsx:260–268`):**

```ts
useEffect(() => {
  const handler = (e: Event) => {
    const { conversationId } = (e as CustomEvent<{ conversationId: string }>).detail
    handleLoadConversation(conversationId)
  }
  window.addEventListener('laglig:load-conversation', handler)
  return () => window.removeEventListener('laglig:load-conversation', handler)
}, [handleLoadConversation])
```

### Scope Boundaries (important)

- **In scope:** right-sidebar chat only — the component rendered by `components/features/ai-chat/chat-sidebar.tsx` with `contextType: 'global'`.
- **Out of scope:** the modal-embedded chat panel (`components/features/ai-chat/chat-panel.tsx`) used on task / law / change modals.
