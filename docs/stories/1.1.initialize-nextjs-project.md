# Story 1.1: Initialize Next.js Project

## Status
Draft

## Story

**As a** developer,
**I want** to set up a Next.js 16 project with TypeScript, Tailwind CSS, and essential tooling,
**so that** I have a modern development environment ready for rapid feature development.

## Acceptance Criteria

1. Next.js 16+ installed with App Router (not Pages Router)
2. TypeScript configured with strict mode enabled
3. Tailwind CSS installed and configured with custom color tokens (primary, success, warning, error)
4. ESLint + Prettier configured for code quality and formatting
5. Git repository initialized with `.gitignore` (exclude `node_modules`, `.env.local`, `.next`)
6. Project runs locally with `npm run dev` showing default Next.js homepage
7. No build errors or TypeScript warnings

## Tasks / Subtasks

- [ ] Initialize Next.js 16 project (AC: 1)
  - [ ] Run `npx create-next-app@latest laglig-se --typescript --tailwind --app --eslint`
  - [ ] Verify App Router is selected (not Pages Router)
  - [ ] Test that dev server starts: `npm run dev`

- [ ] Configure TypeScript strict mode (AC: 2)
  - [ ] Update `tsconfig.json` with `"strict": true`
  - [ ] Update `tsconfig.json` with path aliases: `@/*` → `./src/*`
  - [ ] Verify no TypeScript errors in initial setup

- [ ] Configure Tailwind CSS with custom tokens (AC: 3)
  - [ ] Update `tailwind.config.ts` with custom color palette:
    - primary: OpenAI-inspired blue (#0066FF)
    - success: Green (#10B981)
    - warning: Yellow (#F59E0B)
    - error: Red (#EF4444)
  - [ ] Verify Tailwind classes work in test component

- [ ] Configure ESLint + Prettier (AC: 4)
  - [ ] Install Prettier: `npm install --save-dev prettier`
  - [ ] Create `.prettierrc` with project standards
  - [ ] Install eslint-config-prettier to prevent conflicts
  - [ ] Add format script to `package.json`: `"format": "prettier --write ."`
  - [ ] Test: `npm run lint` and `npm run format`

- [ ] Initialize Git repository (AC: 5)
  - [ ] Run `git init`
  - [ ] Verify `.gitignore` includes: `node_modules/`, `.env.local`, `.next/`, `.vercel/`
  - [ ] Create initial commit: `git add . && git commit -m "Initial Next.js 16 setup"`

- [ ] Verify project runs without errors (AC: 6, 7)
  - [ ] Start dev server: `npm run dev`
  - [ ] Open browser to `http://localhost:3000`
  - [ ] Verify Next.js default page loads
  - [ ] Run `npm run build` to verify production build works
  - [ ] Confirm zero TypeScript errors and zero ESLint warnings

## Dev Notes

### Technology Stack
- **Framework:** Next.js 16 (React 19)
- **Language:** TypeScript 5.x with strict mode
- **Styling:** Tailwind CSS 3.x
- **Package Manager:** npm (Node.js 20.9+ required)
- **Build Tool:** Turbopack (default in Next.js 16)

### Project Structure
```
laglig-se/
├── app/                    # Next.js App Router pages
│   ├── layout.tsx          # Root layout
│   ├── page.tsx            # Homepage
│   └── globals.css         # Global styles
├── components/             # React components
├── lib/                    # Utility functions
├── prisma/                 # Database schema (added in Story 1.2)
├── public/                 # Static assets
├── .env.local              # Environment variables (git-ignored)
├── tailwind.config.ts      # Tailwind configuration
├── tsconfig.json           # TypeScript configuration
└── package.json            # Dependencies
```

### Key Configuration Files

**tsconfig.json (strict mode):**
```json
{
  "compilerOptions": {
    "strict": true,
    "target": "ES2022",
    "lib": ["ES2022", "DOM"],
    "jsx": "preserve",
    "moduleResolution": "bundler",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

**tailwind.config.ts (custom colors):**
```typescript
import type { Config } from 'tailwindcss'

const config: Config = {
  content: ['./app/**/*.{js,ts,jsx,tsx,mdx}', './components/**/*.{js,ts,jsx,tsx,mdx}'],
  theme: {
    extend: {
      colors: {
        primary: '#0066FF',
        success: '#10B981',
        warning: '#F59E0B',
        error: '#EF4444',
      },
    },
  },
  plugins: [],
}
export default config
```

### Important Notes from Architecture
- **Next.js 16 Breaking Change:** All route params and searchParams are now async and must be awaited
- **Turbopack:** Default in Next.js 16, provides 2-5× faster builds than Webpack
- **React 19:** Includes new hooks like `useActionState`, use sparingly in MVP
- Path aliases (`@/*`) are critical for clean imports - configure from start

### Testing

**Test Location:** This story primarily validates setup, not code. Verification is manual.

**Testing Standards from Architecture Section 16:**
- **Framework:** Vitest + React Testing Library
- **Test Location:** `tests/unit/` and `tests/integration/`
- **Coverage Target:** 60-70% overall
- **Run Tests:** `npm run test`
- **Watch Mode:** `npm run test:watch`

**Setup Verification:**
- Dev server starts without errors: `npm run dev`
- Build completes without errors: `npm run build`
- Linting passes: `npm run lint`
- Formatting works: `npm run format`
