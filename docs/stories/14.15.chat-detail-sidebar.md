# Story 14.15: Chat Detail Sidebar — Inline Context Panel

<!-- Source: Brownfield enhancement to existing chat interface -->
<!-- Context: Extends inline citations and future inline items with a contextual sidebar -->

## Status

Draft

## Story

**As a** compliance manager chatting on the Hem page,
**I want** to click inline items (citations, tasks, law lists, documents) in AI responses to open a detail sidebar alongside the conversation,
**so that** I can inspect full context without losing my place in the chat.

## Vision: Chat as Compliance Workspace

The guiding principle is the same one that makes Cursor and Claude Code effective: **the conversation is the primary workspace, not a Q&A log.** Everything the user needs to understand and act on what the AI said should be reachable without leaving.

In Cursor, the AI mentions a file and the diff is right there — you don't switch tabs to inspect it. In Claude Code, tool results expand inline — you see what happened without navigating away. The conversation and the artifacts it references live side by side, so you can reason about both simultaneously.

Compliance tooling today is the opposite. The AI cites a law — the user opens a new tab. It references a task — they switch to the task page. It mentions a change — they go find the assessment. The user is the integration layer, manually stitching context across tabs and tools.

This story makes the chat the workspace:
- **Citation in chat** = like Cursor's file mention → click to see the actual law text
- **Task chip in chat** = like inline terminal output → click to see details and interact
- **Law list reference** = like a diff view → see exactly what's relevant in context
- **Change assessment** = like a multi-file edit summary → AI analysis + source, side by side

The inline chip is the collapsed state. The sidebar is the expanded state. Same data, two levels of detail, zero context switches. Every new inline type added makes the workspace more complete — a compounding flywheel, not a one-off feature.

## Context

The AI chat on `/dashboard` already renders inline citation pills with hover cards (Story 14.11). This story introduces a detail sidebar that opens on the right when inline items are clicked, shifting the chat left. This establishes the foundation for all future inline interactive elements.

### Two-Tier Interaction Model

| Tier | Trigger | Inline items | Behavior |
|------|---------|-------------|----------|
| **Preview + sidebar** | Hover = quick peek, Click = full sidebar | Citations, law text references | Hover card stays as-is; click promotes to sidebar with full section text and surrounding context |
| **Direct sidebar** | Click = open sidebar immediately | Tasks, law lists, documents, change assessments | No hover preview — the detail panel is the primary interaction |

## Acceptance Criteria

### Core Sidebar Shell

1. A `ChatDetailSidebar` component renders as a right-side panel (~400px wide) within the Hem chat conversation layout.
2. When no detail is open, the sidebar is hidden (width 0) and the chat occupies the full area as today.
3. When a detail is opened, the sidebar animates in (`transition-all duration-300`) and the chat area shrinks, with its `max-w-3xl` content naturally re-centering in the remaining space.
4. The sidebar has a header with the item title/type and a close button (X). Clicking close or pressing Escape dismisses it.
5. Only one detail can be open at a time — opening a new item replaces the current sidebar content.

### State Management

6. A `ChatDetailContext` (React context) provides `openDetail(type, data)` and `closeDetail()` to any descendant component within the chat.
7. The context supports a discriminated union of detail types: `citation`, `task`, `document`, `law-list` (extensible).
8. Opening a detail with the same type and identifier as the currently open one closes the sidebar (toggle behavior).

### Citation Integration (Tier 1: Preview + Sidebar)

9. `CitationPillInline` retains its existing hover card behavior unchanged.
10. Clicking a resolved citation pill calls `openDetail('citation', sourceData)` to open the sidebar.
11. The citation sidebar view shows: document title, document number, full section text (from source snippet or fetched), and a "Visa i lagläsaren" link to the document reader.
12. The currently-open citation pill receives a visual active state (e.g. ring or highlighted background).

### Responsive Behavior

13. On screens below `lg` (mobile/tablet), clicking an inline item opens a bottom sheet/drawer instead of a sidebar.
14. The sidebar does not render when `HemChat` is in `panel` mode (right sidebar) — inline items fall back to their default behavior (hover cards, links).

### Non-Regression

15. Existing hover card behavior on citation pills is unchanged.
16. Chat input, message list scrolling, and streaming continue to work correctly when the sidebar is open.
17. Conversation history, export, and new conversation flows are unaffected.

## Dev Technical Guidance

### Existing System Context

- **Chat layout**: `hem-chat.tsx` conversation state is a flex column with `max-w-3xl` centered content. The outer div needs to become a flex row to accommodate the sidebar.
- **Citation system**: `citation-pill.tsx` → hover via controlled `HoverCard`. Source data comes from `CitationSourceContext`. The `resolveSource()` function returns `{ title, snippet, documentNumber, slug, anchorId, path }` — this is the data to pass to the sidebar.
- **Layout store**: `lib/stores/layout-store.ts` manages sidebar states. The detail sidebar is chat-internal (React context), not a layout-store concern.

### Key Files to Modify

| File | Change |
|------|--------|
| `components/features/ai-chat/chat-detail-sidebar.tsx` | **New** — sidebar shell + citation detail view |
| `lib/contexts/chat-detail-context.tsx` | **New** — context provider + hook |
| `components/features/dashboard/hem-chat.tsx` | Wrap conversation in provider, flex row layout |
| `components/features/ai-chat/citation-pill.tsx` | Add onClick → openDetail |

### Animation Approach

```tsx
// Sidebar container — width transition
<div className={cn(
  "shrink-0 border-l border-border overflow-hidden transition-all duration-300",
  isOpen ? "w-[400px]" : "w-0"
)}>
```

The chat area's `flex-1` naturally absorbs the space change. The `max-w-3xl` on the message list re-centers automatically.

### Extensibility Pattern

Future inline types (tasks, documents, law lists) add to the discriminated union and register a detail view component:

```tsx
// In chat-detail-sidebar.tsx
switch (item.type) {
  case 'citation': return <CitationDetail data={item.data} />
  case 'task':     return <TaskDetail data={item.data} />     // future
  case 'document': return <DocumentDetail data={item.data} />  // future
}
```

### Mobile Bottom Sheet

Use the existing `Sheet` component from shadcn/ui (`components/ui/sheet.tsx`) with `side="bottom"` for mobile.

## Risk Assessment

### Implementation Risks

- **Primary Risk**: Layout shift breaking chat scroll position when sidebar opens
- **Mitigation**: The scroll container stays within the chat flex-1 area; only the available width changes. Test that auto-scroll-to-bottom still works during streaming.
- **Verification**: Open sidebar mid-stream and confirm messages continue appending correctly

### Rollback Plan

- Revert the `hem-chat.tsx` layout change — the sidebar is purely additive
- Citation pills' onClick can be removed independently of the hover behavior

## Future Extensions (Not in Scope)

- Task inline chips → `openDetail('task', taskData)` with task status, description, assignee
- Law list inline items → `openDetail('law-list', lawListData)` with applicability details
- Document references → `openDetail('document', docData)` with document preview
- Sidebar actions: "Skapa uppgift", "Markera som granskad", etc.
- Deep-link support: URL params encoding which detail is open
