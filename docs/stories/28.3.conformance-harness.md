# Story 28.3: DataTable Conformance Harness

**Epic:** 28 — Unified DataTable Core (`docs/prd/epic-28-unified-datatable-core.md`)
**Status:** Done (2026-07-08) — 11/11 Playwright + 29 Vitest green

## What was built

**Playwright conformance suite** (`tests/e2e/epic-28/`):

- `conformance.ts` — the enroll-a-table helper kit: `setHarnessWidth`,
  `expectView`, `expectNoErrorOverlay`, `collectPageErrors`,
  `expectNoRowOverlap` (virtual-row transform scan). A table enrolls by
  writing a spec that drives its own surface and calls these assertions.
- `data-table-harness.spec.ts` (7 tests, unauthed, against
  `/styleguide/data-table`): renderer switch with hysteresis-band
  verification; the 10× width-thrash test (no render loops, no
  ResizeObserver errors, no flapping); sorting in both renderers (table
  headers + card dropdown); selection surviving the renderer swap;
  expansion under virtualization at top/middle/bottom of 1,000 rows with
  overlap scanning; column-resize persistence round-trip + corrupt-payload
  degradation.
- `krav.authed.spec.ts` (3 tests, real page, read-only): URL sort survives
  reload; narrow viewport → cards + compact toolbar → restore; row click →
  law modal → Escape.
- `playwright.config.ts`: `.authed.spec.ts` naming convention routes specs
  to the authed project (the epic-21 testMatch had gone dormant — its spec
  dir no longer exists).

**Vitest persistence suite**: shipped in 28.1 (`tests/unit/data-table/`,
29 tests — sanitize contract, clamp, view resolution, flattening).
laglistor's v0–v4 payload fixtures land with Story 28.8 per the epic plan.

## Bugs the suite caught (fixed in this story)

1. **Hydration bug in `useLocalStorageColumnState` (production-class).**
   Reading localStorage in the useState initializer made the client's
   hydration render diverge from SSR HTML — React never patches the
   mismatched width attributes, and since later renders keep producing the
   same virtual values, persisted column widths silently failed to apply
   until an unrelated state change. Fix: initialize with server-identical
   defaults, load storage in a post-hydration effect. Verified live on
   /krav (persisted 300px width now applies on fresh load).
2. TanStack's alphanumeric sort is number-aware (test expectation fixed,
   documented behavior).

## Shared-infra repairs (auth.setup.ts, pre-existing breakage)

- Cookie-consent banner now dismissed before login (blocked the form).
- Submit via Enter (the button's animated shadow never passes Playwright's
  stability check).
- Multi-workspace accounts: pick the first workspace on the "Välj
  arbetsplats" screen, then **wait for the `active_workspace_id` cookie**
  before capturing storage state — capturing earlier races the server
  action and every authed spec bounces back to the picker.

## Learnings for future enrollments

- Fresh browser profiles open the AI-chat sidebar by default (480px) —
  table-view assertions need a wide viewport (1728) or a folded sidebar.
- Scope header-button locators to `getByRole('table')` — nav items share
  labels with column headers (Regelverk).
- Dev-server on-demand compilation needs generous first-load timeouts.
