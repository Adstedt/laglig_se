## Story

**As a** user adding sections to a styrdokument with the agent,
**I want** a single approved section-add on an already-open draft to update that draft in place,
**so that** adding sections one at a time doesn't re-introduce the version-number inflation that Story 17.22 removed for edits.

## Context

Story 17.22 stopped version-number inflation for agent edits on an open draft — but scoped to `UPDATE_DOCUMENT`. The **single** `ADD_DOCUMENT_SECTION` dispatch arm ([`pending-agent-actions.ts`](../../app/actions/pending-agent-actions.ts):904) still calls `saveDocumentVersion`, which mints a new version every time. (Section-adds *are* already handled in place by the **consolidated** path, since `applyConsolidatedDocumentEdits` folds update + add into one write — so this gap only affects single, non-batched section-adds.) The result is an inconsistency: UPDATE single = in-place, ADD consolidated = in-place, but ADD single = mints a version. This story removes the inconsistency.

## Acceptance Criteria

1. **In-place add when a draft is open.** When approving a single `ADD_DOCUMENT_SECTION` proposal where the live document has `current_draft_version_id != null`, the dispatch routes the non-branch arm to `updateDraftVersionInPlace` (instead of `saveDocumentVersion`) — the section-added tree is written to the existing draft row with **no** new `WorkspaceDocumentVersion` and **no** `current_version_number` bump.
2. **First-edit branch unchanged.** APPROVED-no-draft + `creates_draft` still branches exactly one draft via `createDraftFromApprovedWithEdit` (unchanged).
3. **Per-edit audit trail + authorship stamp.** The in-place add writes a `document_draft_edited` ActivityLog row; the dispatch stamps agent authorship by the returned audit-row id with an `operation` discriminator (e.g. `in_place_add_section`) — mirroring 17.22's single-path fork (`:659–746`). The branch arm keeps stamping `document_version_saved` by version_number.
4. **Non-regression.** Consolidated batches (already in place) unchanged; `saveDocumentVersion` non-agent callers unchanged; APPROVED-no-draft Path C guard intact.
