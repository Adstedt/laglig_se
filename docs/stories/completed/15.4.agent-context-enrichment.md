# Story 15.4: Agent Context Enrichment

## Status

Done

## Story

**As a** user chatting with the compliance agent,
**I want** the agent to know my company's tax registration, business description, foreign ownership, and regulatory status,
**so that** it gives me more precisely tailored compliance advice without me having to explain my situation.

## Pre-requisites / Already Delivered

- **Story 15.1 (Done):** Schema migration added enrichment fields to `CompanyProfile` (all nullable). `BolagsApiClient` with `fetchCompany()` + `mapBolagsApiToProfile()`.
- **Story 15.2 (Done):** Onboarding auto-fill persists enrichment fields via `createWorkspace()`. Settings page shows Bolagsverket-data card. `business_description` already in profile completeness (+5%, clamped at 100). `calculateProfileCompleteness()` already includes `business_description`.

This story surfaces the enrichment data in the **agent system prompt** and **get_company_context tool** тАФ no schema changes needed.

## Acceptance Criteria

### formatCompanyContext Updates

1. `formatCompanyContext()` in `lib/agent/system-prompt.ts` updated to include new fields:
   - `business_description` тЖТ `"- Verksamhet: {description}"` (free-text, richer than SNI label)
   - `tax_status.f_tax` тЖТ `"- F-skatt: Ja/Nej"`
   - `tax_status.vat` тЖТ `"- Momsregistrerad: Ja/Nej"`
   - `tax_status.employer` тЖТ `"- Registrerad arbetsgivare: Ja/Nej"`
   - `foreign_owned` тЖТ `"- Utlands├дgt: Ja (moderbolag: {parent_company_name})"` (only if true)
   - `fi_regulated` тЖТ `"- Finansinspektionen-reglerad: Ja"` (only if true)
   - `ongoing_procedures` тЖТ `"- P├еg├еende f├╢rfaranden: {type}"` (only if present)
   - `active_status` тЖТ only surface if deregistered: `"- Status: Avregistrerad"`
   - `registered_date` / `founded_year` тЖТ `"- Registrerad: {year}"`
2. New fields appended after existing fields тАФ existing output unchanged for profiles without new data
3. When `business_description` is available, it appears right after company name (high agent value)
4. `tax_status.employer === true` reinforces the self-reported `activity_flags` signal тАФ both are displayed; no deduplication logic needed. The agent sees both signals and can use the authoritative tax registration to say "ni har anst├дllda" with certainty

### get_company_context Tool Updates

5. `get_company_context` tool in `lib/agent/tools/get-company-context.ts` updated to include new fields in its response:
   - `businessDescription`, `taxStatus`, `foreignOwned`, `parentCompanyName`, `fiRegulated`, `ongoingProcedures`, `activeStatus`, `registeredDate`
6. Tool description updated to mention the new data points available

### Profile Completeness Updates

7. `calculateProfileCompleteness()` in `lib/profile-completeness.ts` updated:
   - `tax_status` added as extended field (5%)
   - `business_description` already counted (from Story 15.2) тАФ no change
   - `registered_date` or `founded_year` already counted тАФ no change
   - Total possible score remains clamped at 100

### Data Source Indicator

8. When agent calls `get_company_context`, response includes `dataSource: "bolagsapi" | "manual" | null` and `lastEnrichedAt: ISO date | null`
9. If `dataSource === "manual"` and profile is incomplete, agent's existing behavior (suggest filling profile) continues unchanged
10. If `dataSource === "bolagsapi"`, agent can trust the data as authoritative and use it confidently in advice

### Testing

11. At least 8 unit tests covering:
    - `formatCompanyContext()`: profile with all new fields, profile with no new fields (backward compat), profile with partial new fields
    - `get_company_context` tool: returns new fields, handles nulls
    - `calculateProfileCompleteness()`: scores correctly with new field (`tax_status`)
    - Backward compatibility: existing profiles without new fields produce identical output as before
