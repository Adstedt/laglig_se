**`WorkspaceDocumentTemplate` â€” no `is_active` unique constraint, no status enum:**

The model has `is_active: Boolean @default(true)` â€” NOT a status enum. There is no `DocumentTemplateStatus` enum. The `is_active` flag is sufficient for filtering active/inactive templates.

Since `name` has no unique constraint, the seed script uses deterministic UUIDs for idempotent upserts:

**Auth pattern for `getDocumentTemplates()`:**

Use `withWorkspace()` for auth consistency â€” templates are global but the user must be authenticated. The function doesn't need `workspaceId` for the query, but wrapping in `withWorkspace()` ensures only authenticated workspace members can access templates.

```typescript
export async function getDocumentTemplates(): Promise<ActionResult<unknown>> {
  try {
    return await withWorkspace(async () => {
      const templates = await prisma.workspaceDocumentTemplate.findMany({
        where: { is_active: true },
        orderBy: { sort_order: 'asc' },
        select: { id: true, name: true, description: true, document_type: true, content_json: true, sort_order: true },
      })
      return { success: true, data: templates }
    })
  } catch (error) { ... }
}
```
