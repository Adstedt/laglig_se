# Story 17.3: Document Versioning — History, Diff & Restore

## Status

Done

## Story

**As a** workspace member,
**I want** to view version history, compare changes between versions, and restore previous versions,
**so that** I have full traceability of document changes for audit and compliance purposes.

## Story Context

**Existing System Integration:**

- Integrates with: Tiptap editor from Story 17.2, `WorkspaceDocument`/`WorkspaceDocumentVersion` models from Story 17.1, existing server actions in `app/actions/documents.ts` (from Stories 17.1 + 17.2), `ActivityLog` model
- Technology: `diff` npm package (already installed: `"diff": "^8.0.2"`, types: `"@types/diff": "^8.0.0"`), React
- Follows pattern: `withWorkspace()` auth from `@/lib/auth/workspace-context`, `ActionResult<T>` return type, ActivityLog audit pattern

- `WorkspaceDocumentVersion` (Prisma: `prisma.workspaceDocumentVersion`)
- Enums: `WorkspaceDocumentType`, `WorkspaceDocumentStatus`, `WorkspaceDocumentVersionSource`

## Acceptance Criteria

**Version History:**

1. Version history panel accessible from document editor (sidebar drawer or slide-out panel)
2. Version list shows per entry: version number, author (name + avatar), timestamp (relative + absolute), change summary text, source badge (Tiptap / Import / Agent)
3. Every save creates a new immutable `WorkspaceDocumentVersion` row with incremented `version_number`
4. Auto-save batching: rapid edits within a 30-second window are consolidated into a single version — the autosave hook suppresses server saves during the window and only commits when the window expires (preserving row immutability)
5. Current version clearly indicated in the version list (e.g., badge or highlight)
6. Version count badge visible on the document metadata bar in editor

**Diff View:**

7. User can select any two versions to compare side-by-side or inline diff
8. Diff highlights: added text (green), removed text (red), changed text (yellow/amber)
9. Diff computed from `extracted_text` or `content_html` (not raw JSON)
10. Diff view is read-only (cannot edit from diff view)

**Restore:**

11. "Återställ version" (restore) button on any historical version
12. Restoring creates a NEW `WorkspaceDocumentVersion` from the old version's content (does not delete any later versions — immutability preserved)
13. Restore version has `change_summary` auto-set to "Återställning från version {N}"
14. After restore, editor loads the restored content as the current version

**Audit Trail:**

15. ActivityLog entries created for: `document_version_saved`, `document_version_restored`
16. ActivityLog entries include `entity_type: 'document'`, `entity_id: documentId`, `action`, and relevant `new_value` (version number, change summary)

**Integration Requirements:**

17. Auto-save from Story 17.2 integrates with the 30-second batching window
18. Existing editor functionality unaffected by version panel addition

**Quality Requirements:**

19. Version history loads within 1 second for documents with up to 200 versions
20. Diff computation completes within 2 seconds for documents up to 50 pages

## Dev Notes

**`diff` npm package — already installed:**

The `diff` package (`^8.0.2`) is already in `package.json` with `@types/diff` for TypeScript support. No installation needed.

```typescript
import { diffWords } from 'diff'

export function computeDiff(oldText: string, newText: string) {
  return diffWords(oldText, newText)
  // Returns: [{ value: string, added?: boolean, removed?: boolean }]
}
```

**Rendering diff segments:**

```tsx
{segments.map((seg, i) => (
  <span
    key={i}
    className={cn(
      seg.added && 'bg-green-100 text-green-900',
      seg.removed && 'bg-red-100 text-red-900 line-through',
      // unchanged text: no background — only added/removed are highlighted
    )}
  >
    {seg.value}
  </span>
))}
```

**Version batching — client-side suppression (NOT server-side UPDATE):**

The epic's key principle is "`WorkspaceDocumentVersion` rows are never updated — every committed version is immutable." The 30-second batching is implemented **client-side** in the autosave hook:

1. Auto-save debounce fires → check if 30+ seconds since `lastVersionSavedAt`
2. If yes → call `saveDocumentVersion()` as normal → new immutable row
3. If no → re-schedule the save for when the window expires (`setTimeout(save, remainingMs)`)
4. Manual save always bypasses the window

This prevents hundreds of micro-versions while preserving row immutability. No `updateDocumentVersion()` action is needed.

**`saveDocumentVersion()` — existing signature (from Story 17.2):**

```typescript
export async function saveDocumentVersion(
  documentId: string,
  contentJson: object,
  changeSummary?: string,
  title?: string
): Promise<ActionResult<{ id: string; versionNumber: number }>>
```

The hook tracks `lastVersionSavedAt` client-side using `Date.now()` on successful save — no need to change the return type.

**`getDocumentVersions()` — author name resolution needed:**

The existing action (Story 17.1) returns `created_by` as a plain string. `WorkspaceDocumentVersion.created_by` has no User FK (to allow `"agent"` as a value). To display author names + avatars (AC 2), enhance `getDocumentVersions()` to:
1. Collect unique `created_by` values (excluding `"agent"`)
2. Run `prisma.user.findMany({ where: { id: { in: userIds } }, select: { id, name, avatar_url } })`
3. Merge user info into the response, with fallback for `"agent"` → `{ name: "AI-assistent", avatar_url: null }`

**Diff view — fetching version content:**

`getDocumentVersions()` currently selects `id, version_number, source, change_summary, created_by, created_at` — it does NOT include `extracted_text` or `content_html`. For the diff view, create a `getDocumentVersionContent(documentId, versionId)` action that fetches the `extracted_text` of a specific version, or fetch both versions' text on demand when the user opens the diff dialog.
