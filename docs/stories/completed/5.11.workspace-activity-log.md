# Story 5.11: Enterprise Activity Log UI & CSV Export

## Status

Done — implemented as part of Story 6.10

## Story

**As an** Enterprise workspace admin,
**I want** to browse and export a filterable audit trail of workspace activity,
**so that** I can demonstrate compliance and investigate incidents.

## Context & Dependencies

- **Builds on:** Existing `ActivityLog` model in `prisma/schema.prisma` (added in Epic 6) — already logs entity changes with `entity_type`, `entity_id`, `action`, `old_value`, `new_value`
- **Builds on:** Story 5.2 (completed) — role-based permissions (`activity:view`)
- **Implemented by:** Story 6.10 (Global Activity Log Page) — the UI, server action, filters, pagination, and CSV export were all built under 6.10
- **Note:** This story does NOT create a new `WorkspaceActivityLog` model. The existing `ActivityLog` model is the single source of truth.
- **Schema reference:** `prisma/schema.prisma` lines 1143-1167

## Acceptance Criteria

1. ~~Activity Log page accessible at `/settings/activity-log`~~ → Implemented at `/workspace/activity` (`app/(workspace)/workspace/activity/page.tsx`)
2. ~~Page gated to Enterprise tier~~ → Not yet gated; currently accessible to all workspace members via `withWorkspace`
3. ~~Only users with `activity:view` permission can access~~ → Auth handled by `withWorkspace`; explicit `activity:view` permission check not wired in
4. ✅ Activity list shows: timestamp, user (name + avatar), action, entity type, entity ID
5. ✅ Filterable by: User, Entity type (multi-select), Action (multi-select), Date range
6. ✅ Paginated (50 items per page) with cursor-based "Ladda fler" button
7. ✅ CSV export via `/api/workspace/activity-log/export/route.ts` — Swedish labels, proper CSV escaping, 10k row cap
8. ❌ Log retention cron job NOT implemented — no `/api/cron/cleanup-activity-logs` route exists
9. ✅ Performance: queries use existing indexes on `[workspace_id, created_at]` and `[entity_type, entity_id]`

### Implementation Differences from Original Spec

- **No API route for list** — uses a server action (`withWorkspace`) instead of a REST endpoint
- **No Enterprise tier gate** — page is accessible to all workspace members
- **No `requirePermission('activity:view')`** — auth is via `withWorkspace` only
- **Filters are multi-select** — `entityType` and `action` accept arrays (more flexible than original spec)
- **CSV includes more columns** — Användare, E-post, Gammalt/Nytt värde (original spec only had 5 columns)
