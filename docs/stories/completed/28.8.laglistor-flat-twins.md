# Story 28.8: Laglistor Flat Twins on the Core

**Epic:** 28 — Unified DataTable Core (`docs/prd/epic-28-unified-datatable-core.md`)
**Status:** Done (2026-07-08)

## Core capability added

**Row drag-reorder** (`dnd: 'self' | 'external'`): injected drag-handle
chrome column, sortable rows (useSortable grips, virtual + plain modes),
core-owned DndContext with vertical restriction in 'self' mode, and an
**optimistic order overlay** — the visual order holds after a drop until
fresh data arrives (identity change resets), replacing the legacy
localItems pattern. 'external' preserves the legacy `disableDndContext`
contract for nested grouped use. Chrome columns are pinned leftmost even
against persisted column orders that predate their ids.

## Migrations

- **`document-list-table.tsx`** (1,478 → ~900 lines, all domain): all 6
  inline editors, header tooltips, change indicator, high-risk warning,
  task-progress/last-activity cells, group editor — verbatim. Chrome
  select/drag columns now injected; `hideGroupColumn` maps to a visibility
  merge; reorder persists via the same 500ms-debounced `onReorderItems`.
  Dokument column is stickyLeft + fill. Cards below 800px with interactive
  status/priority badges and editor meta rows.
- **`compliance-detail-table.tsx`** (1,661 → 1,052 lines): expansion moved
  onto the core's controlled ExpandedState (`row.toggleExpanded()` chevron;
  ExpandedRowContent unwrapped from its TableCell — the core supplies the
  cell), `rowHeight: 'tall'` (72px), same reorder/selection/chrome
  treatment. TruncatedTextCell / ExpandedRowContent / KravpunkterCountCell /
  HeaderWithTooltip preserved verbatim; the local SortableHeader copy and
  both legacy row components deleted.
- **Store untouched**: both persisted column-state slots keep their exact
  shape — no v5 migrate was needed (the plan's v5 bump was contingent on a
  shape change that never materialized). Stale `select`/`dragHandle` ids in
  persisted sizing/order are inert.
