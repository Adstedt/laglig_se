# Story 28.7: Personalregister — GroupedDataTable

**Epic:** 28 — Unified DataTable Core (`docs/prd/epic-28-unified-datatable-core.md`)
**Status:** Done (2026-07-08). Epic 7 was merged (PR #92) before this ran;
the user explicitly authorized proceeding ahead of formal checklist closure.

## Core capabilities added (personalregister as first consumer)

- **`GroupedDataTable`**: collapsible sections over one column definition,
  each section its own DataTable; controlled/internal `collapsed` Set;
  `perSectionSorting` (independent local sort per section — hook lives in
  a per-section component); per-section `empty` content; `sectionDnd` with
  the parent-owned DndContext, header-prioritizing collision detection,
  droppable headers (`data-dt-section-trigger` hook for tests) and a
  `renderDragOverlay` slot. The drag SOURCE stays consumer-side (a
  `useDraggable` grip cell) — no domain dnd in the core.
- `meta.dt.nowrap` (clip cell content — the "personnummer wraps to two
  lines" checkpoint), `group/head` on core header cells (hover-reveal sort
  affordance support).

## Migration

`employee-list-table.tsx` (1,178 → ~870 lines, almost all domain):

- All masking-aware cells verbatim (personnummer repo-masked/formatted,
  salary mask-or-format, "Ej ifylld" empties, completeness badge).
- Sections: groups + Ogrupperad last; headers carry folder icon + the
  unfiltered "{n}/{m} kompletta" rollups (QA UX-001 source preserved);
  per-section empty distinguishes truly-empty ("Dra anställda hit") from
  filtered-empty ("Inga träffar i denna grupp").
- Drag-into-group via the grip cell; overlay = name card; move handler
  keeps the "no-op when dropped on its own group" guard.
- **Persistence contract untouched**: column state remains parent-owned
  through `employee-column-state.ts` (per-workspace localStorage key,
  sanitize-on-read, salary default-hidden). The module was not modified.
- Flat mode (no groups) renders a plain DataTable; both modes get cards
  below 800px — the register's first mobile treatment. Salary is
  card-hidden unconditionally (screen-share privacy).

## Behavioral deltas (deliberate, small)

- Drop-target highlight shows on any header hover during a drag (legacy
  highlighted only when the drop would actually move the employee); the
  no-op drop guard itself is preserved.
- The dragged row no longer dims in place (the drag source is the grip,
  not the row); the DragOverlay carries the visual feedback.

## Verification

- tsc, eslint, 29 unit tests green; full epic-28 suite green (19 checks).
- Live (Nordviken): flat mode + card flip verified against real employees.
  **Grouped mode not exercisable live — the workspace has no employee
  groups**; the two grouped specs (rollups + collapse, per-section sort
  independence) self-skip and activate when groups exist. First manual
  pass with groups should verify: collapse-all/expand-all, rollup counts
