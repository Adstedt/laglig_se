# Story 17.5: .docx Import — Upload & Convert to Tiptap

## Status

Done

## Story

**As a** workspace member,
**I want** to import existing Word documents into Laglig,
**so that** I can bring my current compliance documentation into the system without rewriting everything from scratch.

## Story Context

**Existing System Integration:**

- Integrates with: `WorkspaceDocument` and `WorkspaceDocumentVersion` models from Story 17.1, document creation flow from Story 17.2, Supabase Storage (existing `workspace-files` bucket via `getStorageClient()` from `@/lib/supabase/storage`)
- Technology: `mammoth` npm package (.docx to HTML — **not yet installed, Task 0**), `cheerio` (HTML parsing — already installed: `^1.1.2`), custom HTML to Tiptap JSON transformer
- Follows pattern: `withWorkspace()` auth from `@/lib/auth/workspace-context`, `ActionResult<T>` return type, server action mutation pattern in existing `app/actions/documents.ts` (already ~900 lines from Stories 17.1-17.4), Zod validation schemas in `lib/validation/documents.ts`, file upload pattern from `app/actions/files.ts`
- Touch points: Document creation dialog, existing `app/actions/documents.ts` (add import action), `lib/validation/documents.ts` (add import schema)

**Model naming — `WorkspaceDocument*` prefix required:** The existing schema has `model DocumentVersion` and `enum DocumentStatus` for the SFS legal document system. All DMS models use the `WorkspaceDocument*` prefix:
- `WorkspaceDocument` (Prisma: `prisma.workspaceDocument`)
- `WorkspaceDocumentVersion` (Prisma: `prisma.workspaceDocumentVersion`)
- Enums: `WorkspaceDocumentType`, `WorkspaceDocumentStatus`, `WorkspaceDocumentVersionSource` (already has `IMPORT` value)

## Acceptance Criteria

**Import Flow:**

1. "Importera dokument" button available in document creation area
2. Accepts `.docx` files only (not `.doc` — legacy format not supported)
3. Maximum file size: 25MB per .docx file
4. Upload converts .docx to HTML (via `mammoth`) to Tiptap JSON (via custom transformer)
5. Import dialog shows preview of converted content before user confirms creation
6. User can set title (pre-filled from filename), document type, and document number during import
7. On confirm: creates `WorkspaceDocument` + `WorkspaceDocumentVersion` v1 with `source: 'IMPORT'`, logged to ActivityLog as `document_created`

**Conversion Quality:**

8. Conversion preserves: headings (H1-H6 mapped to H1-H3), paragraphs, bold, italic, underline, bullet lists, numbered lists, tables, images, links
9. Original .docx stored in Supabase Storage as archival copy (referenced by `WorkspaceDocumentVersion.storage_path`)
10. Clear messaging in import dialog about potential format loss: "Komplex formatering som sidhuvud, sidfot, kolumner och makron kan gå förlorade vid import."

**Batch Import:**

11. Accept multiple .docx files at once — each becomes a separate `WorkspaceDocument`
12. Progress indicator for batch imports (file X of Y)

**Integration Requirements:**

13. Existing file upload functionality (`WorkspaceFile`) unchanged — import is a separate flow
14. Imported documents are fully editable in Tiptap editor after import

**Quality Requirements:**

15. Import of a 20-page .docx completes within 5 seconds
16. Conversion handles Swedish characters (UTF-8) correctly

**mammoth conversion API:**

```typescript
import mammoth from 'mammoth'

export async function convertDocxToHtml(buffer: Buffer): Promise<{ html: string; messages: string[] }> {
  const result = await mammoth.convertToHtml(
    { buffer },
    {
      styleMap: [
        "p[style-name='Heading 1'] => h1",
        "p[style-name='Heading 2'] => h2",
        "p[style-name='Heading 3'] => h3",
        "p[style-name='Rubrik 1'] => h1",  // Swedish heading style names
        "p[style-name='Rubrik 2'] => h2",
        "p[style-name='Rubrik 3'] => h3",
      ],
    }
  )
  return { html: result.value, messages: result.messages.map(m => m.message) }
}
```

**Tiptap JSON structure reference:**

```json
{
  "type": "doc",
  "content": [
    { "type": "heading", "attrs": { "level": 1 }, "content": [{ "type": "text", "text": "Title" }] },
    { "type": "paragraph", "content": [
      { "type": "text", "text": "Normal text " },
      { "type": "text", "marks": [{ "type": "bold" }], "text": "bold text" }
    ]}
  ]
}
```
