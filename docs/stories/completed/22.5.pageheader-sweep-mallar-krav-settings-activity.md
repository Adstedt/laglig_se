# Story 22.5: PageHeader sweep — Mallar / Krav / Settings / Activity

## Status

Done

## Story

**As a** developer working on workspace UI,
**I want** the four remaining hand-rolled page headers (Mallar, Krav, Settings, Activity) migrated to the canonical `<PageHeader>` primitive shipped in Story 22.3,
**so that** every workspace surface uses the same title chrome (slot order, typography, separator, primary/secondary action placement) and new surfaces have no precedent for hand-rolling a header.

## Background

Story 22.3 introduced `<PageHeader>` and migrated the six tabular surfaces it identified (Laglistor, Kontroller, Cycle detail, Tasks, Styrdokument, Findings). A follow-on cross-workspace audit identified four additional surfaces still using hand-rolled `<div><h1><p></div>` patterns. This story migrates them.

## Acceptance Criteria

### Functional

1. **Mallar** (`app/(workspace)/laglistor/mallar/page.tsx`) — replace inline `<div><h1>Mallar</h1><p>...</p></div>` with `<PageHeader title="Mallar" subtitle="..." />`. Outer `space-y-4` wrapper preserved.

2. **Krav** (`components/features/krav/krav-page-content.tsx`) — header lives in the client component, not the page route. Replace the `<div className="flex items-start justify-between"><div><h1>Krav</h1><p>...</p></div></div>` block (~lines 392-402) with `<PageHeader title="Krav" subtitle="Alla kravpunkter i din arbetsyta — filtrera efter luckor, mina krav eller krav som saknar bevis." />`. Filter toolbar block immediately below stays at same indent.

3. **Settings** (`app/(workspace)/settings/page.tsx`) — happy-path only. Replace inner `<div><h1 className="text-2xl font-semibold">Inställningar</h1><p>...</p></div>` with `<PageHeader title="Inställningar" subtitle="Hantera din arbetsplats och dina preferenser" />`. **Error path** at lines 96–102 (workspace-not-found fallback) **left untouched** — it's an error UI, not a workspace page header.

4. **Activity** (`app/(workspace)/workspace/activity/page.tsx`) — replace `<div className="flex items-start justify-between">` wrapping title block + Exportera CSV button. Export button moves to `<PageHeader.secondaryActions>` slot (with `size="sm"` to match toolbar density).

### Non-functional

5. **Zero behaviour change.** Subtitles preserved verbatim; action handlers (export CSV) preserved; no copy changes.

6. **Visual parity.** All four surfaces' headers now visually match Laglistor / Tasks / Kontroller / Styrdokument:
   - `text-2xl font-semibold tracking-tight` title
   - `text-sm text-muted-foreground` subtitle
   - Trailing `<Separator>` divider (from `<PageHeader>` internals)
