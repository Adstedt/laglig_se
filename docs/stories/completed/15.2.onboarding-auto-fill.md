# Story 15.2: Onboarding Auto-Fill

## Status

Done

## Story

**As a** new user entering my org number during onboarding,
**I want** the form to automatically fill in my company details,
**so that** I don't have to manually type information that's already public.

## Acceptance Criteria

### API Route

1. New API route `POST /api/company/lookup` accepts `{ orgNumber: string }`
2. Server-side call to `fetchCompany()` from `lib/bolagsapi/client.ts` (Story 15.1)
3. Returns mapped company data via `mapBolagsApiToProfile()` (Story 15.1) **plus** raw address components (`street`, `postal_code`, `city`) from the original API response for individual form field population
4. Rate limited: max 10 lookups per minute per user session (Upstash ratelimit)
5. Returns 404 with `{ error: "company_not_found" }` if org number doesn't exist
6. Returns 503 with `{ error: "service_unavailable" }` if API is down — form continues working manually

### Onboarding UX

7. When user types a complete org number (10 digits with or without dash) in the org number field, a lookup is triggered after 500ms debounce
8. Loading state: spinner icon inside the org number input field + subtle "Hamtar foretagsinfo..." text below
9. On success: form fields auto-populated (company name, street address, postal code, city, SNI code, legal form). Employee count is NOT auto-filled — the user is asked this during onboarding. Fields animate in with a brief fade.
10. On failure (API down): no error shown to user — form simply remains editable as before. Console warning logged.
11. On 404 (company not found): subtle inline message "Inget foretag hittades med detta organisationsnummer" below org number field — form remains editable
12. All auto-filled fields remain editable — user can override any value
13. If user edits org number after auto-fill, previous auto-fill is cleared and new lookup triggered
14. Company name field shows "(Hamtat fran Bolagsverket)" badge when auto-filled, removed if user edits

### Data Persistence

15. `createWorkspace()` in `app/actions/workspace.ts` updated to persist new enrichment fields (business_description, tax_status, foreign_owned, parent_company_name, parent_company_orgnr, fi_regulated, active_status, ongoing_procedures, registered_date, data_source, last_enriched_at)
16. If org number belongs to a deregistered company (`active_status === "deregistered"`), show warning: "Detta foretag ar avregistrerat hos Bolagsverket" — allow signup but warn
17. Validation schema `WorkspaceOnboardingSchema` extended with new optional fields that can be auto-filled
18. CompanyProfile created with `data_source: "bolagsapi"` when auto-filled, `data_source: "manual"` when typed manually

### Settings Page Enrichment Visibility

19. Settings company profile tab (`company-profile-tab.tsx`) shows a new "Bolagsverket-data" card when `data_source === "bolagsapi"` — card is hidden when no enrichment data exists
20. Card displays read-only enrichment fields: tax status (F-skatt / Moms / Arbetsgivare as badges), active_status, registered_date, foreign_owned, parent company (name + orgnr), fi_regulated, ongoing procedures (liquidation / restructuring)
21. `business_description` is shown as an editable text field (valuable context the user may want to refine for the AI agent)
22. Card header shows "Hamtat fran Bolagsverket" badge with formatted `last_enriched_at` date
23. `updateCompanyProfile` server action extended to accept and persist `business_description` edits
24. Profile completeness score updated: `business_description` added as an extended field (+5%)

### Testing

25. At least 10 unit tests covering:
    - API route: successful lookup, rate limit, 404, service unavailable
    - Debounce: triggers after 500ms, cancels on new input
    - Auto-fill: populates correct fields, clears on org number change
    - Persistence: enrichment fields saved to CompanyProfile
    - Settings card: renders enrichment fields when data_source is "bolagsapi", hidden when no enrichment data

### API Specifications

**POST /api/company/lookup** — NEW route to create:
- Request: `{ orgNumber: string }` — validated with Zod
- Auth: Requires session (use `getServerSession()`)

### Auto-Fill Field Mapping (API response -> form fields)

The API route returns `{ profile: Partial<MappedProfile>, address: { street?, postal_code?, city? } }`. Use the raw address components (not the mapper's composed string) for form fields:

```
Response field         ->  Form field (setValue key)    Notes
profile.company_name   ->  companyName                  Also shows badge when auto-filled
address.street         ->  streetAddress                Raw from BolagsApiCompany.address
address.postal_code    ->  postalCode                   Raw from BolagsApiCompany.address
address.city           ->  city                         Raw from BolagsApiCompany.address
profile.sni_code       ->  sniCode                      Code only (description stored separately)
profile.legal_form     ->  legalForm                    Mapper outputs only: AB|HB|KB|EF|OVRIGT
                           employeeCount                NOT auto-filled — asked during onboarding
```

**Enrichment fields** (no form input, set via hidden `setValue()` calls — flow through data model):
```
profile.business_description  ->  businessDescription
profile.tax_status            ->  taxStatus              JSON.stringify() before setValue
profile.foreign_owned         ->  foreignOwned
profile.parent_company_name   ->  parentCompanyName
profile.parent_company_orgnr  ->  parentCompanyOrgnr
profile.fi_regulated          ->  fiRegulated
profile.active_status         ->  activeStatus
profile.ongoing_procedures    ->  ongoingProcedures      JSON.stringify() before setValue
profile.registered_date       ->  registeredDate         .toISOString() before setValue
profile.data_source           ->  dataSource             Always "bolagsapi" when auto-filled
```
