# Story 8.24: Enrich Amendments with Proposition Context from Riksdagen API

## Status

Done

## Story

**As a** compliance officer reviewing an amendment assessment,
**I want** the AI to understand *why* a law was changed (not just *what* changed),
**so that** assessments accurately describe the legislative intent and avoid mischaracterizing the nature of the change.

## Context & Dependencies

### The Problem

Amendment documents (SFS texts) describe *what* changed — which sections, what the new wording is. But they don't explain *why*. The government proposition (proposition) that preceded the amendment contains a structured "Propositionens huvudsakliga innehåll" section that concisely explains the purpose, context, and intent of the law change.

Without this context, the AI must guess the intent from the legal text alone, leading to mischaracterizations. In testing (Story 8.22), the AI described SFS 2025:1079 as a "förstärkning av totalförsvarets inflytande" (strengthening of defense authority powers) when the proposition title makes clear it's a name change: "Myndigheten för civilt försvar – ett nytt namn för Myndigheten för samhällsskydd och beredskap."

### Data Quality Analysis (2026-04-02)

- **1,311** SFS 2025 amendments have `full_text` (100% coverage)
- **537** (41%) contain a `Prop.` reference — these are riksdag-decided law changes (lagar)
- **774** (59%) are government ordinances (förordningar) with no proposition — they use "Regeringen föreskriver" instead
- The riksdagen.se API reliably returns proposition data for all tested references
- The "Propositionens huvudsakliga innehåll" section averages **151 words / 1,075 chars** — small enough for prompt injection

### Riksdagen API

- **Search endpoint**: `https://data.riksdagen.se/dokumentlista/?sok={prop_ref}&doktyp=prop&utformat=json`
  - Returns structured JSON with `titel`, `organ` (ministry), `datum`, `dok_id`
- **Text endpoint**: `https://data.riksdagen.se/dokument/{dok_id}.text`
  - Returns full proposition text (HTML-like) with extractable "Propositionens huvudsakliga innehåll" section
- No authentication required, public API
- Both calls are fast (< 1s combined)

4. The assessment system prompt (`loadChangeContext`) includes proposition title and summary when available, formatted as a "Bakgrund" section
5. When no proposition data is available (förordningar, API failure), the prompt gracefully omits the section
6. A backfill script populates proposition data for existing amendments that have `Prop.` references in their `full_text`
7. All existing tests pass, `pnpm build` clean, `tsc` clean

### Proposition Reference Extraction

Regex pattern: `/[Pp]rop\.\s*(\d{4}\/\d{2}:\d+)/`

Examples from amendment full_text:
- `"Prop. 2024/25:205"` → `"2024/25:205"`
- `"prop. 2024/25:171"` → `"2024/25:171"`

**Multiple references:** Some amendments reference multiple propositions (e.g., "Prop. 2024/25:205, prop. 2024/25:169"). Extract the **first** one — it's the primary proposition for this specific amendment. Subsequent references are typically cross-references to other propositions cited in the preamble footnotes.

### API Call Sequence

```
1. Extract prop ref from full_text: "2024/25:205"
2. Search: GET https://data.riksdagen.se/dokumentlista/?sok=2024/25:205&doktyp=prop&utformat=json
   → Response: { dokumentlista: { dokument: [{ titel, organ, datum, dok_id, dokument_url_text }] } }
   → Take first result (highest relevance)
3. Fetch text: GET https://data.riksdagen.se/dokument/{dok_id}.text
   → Extract section after "Propositionens huvudsakliga innehåll" heading
   → Strip HTML tags, take content until next page break
   → Average: 151 words / 1,075 chars
```

### Prompt Integration

The proposition context adds ~150 words to the system prompt. Combined with the amendment full text (avg 2-5K chars) and section changes line, the total change context block stays well under budget.

Format in prompt:
```
## Bakgrund (från propositionen)
**Myndigheten för civilt försvar – ett nytt namn för Myndigheten för samhällsskydd och beredskap** (Försvarsdepartementet)
I propositionen föreslås att Myndigheten för samhällsskydd och beredskap (MSB) ska byta namn till Myndigheten för civilt försvar. Det nya namnet syftar till att tydliggöra och markera myndighetens uppdrag inom civilt försvar. Med anledning av namnbytet föreslår regeringen ändringar i de lagar som innehåller myndighetens nuvarande namn. Lagändringarna föreslås träda i kraft den 1 januari 2026.
```

### Error Handling

- API timeout: 5s timeout on both calls, return null on timeout
- No search results: return null (some props may not be indexed yet)
- Malformed response: return null, log warning
- No "huvudsakligt innehåll" section in text: fall back to `titel` + `organ` only (skip `summary`)
- Rate limiting for backfill: 2 requests/sec (respectful to public API)

### Testing

- **Framework:** Vitest
- **Proposition fetcher tests:** Mock `fetch` globally, test extraction regex (various formats), test API parsing (success, empty, error), test HTML stripping
- **System prompt tests:** Add mock with `proposition_title`/`proposition_summary` set → verify "Bakgrund" section appears; add mock with null fields → verify omitted
- **Key scenarios:**
  1. Amendment with prop reference → proposition data fetched and stored
  2. Amendment without prop reference (förordning) → gracefully skipped
  3. API returns no results → null fields, amendment still saved
  4. API timeout → null fields, amendment still saved
  5. Prompt with proposition data → "Bakgrund" section included
  6. Prompt without proposition data → "Bakgrund" section omitted

### Note

The 59% of amendments without proposition references are government ordinances (förordningar). These don't go through riksdagen and have no proposition. The tool gracefully handles this — it simply doesn't add proposition context for ordinances. The amendment full text (from Story 8.22 changes) remains the primary context for all amendments regardless.
