## Story

**As a** new user going through onboarding,
**I want** to answer 3-5 quick yes/no questions about my business,
**so that** the system can accurately identify which regulations apply to my company beyond what's derivable from the org number alone.

## Acceptance Criteria

### Question Step

1. New onboarding wizard step between "Company Info" and "Confirm": "Verksamhetsfrågor"
2. Step shows 3-5 yes/no toggle questions, dynamically selected based on company profile
3. Questions are rendered as `Switch` toggles with Swedish labels, each with `aria-describedby` linking to its description text (consistent with settings page activity flags pattern)
4. Step is skippable — user can proceed without answering (all default to false)
5. Step header: "Hjälp oss förstå er verksamhet bättre" with subtext "Ju mer vi vet, desto bättre kan vi identifiera relevanta regler."

### Question Selection Logic

6. Questions are selected from a static pool based on company profile signals:
   - **Always asked:** "Hanterar ni personuppgifter utöver anställda?" (personalData), "Har ni kollektivavtal?" (has_collective_agreement)
   - **If SNI starts with 41-43 (construction):** "Bedriver ni bygg- eller anläggningsverksamhet?" (construction)
   - **If SNI starts with 10-12, 55-56 (food/restaurant/hotel):** "Hanterar ni livsmedel?" (food)
   - **If SNI starts with 20-21 (chemicals/pharma):** "Hanterar ni kemikalier eller farliga ämnen?" (chemicals)
   - **If SNI starts with 84 (public admin):** "Är ni en myndighet eller offentligt ägd verksamhet?" (publicSector)
   - **If employee_count > 0 or employer tax status or SNI starts with 85 (education):** "Har ni anställda under 18 år?" (minorEmployees)
   - **If any international indicators:** "Bedriver ni internationell verksamhet?" (internationalOperations)
   - **If SNI starts with 25-33, 49-53 (manufacturing/logistics):** "Använder ni tunga maskiner eller fordon i verksamheten?" (heavyMachinery)
7. Maximum 5 questions shown (prioritized by relevance to SNI code)
8. If landing page URL analysis (Story 16.2) already inferred flags (persisted in OnboardingStore via `saveOnboardingData({ inferredFlags })`), pre-toggle those questions to "yes" with note: "Baserat på er webbplats"

### Data Persistence

9. Activity flag answers (chemicals, construction, food, personalData, publicSector, heavyMachinery, minorEmployees, internationalOperations) map to CompanyProfile `activity_flags` JSON field — serialized as `JSON.stringify({ chemicals: true, food: false, ... })` in FormData (matching existing `taxStatus`/`ongoingProcedures` pattern)
10. `has_collective_agreement` maps to its own dedicated boolean field on CompanyProfile (NOT part of activity_flags JSON) — persisted as a separate FormData entry `fd.append('hasCollectiveAgreement', String(value))`
11. Answers are included in the FormData passed to `createWorkspace()` — same pipeline as enrichment fields from Story 15.2
12. If user skips the step, no activity flags are set (all remain null/false)

### UX Details

13. Smooth transition animation between steps (CSS transition or Framer Motion consistent with landing page)
14. Each question has a one-line explanation below the label (e.g., "Påverkar vilka arbetsmiljöföreskrifter som gäller")
15. Progress indicator (wizard stepper) updated to show 3 steps with labels: "Företagsinfo" → "Verksamhet" → "Bekräfta"

## Dev Notes

### Current Onboarding Steps

The wizard at `app/onboarding/_components/onboarding-wizard.tsx` currently has:
```typescript
const WIZARD_STEPS = [
  { id: 'company-info' },
  // Future: { id: 'law-list' },
  { id: 'confirm' },
] as const
```

This story adds `{ id: 'activity-questions' }` between company-info and confirm. The `handleBack` from confirm step uses `setCurrentStep((prev) => Math.max(0, prev - 1))` which will correctly navigate to the new activity-questions step (step 1), not company-info (step 0).

### Dual Persistence: activity_flags vs has_collective_agreement

**Important:** These go to DIFFERENT database fields:
- Activity flags (chemicals, construction, etc.) → `CompanyProfile.activity_flags` (`Json?` type) — written as a single JSON object
- Collective agreement → `CompanyProfile.has_collective_agreement` (`Boolean`, default false) — written as its own column

The settings page (`company-profile-tab.tsx`) reads both independently:
- `getActivityFlags(profile)` reads from `activity_flags` JSON (lines 152-161)
- `profile.has_collective_agreement` reads the boolean field directly (line 232)

### Inferred Flags via OnboardingStore

If the user came from the landing page (Story 16.2), inferred flags are persisted in localStorage via Story 16.1's OnboardingStore. The onboarding wizard reads these and passes them as `defaultValues` to the activity questions step, which pre-toggles matching questions.

If the user navigated directly to signup (no landing page), no flags are stored and all questions default to false.
