# Story 15.1: Schema Migration & BolagsAPI Client

## Status

Done

## Story

**As a** developer,
**I want** a type-safe BolagsAPI client and enriched CompanyProfile schema,
**so that** the platform can fetch and store authoritative company data from Bolagsverket/SCB.

## Acceptance Criteria

### Schema Migration

1. New nullable fields added to `CompanyProfile` model:
   - `business_description String?` — free-text activity description from Bolagsverket
   - `tax_status Json?` — `{ f_tax: boolean, vat: boolean, employer: boolean }`
   - `foreign_owned Boolean @default(false)` — foreign parent company flag
   - `parent_company_name String?` — parent company name if subsidiary
   - `parent_company_orgnr String?` — parent company org number
   - `fi_regulated Boolean @default(false)` — Finansinspektionen regulated
   - `active_status String?` — "active" or "deregistered"
   - `ongoing_procedures Json?` — liquidation/restructuring flags
   - `registered_date DateTime?` — company registration date
   - `data_source String?` — "bolagsapi" or "manual"
   - `last_enriched_at DateTime?` — when data was last fetched from API
2. Migration is additive only — no column drops or renames, no data loss
3. Migration runs cleanly on existing database with populated CompanyProfile rows
4. Existing fields remain unchanged — no breaking changes to current queries

### BolagsAPI Client

5. Client module at `lib/bolagsapi/client.ts` with `fetchCompany(orgNumber: string)` function
6. Bearer token authentication via `BOLAGSAPI_API_KEY` environment variable
7. Calls `GET https://api.bolagsapi.se/v1/company/{orgnr}` with proper headers
8. 5-second timeout — returns `null` on timeout (does not throw)
9. Type-safe response type `BolagsApiCompany` matching the API schema (name, orgnr, org_form, address, business, tax_status, registered_date, company_size, website, foreign_owned, parent_company, fi, status, ongoing_procedures, meta)
10. Error handling: returns `null` for 404 (company not found), throws on auth errors (401/403), returns `null` on rate limit (429) or server errors (5xx)

### Response Mapping

11. Mapper function `mapBolagsApiToProfile(response: BolagsApiCompany)` at `lib/bolagsapi/mapper.ts`
12. Maps API response to `Partial<CompanyProfile>` with field-level mapping:
    - `name` → `company_name`
    - `orgnr` → `org_number` (formatted as XXXXXX-XXXX)
    - `org_form.code` → `legal_form`
    - `address.street + postal_code + city` → `address`
    - `address.municipality` → `municipality`
    - `business.sni_codes[0].code` → `sni_code`
    - `business.sni_codes[0].description` → `industry_label`
    - `business.description` → `business_description`
    - `tax_status` → `tax_status` (as JSON)
    - `registered_date` → `registered_date`, `founded_year` (extract year)
    - `company_size` → `employee_count_range` (map "small"→RANGE_1_9, "medium"→RANGE_10_49, "large"→RANGE_250_PLUS; if unrecognized value→UNKNOWN)
    - `website` → `website_url`
    - `foreign_owned` → `foreign_owned`
    - `parent_company` → `parent_company_name`, `parent_company_orgnr`
    - `fi` → `fi_regulated` (true if fi object is non-null and has registrations)
    - `status` → `active_status`
    - `ongoing_procedures` → `ongoing_procedures` (as JSON)
13. Mapper handles missing/null fields gracefully — only sets fields that have data
14. Sets `data_source: "bolagsapi"` and `last_enriched_at: new Date()` on mapped output

### Validation Endpoint

15. `GET /v1/validate/{orgnr}` wrapper: `validateOrgNumber(orgNumber: string)` returns `{ valid: boolean, exists: boolean, formatted: string }`
16. Validates checksum and existence in one call

### Testing

17. At least 10 unit tests covering:
    - Client: successful fetch, timeout, 404, auth error, rate limit
    - Mapper: full response mapping, partial response (missing fields), null handling
    - Validation: valid org number, invalid checksum, non-existent company

**Timeout pattern** — follow `lib/admin/health.ts` `fetchWithTimeout()` for the 5-second timeout:
[Source: lib/admin/health.ts]

```typescript
async function fetchWithTimeout(url: string, timeoutMs: number): Promise<Response> {
  const controller = new AbortController()
  const timeout = setTimeout(() => controller.abort(), timeoutMs)
  try {
    const response = await fetch(url, { signal: controller.signal })
    return response
  } finally {
    clearTimeout(timeout)
  }
}
```

Note: `lib/external/riksdagen.ts` does NOT have a timeout mechanism — it only has retry logic. Use the `health.ts` pattern above for timeouts.
