# Story 17.2: Tiptap Editor Component & Document Creation UI

## Status

Done

## Story

**As a** workspace member,
**I want** to create and edit documents using a rich text editor in Laglig,
**so that** I can author compliance documents (policies, risk assessments, action plans) directly in the app.

## Story Context

**Existing System Integration:**

- Integrates with: `WorkspaceDocument` and `WorkspaceDocumentVersion` models from Story 17.1, server actions in `app/actions/documents.ts`, sidebar navigation (`components/layout/left-sidebar.tsx`), workspace routing (`app/(workspace)/`)
- Technology: Tiptap (ProseMirror), React, shadcn/ui, Tailwind CSS
- Follows pattern: Existing workspace page layout (Server Component page + Client Component feature, like `app/(workspace)/tasks/page.tsx`), server actions via `withWorkspace()`
- Touch points: New route at `/workspace/documents/[documentId]/edit`, new sidebar nav entry ("Dokument" at `/workspace/documents`), new creation dialog

## Acceptance Criteria

**Editor Core:**

1. Document editor page at `/workspace/documents/[documentId]/edit` using Tiptap
2. Editor toolbar with formatting controls:
   - Text: bold, italic, underline, strikethrough
   - Headings: H1, H2, H3
   - Lists: bullet, numbered
   - Tables: insert, add/remove rows & columns, merge cells
   - Alignment: left, center, right
   - Colors: text color, highlight color
   - Other: links, inline images, horizontal rule, undo/redo
3. Slash command menu (`/`) for quick insertion of headings, tables, lists, images, horizontal rules
4. Editor saves content as Tiptap JSON to `WorkspaceDocumentVersion.content_json`
5. Auto-save with 2-second debounce after last keystroke — only creates new `WorkspaceDocumentVersion` if content actually changed (note: Story 17.3 will later add 30-second version batching on top of this)
6. Manual "Spara" button also available
7. Document title editable inline at top of editor page
8. Page-like container styling (A4 max-width, white background, subtle shadow) for a document feel
9. Editor responsive — usable on tablet screens (>=768px)

**Document Creation:**

10. "Nytt dokument" button accessible from workspace documents area
11. Creation dialog with fields: title (required), document type (`WorkspaceDocumentType` enum dropdown), optional template selection (Story 17.7 will populate templates)
12. Creating a document calls existing `createDocument()` server action (from Story 17.1) and navigates to the editor
13. If template selected, pass `templateId` — `createDocument()` already handles template content pre-fill

**Navigation & Chrome:**

14. Editor page shows document metadata bar: status badge, version number, last saved timestamp, author
15. Back navigation to document browser
16. Keyboard shortcut support: Ctrl/Cmd+B (bold), Ctrl/Cmd+I (italic), Ctrl/Cmd+Z (undo), Ctrl/Cmd+Shift+Z (redo), Ctrl/Cmd+S (save)

**Integration Requirements:**

17. Existing workspace pages and navigation unchanged
18. New page follows existing workspace layout patterns

**Quality Requirements:**

19. Editor loads within 1 second on standard connection
20. No layout shift when editor initializes (skeleton/loading state)

## Dev Notes

### Key Architecture Notes

**Tiptap is already used** — `components/ui/rich-text-editor.tsx` is a Jira-style editor using Tiptap. The document editor is a DIFFERENT, larger component (A4 full-page style vs. compact inline editor). Do NOT modify `rich-text-editor.tsx`. Create a new document-specific editor in `components/features/documents/editor/`.

**Model naming — `WorkspaceDocument*` prefix required** — The existing schema has `model DocumentVersion` and `enum DocumentStatus` for the SFS legal document system. All DMS models use the `WorkspaceDocument*` prefix:
- `WorkspaceDocument` (Prisma: `prisma.workspaceDocument`)
- `WorkspaceDocumentVersion` (Prisma: `prisma.workspaceDocumentVersion`)
- `WorkspaceDocumentTemplate` (Prisma: `prisma.workspaceDocumentTemplate`)
- Enums: `WorkspaceDocumentType`, `WorkspaceDocumentStatus`, `WorkspaceDocumentVersionSource`

**`generateHTML` import** — `generateHTML` is exported from `@tiptap/core` (a transitive dependency of `@tiptap/starter-kit`). There is NO standalone `@tiptap/html` package. Always import as:
```typescript
import { generateHTML } from '@tiptap/core'
```

**Hook location** — All custom hooks live in `lib/hooks/`. The autosave hook must be at `lib/hooks/use-document-autosave.ts`, not at a root `hooks/` directory (which does not exist).

**Prisma client** — Import as `import { prisma } from '@/lib/prisma'` (not `@/lib/db/prisma`).

**`WorkspaceDocumentVersion.created_by`** — This is a plain `String` field with NO User FK (because agent-authored versions store `"agent"`, which would violate a FK constraint). Set to `userId` from `withWorkspace()` for human saves. Resolve display names at query time if needed.

**Tiptap Editor Setup Pattern:**

```tsx
'use client'

import { useEditor, EditorContent } from '@tiptap/react'
import { generateHTML } from '@tiptap/core'  // NOT from '@tiptap/html'
import StarterKit from '@tiptap/starter-kit'
import Table from '@tiptap/extension-table'
import TableRow from '@tiptap/extension-table-row'
import TableCell from '@tiptap/extension-table-cell'
import TableHeader from '@tiptap/extension-table-header'
import Image from '@tiptap/extension-image'
import TextAlign from '@tiptap/extension-text-align'
import Placeholder from '@tiptap/extension-placeholder'
import Underline from '@tiptap/extension-underline'
import Link from '@tiptap/extension-link'
import Color from '@tiptap/extension-color'
import TextStyle from '@tiptap/extension-text-style'
import Highlight from '@tiptap/extension-highlight'

const editor = useEditor({
  extensions: [
    StarterKit.configure({ heading: { levels: [1, 2, 3] } }),
    Table.configure({ resizable: true }),
    TableRow, TableCell, TableHeader,
    Image.configure({ inline: true }),
    TextAlign.configure({ types: ['heading', 'paragraph'] }),
    Placeholder.configure({ placeholder: 'Börja skriva...' }),
    Underline, Link, Color, TextStyle,
    Highlight.configure({ multicolor: true }),
  ],
  content: initialContentJson, // Tiptap JSON from WorkspaceDocumentVersion
  immediatelyRender: false,    // REQUIRED — prevents SSR hydration mismatch
  onUpdate: ({ editor }) => {
    // Trigger debounced auto-save
  },
})
```

**Generating HTML server-side (for `content_html` in `saveDocumentVersion`):**

```typescript
import { generateHTML } from '@tiptap/core'  // correct import

// ... import same extensions used in the editor

const html = generateHTML(contentJson, [StarterKit, Table, TableRow, ...])
```

```typescript
  documentId: string,
  contentJson: Record<string, unknown>,
  changeSummary?: string

  try {
    return await withWorkspace(async ({ workspaceId, userId }) => {
      // 1. Verify document belongs to workspace
      // 2. Get current version_number

      // 4. Generate extracted_text by stripping HTML
      // 5. Create WorkspaceDocumentVersion with created_by: userId (plain string)
      // 6. Update WorkspaceDocument.current_version_id and current_version_number
      // (steps 5-6 in a transaction)
```
