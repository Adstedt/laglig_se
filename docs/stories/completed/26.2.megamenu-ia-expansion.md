# Story 26.2: Megamenu IA Expansion (Branscher + Områden)

## Status

Done

## Story

**As a** prospect exploring Laglig.se from any page,
**I want** the navbar and footer to expose real industry (Branscher) and topic (Områden) pages — with working links from every page, not just the homepage,
**so that** the page list users can discover matches the page list editorial ships, and the marketing funnel is navigable end-to-end.

## Acceptance Criteria

### Desktop megamenu (NavbarV3)

1. **Branscher** dropdown lists the 7 Tier-1 industries with exact route slugs: Restaurang & hotell → `/branscher/hotell-restaurang`, Bygg & anläggning → `/branscher/bygg`, Vård & omsorg → `/branscher/vard-omsorg`, Industri & tillverkning → `/branscher/industri`, Transport & logistik → `/branscher/transport`, IT & tech → `/branscher/it`, Fastighet → `/branscher/fastighet`. Existing icons kept.
2. **"Handel & e-handel" is removed** from the Branscher menu and footer — no page is planned for it anywhere in Epic 26, and a permanently disabled item is worse than absence. (Decision recorded; PO can re-add when a page is planned.)
3. New **Områden** dropdown (between Branscher and Regelverk) lists all 8 Tier-2 topics: GDPR & dataskydd → `/omraden/gdpr`, NIS2 → `/omraden/nis2`, Arbetsmiljö → `/omraden/arbetsmiljo`, Brandskydd → `/omraden/brandskydd`, Miljö → `/omraden/miljo`, Visselblåsarlagen → `/omraden/visselblasarlagen`, Penningtvätt → `/omraden/penningtvatt`, ISO 14001 → `/omraden/iso-14001`. Same 2-column `FeatureLink` pattern as Branscher, lucide icons per topic.
4. **Produkt** dropdown items swap to feature routes with the mapping: Efterlevnad (relabel **Laglista**) → `/funktioner/laglista`, Lagändringar → `/funktioner/lagandringar`, Uppgifter → `/funktioner/uppgifter`, Styrdokument → `/funktioner/styrdokument`, Kontroll → `/funktioner/kontroller`, AI-agenten → `/funktioner/ai-agent`, plus new item **Kravpunkter** → `/funktioner/kravpunkter` (7 items total). **Regelverk and the "Mer" column stay as-is** (Mer anchors become root-relative per AC 8).

### Build-state awareness ("Kommer snart")

5. Link state is **driven by published content, not hardcoded**: a server-side helper reads `listMarketingSlugs()` per kind and passes the published route list into `NavbarV3`/`FooterV3` (new optional serializable prop, default `[]`). For each menu item:
   - route published → renders as a real `next/link` (client-side nav + prefetch)
   - route NOT published, item has a homepage-anchor fallback (Produkt items) → renders the root-relative anchor (current behavior preserved, no dead ends)
   - route NOT published, no fallback (Branscher/Områden) → renders non-interactive with a "Kommer snart" badge (visually muted, `aria-disabled="true"`, not focusable as a link)
6. When a new MDX page ships (26.4–26.8), its menu item flips to a live link **with zero nav code changes** (the prop is computed from the content folder at render time).

### Cross-page anchor fix (bug exposed by 26.1)

7. All bare homepage anchors in NavbarV3 (`#pricing`, `#how-it-works`, `#ai`, `#skala`, `#faq`, `#testa`, `#byraer`) become **root-relative** (`/#pricing`, …) so the chrome works identically on `/` and on every marketing page. Verified on a marketing page: clicking "Priser" navigates to the homepage pricing section.
8. Same fix in FooterV3 for its Produkt column + "För revisorer & byråer".

### Mobile sheet

9. Mobile sheet gains an **Områden** accordion section; Branscher section reflects the new 7-item list with the same published/kommer-snart logic; Produkt section reflects AC 4's mapping.

### Footer (FooterV3)

10. Branscher column lists the same 7 industries (published → link, else "Kommer snart" muted text). "Alla branscher" entry is **removed** (no `/branscher` index page exists in this epic; re-add with a future index story).
11. New **Områden** column with the 8 topics, same logic. Column order: Produkt, Branscher, Områden, Regelverk, Företag (grid adjusts to 5 columns on desktop, stacks on mobile per existing pattern).

### Quality gates

12. Unit tests: link-resolution helper (published → route; unpublished+fallback → root-relative anchor; unpublished without fallback → kommer-snart), nav-data module (slug exactness for all 22 routes against the epic's canonical list). Component test: mobile sheet renders "Kommer snart" state and live-link state correctly from the prop.
13. No regression: homepage anchors still scroll on `/` (root-relative anchors resolve there too), Regelverk dropdown unchanged, login/signup CTAs unchanged, cookie banner + mobile sheet unaffected, `pnpm lint && pnpm typecheck && pnpm test` green.
14. Design language: new Områden menu + footer column match the existing dropdown/footer treatment exactly (Safiro `font-medium` where titles use it, existing trigger/link classes, no new colors).

### The anchor bug (why AC 7–8 exist)

26.1 made NavbarV3 + FooterV3 render on `/funktioner/*`, `/branscher/*`, `/omraden/*` via `<MarketingShell>` (`components/marketing/templates/marketing-shell.tsx`). Bare `#pricing`-style anchors are dead on those pages. Root-relative `/#pricing` behaves identically on the homepage (browser resolves to same-page anchor) and correctly navigates home from marketing pages. This is the minimal fix; do NOT add scroll-state libraries.

### Published-route detection (AC 5–6 design)

`lib/marketing/content.ts` (26.1) exports `listMarketingSlugs(kind)` — server-only (node:fs). NavbarV3/FooterV3 are client components → they cannot call it. Both render points are server components (`app/page.tsx`, `MarketingShell`), so: compute `getPublishedMarketingRoutes(): string[]` server-side (e.g. `['/branscher/bygg', '/funktioner/kontroller', …]`) and pass as a serializable prop. Default `[]` keeps every item in fallback/kommer-snart state — safe for any other render site. Underscore drafts are already excluded by `listMarketingSlugs`. [Source: lib/marketing/content.ts (Story 26.1); architecture/17-coding-standards.md#17.3 — Server Components for data, client for interactivity]
