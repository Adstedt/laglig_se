# Story 22.7: Tasks 5-tab atom alignment + toolbar restructure (Scope C)

## Status

Done

## Story

**As a** user navigating the five Tasks tabs (Sammanfattning, Aktiva, Lista, Kalender, Alla uppgifter),
**I want** the same task atoms (status pill, priority pill, overdue indicator, empty state) to look identical across every tab where they appear,
**so that** I'm not re-learning a task's "weight" each time I switch view, and the chrome (header, toolbar, primary CTA) sits in the same absolute position regardless of which tab I'm on.

## Background

The Tasks page rendered the same task data through five different visual languages — five status pill implementations, three priority renderings, four overdue treatments, two-and-a-half empty-state shapes. Per the audit prototype `_prototypes/epic-22-tasks-tabs-alignment.html`, the user picked **Scope C** (atoms + empty states + toolbar restructure). This story is that scope, as-built.

The toolbar restructure followed a separate sub-decision: instead of inlining bespoke two-row JSX in `task-workspace/index.tsx`, we evolved `<TableToolbar>` to support an optional `tabs` slot — Tasks now shares the toolbar primitive with Kontroller and Styrdokument. This sets up `<UnifiedToolbar>` for eventual retirement at Laglistor (its only remaining caller).

## Acceptance Criteria

### Functional — primitives & utilities

1. **`<ColorTagBadge>` primitive** at `components/ui/color-tag-badge.tsx`. Soft-fill pill tinted by an arbitrary hex color. Props:
   - `name: string`, `color: string` (hex like `#2563eb`)
   - `size?: 'sm' | 'md'` (default `'md'`); `showDot?: boolean` (default `true`); `className?`
   - Output: backgroundColor = `${color}24` (~14% alpha), border = `${color}59` (~35% alpha), text = `${color}`, optional 8px dot using full color
   - Used by Lista (via `task-status-editor`), Alla uppgifter (status cell). Calendar exempt (uses fixed semantic palette tied to its legend).

2. **`<EmptyState>` + `<EmptyState.Icon>` primitive** at `components/ui/empty-state.tsx`. Props: `icon?`, `title?`, `description?`, `action?`, `className?`. Optional `<EmptyState.Icon>` wraps an icon in `rounded-full bg-muted p-4` for the canonical "decorated" empty.

3. **`isTaskOverdue` helper** at `lib/utils/task-utils.ts`. Replaces 5 inline duplicates across the task tabs. Definition: `!task.column.is_done && task.due_date && new Date(task.due_date) < new Date()`.

4. **`<TableToolbar>` evolution** — adds optional `tabs?: ReactNode` slot. When `tabs` is provided, renders two-row mode: `tabs` on Row 1 (full width), `search + filters` left-aligned + `rightSlot` pinned right on Row 2. When `tabs` is omitted, renders existing single-row mode unchanged. `tabs` and `views` mutually exclusive.

### Functional — page-level changes

5. **"Ny uppgift" CTA → `PageHeader.primaryAction`.** Lifted from inside `<UnifiedToolbar>` to the page header. Required moving `<PageHeader>` rendering from `app/(workspace)/tasks/page.tsx` (server component) into `task-workspace/index.tsx` (client component) so it can call `setCreateTaskModalOpen`.

6. **Drop `<UnifiedToolbar>` from Tasks.** `task-workspace/index.tsx` now uses `<TableToolbar tabs={<TabNavigation/>} search={...} filters={<TaskFilterBar/>} rightSlot={isListTab ? <ColumnSettings/> : undefined} />`. `<UnifiedToolbar>` itself stays in the codebase (Laglistor is still a caller); only Tasks migrates off.

7. **Atom migrations across tabs:**
   - **Lista**: status cell uses `<ColorTagBadge>` (via `task-status-editor`); priority cell already on canonical `<Badge tone>` from `getPriorityBadgeProps` (no change). Overdue rows: drop `bg-destructive/5` row tint, replace with `border-l-2 border-l-destructive` + red title text + `<AlertCircle>` prefix on the title.
   - **Aktiva (Kanban)**: per-card priority badge added (next to title in a `flex justify-between`) using inline `getPriorityBadgeProps + Badge` — gives explicit priority alongside the existing left-border scan cue. Overdue cards: red border kept, plus `<AlertCircle>` icon prefix on title. Per-column "Inget i denna kolumn ännu" placeholder rendered inside the droppable div when `tasks.length === 0` (NOT as a sibling of the Sortable/draggable items, per dnd-kit gotcha — keeps drop targeting intact).
   - **Alla uppgifter**: drops local `PRIORITY_CONFIG`; status cell → `<ColorTagBadge>`; priority cell → inline `getPriorityBadgeProps + Badge`. `StatusIcon` swaps `Clock` → `AlertCircle` for the overdue case. Overdue rows get `border-l-2 border-l-destructive`.
   - **Kalender**: keeps hardcoded green/blue/red palette (3-state semantic with legend). `isTaskOverdue` adopted; full `<EmptyState>` early-return when no tasks have due dates.
   - **Sammanfattning**: bar charts unchanged (charts, not status displays — out of scope).

8. **Empty states migrated to `<EmptyState>`:**
   - Lista: existing hand-rolled block → `<EmptyState icon={<EmptyState.Icon><ListTodo/></EmptyState.Icon>} title="Inga uppgifter matchar" description="Justera dina filter eller skapa en ny uppgift" />`
   - Alla uppgifter: in-table `colSpan` empty fallback → wrapping `filteredTasks.length === 0` early-return
   - Kanban per-column placeholder (small inline form, not full primitive — context is a 280px column)
   - Kalender month-empty (full primitive, early-return)

9. **Toolbar polish (mechanical fixes):**
   - Tasks `FilterPopover` and `SelectTrigger` triggers: drop redundant `h-8` overrides (`size="sm"` already gives `h-9`)
   - `DueDateFilterPopover`: drop redundant `className="h-8"` override
   - `Ansvarig:` label prefix removed from filter row (parity with Status / Prioritet / Förfallodatum which have no label prefix); placeholder becomes `Ansvarig`, width bumped `120 → 140` to fit label + chevron

## Dev Notes

### Q1 (resolved)
**Status color source** — workspace `column.color` from the DB. The `<TaskColumn>` Prisma model already has a `color` field; `<ColorTagBadge>` consumes it directly. Calendar exempt (uses 3-state semantic palette: green=done / blue=active / red=overdue, tied to its bottom-of-grid legend).

### Q2 (resolved)
**Per-card status badge on Kanban** — no. Column position already signals status (you're IN the "Att göra" column). Adding a per-card status badge would be redundant. Per-card *priority* badge is added (priority isn't signalled by column).

### Lista row treatment decision
Original Lista overdue used `bg-destructive/5` (full-row pink tint). With the new badges (status + priority both with backgrounds), that row tint became visually noisy. Decision: drop the row tint, replace with `border-l-2 border-l-destructive` + red title text + `<AlertCircle>` prefix on the title. Lighter touch, same signal strength.

### dnd-kit gotcha (Kanban per-column empty)
The per-column placeholder MUST render INSIDE the `<div ref={setDroppableRef}>` wrapper (replacing the `tasks.map(...)` when empty), NOT as a sibling. dnd-kit measures the droppable rect and counts children; rendering the placeholder outside the droppable breaks drop targeting on empty columns. The plan calls this out explicitly.
