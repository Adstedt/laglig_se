# Story 17.6: .docx & PDF Export

## Status

Done

## Story

**As a** workspace member,
**I want** to export documents as Word (.docx) or PDF files,
**so that** I can share compliance documents with auditors, authorities, or colleagues outside Laglig.

## Story Context

**Existing System Integration:**

- Integrates with: Tiptap editor from Story 17.2 (`components/features/documents/editor/document-editor.tsx`), `WorkspaceDocumentVersion` model (`content_json`, `content_html` fields — from Story 17.1), document metadata (`title`, `document_number`, `status`, `current_version_number`, `approved_at` on `WorkspaceDocument`), version history panel from Story 17.3 (`components/features/documents/editor/version-history-panel.tsx`)
- Technology: `docx` npm package (Word generation — **not yet installed, Task 0**), `puppeteer-core` + `@sparticuz/chromium` (PDF generation — **not yet installed, Task 0**)
- Follows pattern: API route export with `getWorkspaceContext()` auth (same as `app/api/workspace/activity-log/export/route.ts`), `NextResponse` with `Content-Disposition: attachment`. Note: API routes use `getWorkspaceContext()` (not `withWorkspace()` which is for server actions).
- Touch points: Editor page, version history panel, new API route

**Model naming — `WorkspaceDocument*` prefix required:**
- `WorkspaceDocument` (Prisma: `prisma.workspaceDocument`)
- `WorkspaceDocumentVersion` (Prisma: `prisma.workspaceDocumentVersion`)

## Acceptance Criteria

**Export UI:**

1. "Exportera" dropdown button on document editor page with options: "Word (.docx)" and "PDF (.pdf)"
2. Export generates from current version by default
3. Option to export a specific historical version (accessible from version history panel)

**Word Export:**

4. Word export uses `docx` npm package to generate .docx from Tiptap JSON
5. Maps Tiptap nodes to Word elements: headings → Word heading styles, paragraphs, bold/italic/underline, bullet/numbered lists, tables (with column widths and merged cells), images (embedded), links
6. Tables render correctly with cell borders, column widths, and merged cells

**PDF Export:**

7. PDF export renders document content as PDF
8. PDF approach: render `content_html` via Puppeteer (server-side headless Chrome) using `@sparticuz/chromium` for Vercel compatibility
9. PDF includes document metadata header (same as Word)

**Metadata Header:**

10. Both export formats include a metadata header on the first page: document title, document number (if set), version ("v{N}"), status, approved date (if approved), workspace/organization name
11. Consistent Laglig branding — clean, professional layout

**File Naming & Download:**

12. Export filename format: `{title}-v{version}-{YYYY-MM-DD}.docx` or `.pdf`
13. Special characters in title sanitized for filename safety
14. Download triggers browser save-as dialog

**Quality:**

15. Images embedded in exports (not external URLs)
16. Export of a 50-page document completes within 10 seconds
17. Swedish characters (å, ä, ö) render correctly in both formats

**API route auth pattern (from `app/api/workspace/activity-log/export/route.ts`):**

```typescript
import { NextRequest, NextResponse } from 'next/server'
import { getWorkspaceContext, WorkspaceAccessError } from '@/lib/auth/workspace-context'

export const maxDuration = 30 // PDF generation can be slow

export async function GET(
  request: NextRequest,
  { params }: { params: Promise<{ documentId: string }> }
) {
  try {
    const { workspaceId } = await getWorkspaceContext()
    const { documentId } = await params
    // ... fetch document enforcing workspace_id match ...
  } catch (error) {
    if (error instanceof WorkspaceAccessError) {
      const status = error.code === 'UNAUTHORIZED' ? 401 : 403
      return NextResponse.json({ error: error.message }, { status })
    }
    return NextResponse.json({ error: 'Ett oväntat fel uppstod' }, { status: 500 })
  }
}
```

Note: This is `getWorkspaceContext()` (API route pattern), NOT `withWorkspace()` (server action pattern). They are different — `getWorkspaceContext()` returns `{ workspaceId, userId }` directly; `withWorkspace()` wraps a callback.

**Puppeteer PDF generation (Vercel-compatible):**

```typescript
import chromium from '@sparticuz/chromium'
import puppeteer from 'puppeteer-core'

export async function generatePdf(html: string): Promise<Buffer> {
  const browser = await puppeteer.launch({
    args: chromium.args,
    defaultViewport: chromium.defaultViewport,
    executablePath: await chromium.executablePath(),
    headless: chromium.headless,
  })
  try {
    const page = await browser.newPage()
    await page.setContent(html, { waitUntil: 'networkidle0' })
    const pdf = await page.pdf({
      format: 'A4',
      margin: { top: '2cm', bottom: '2cm', left: '2cm', right: '2cm' },
    })
    return Buffer.from(pdf)
  } finally {
    await browser.close()
  }
}
```

Why `puppeteer-core` + `@sparticuz/chromium`: Vercel has ~250MB compressed size limit. Full `puppeteer` bundles ~300MB Chromium. `puppeteer-core` is library-only. `@sparticuz/chromium` provides a Vercel-compatible binary.
