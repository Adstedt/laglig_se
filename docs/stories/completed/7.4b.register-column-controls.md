# Story 7.4b: Register column controls — show/hide + resize

## Status

Done

## Story

**As an** HR manager,
**I want** to hide/show and resize the register's columns,
**so that** I can tailor the view — including hiding the Personnummer column while screen-sharing.

## Acceptance Criteria

1. A **"Kolumner"** control on the register (placement/affordance mirroring how the Laglistor table exposes column visibility — bind to its actual pattern) toggles column visibility per column.
2. **Anställd is not hideable** (primary/identity column); structural columns (drag handle, group select where applicable) are excluded from the toggle list. Everything else — including **Personnummer** (the screen-share privacy case) — is hideable.
3. Columns are **resizable** by dragging header edges, with **min/max clamping per column** (mirror the law table's `COLUMN_SIZE_BOUNDS` approach + `columnResizeMode: 'onEnd'` — this is the fix for the law table's historical drag-to-infinite-width bug; do not reintroduce it).
4. Visibility + sizing **persist per user** across reloads (bind to the law table's actual persistence mechanism; if it's coupled to the document-list Zustand store, use `localStorage` keyed by workspace id instead — do NOT extend the document-list store).
5. **No column reordering** (deliberately out of scope — user decision).
6. Hidden columns interact sanely with the rest: search still matches personnummer when the column is hidden (search is over data, not cells); group sections/rollups unaffected.

## QA Results

### Review Date: 2026-07-03

### Reviewed By: Quinn (Test Architect)

### Code Quality Assessment

Excellent. The story's core risk — reintroducing the law table's drag-to-infinite-width bug — is structurally closed: every sizing path funnels through `clampEmployeeColumnSizing` before anything is stored or persisted. I traced all of them: (1) the `onEnd` commit — `handleColumnSizingChange` resolves the functional updater against the same parent state every section instance renders from, then clamps before store+persist; (2) stale localStorage — `loadEmployeeColumnState` clamps on read AND `saveEmployeeColumnState` clamps on write, so the key can never hold an out-of-bounds width; (3) the live drag preview — `getColumnWidth` clamps `startSize + deltaOffset` per frame; (4) the render fallback clamps defensively. The bounds table matches the column defs' `size`/`minSize`/`maxSize` 1:1 for all ten ids (catalog-invariant test guards drift), and the register sanitizer is deliberately stricter than the law table's `clampColumnSizing` (unknown ids dropped instead of passed through; NaN/Infinity rejected).

Non-hideable guarantee (AC2) is genuine defense in depth — all three layers verified: `enableHiding: false` on `name`/`dragHandle` defs; `sanitizeEmployeeColumnVisibility` strips both ids on load, on save, and on every change (so even the shared control's "Återställ standard" emitting `name: true` is normalized away harmlessly); and the `mandatory` → disabled "Obligatorisk" checkbox (RTL asserts `data-disabled`).

Shared-state-across-sections is sound: one `columnState` above all per-group TanStack instances, threaded via a memoized `columnControls`. The dev-flagged functional-updater resolution is correct — the updater resolves against exactly the state every instance was rendered with. I probed for the same-tick race: visibility changes originate only from the single Kolumner control (full state, not updater) and sizing only from one grip at a time, so a clobber is unreachable from today's UI — but the two handlers do rebuild the full state from the other half's closure value with non-functional setState, which is a latent trap for future callers (STATE-001, low).

Interaction seams check out: the resize grip is a separate absolutely-positioned sibling of the sort button (pointer-down stopPropagation; a grip drag cannot synthesize a click on the button per browser click-target semantics), dnd-kit listeners live only on the row drag-handle button so headers never fight it, the hover/focus sort reveal is intact (including `group-focus-within` — A11Y-001 from the 7.4 gate is addressed in this code), and the `table-fixed` + `minWidth` + trailing spacer layout is mirrored by a matching spacer row cell so header/body column counts stay aligned in every section.
