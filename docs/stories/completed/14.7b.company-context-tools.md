# Story 14.7b: Company Context Tool

## Status

Ready for Review

## Story

**As a** compliance partner agent,
**I want** a tool to access the company's profile and compliance posture,
**so that** I can give company-specific guidance grounded in the user's actual context.

## Context & Dependencies

This is the second of three stories split from the original 14.7 (Agent Tool Definitions). It adds a read-only tool that surfaces workspace-specific company data: the company profile, tracked law list summary, compliance status distribution, and pending change count. This tool transforms the agent from a generic legal Q&A bot into a company-aware compliance partner.

**Scope reduction (2026-03-09):** Originally included three tools. `search_company_files` and `get_compliance_history` have been deferred — see "Deferred Tools" section below for rationale.

**Key existing infrastructure:**

- **Tool infrastructure** (Story 14.7a — Done): Shared types (`ToolResponse`, `ToolError`, `ToolMeta`), utility helpers (`wrapToolResponse`, `wrapToolError`), factory function pattern `createAgentTools(workspaceId)`, chat route already wired with `tools` + `stopWhen: stepCountIs(5)`. This story adds the company context tool to the existing registry.
- **Existing server actions**: Company profile (`getCompanyProfile()` in `app/actions/company-profile.ts`), change events (`getUnacknowledgedChangeCount()` in `app/actions/change-events.ts`) — `get_company_context` wraps these existing functions. [Source: app/actions/]

**Depends on:** 14.7a (tool infrastructure, types, factory pattern, chat route wiring)

**Soft dependency:** 14.4 (WorkspaceProfile settings UI — not yet built). The `CompanyProfile` model already exists in the schema with all fields migrated (lines 160-184 in `prisma/schema.prisma`). The `getCompanyProfile()` action works and auto-creates via upsert. The "soft dep" is that most profile fields will be NULL until 14.4/14.5 provide a settings UI to populate them.

**Depended on by:** 14.9 (system prompt references company context tools), 14.10 (assessment flow)

## Acceptance Criteria

### Company Context Tool

1. `get_company_context` — company profile + compliance posture. Parameters: none (workspace resolved from factory closure). Returns: company profile fields (`companyName`, `orgNumber`, `sniCode`, `industryLabel`, `employeeCountRange`, `profileCompleteness`), tracked law list names with item counts, compliance status distribution, recent unacknowledged change count, `profileComplete` boolean
2. Tool description written as comprehensive docstring following 14.7a rigor — purpose, when to use, what it returns, edge cases
3. Response shape follows `ToolResponse<T>` with `_meta: { tool, executionTimeMs, resultCount }` pattern
4. Error responses include specific, actionable guidance in Swedish
5. Tool registered in the `createAgentTools` factory function in `lib/agent/tools/index.ts`
6. Graceful degradation: if profile fields are NULL (pre-14.4 settings UI), return what's available with `profileComplete: false` so the agent can nudge the user
7. At least 3 unit tests covering: response shape with `_meta`, mock server action integration, graceful degradation with NULL fields

### Security Review

No concerns. The `$queryRaw` uses Prisma's tagged template literal parameterization — `${workspaceId}` is automatically parameterized and safe from SQL injection. The `workspaceId` comes from the factory closure (set from authenticated session in the chat route), not from user input. No auth/payment files touched. Read-only tool — no mutations beyond the auto-create upsert (which mirrors existing server action behavior).

### Performance Considerations

The tool makes 4 sequential DB queries (profile, law lists, groupBy, raw count). For typical workspaces these are all fast (sub-10ms each). A `Promise.all` parallelization could reduce latency by ~50% but is not required — this is a single-call-per-conversation tool, not a hot path. Tagged as future improvement.
