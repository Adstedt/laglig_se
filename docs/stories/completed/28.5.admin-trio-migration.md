# Story 28.5: Admin Trio — One Factory, Three Tables

**Epic:** 28 — Unified DataTable Core (`docs/prd/epic-28-unified-datatable-core.md`)
**Status:** Done (2026-07-08)

## What changed

`template-table.tsx` / `workspace-table.tsx` / `user-table.tsx` (~1,086
lines, ~90% mutually identical) → **749 lines total** including the new
shared scaffolding:

- `components/admin/admin-data-table.tsx` — `useUpdateSearchParams`
  (page-reset convention preserved), `useAdminSearch` (300ms debounced →
  URL), `useAdminSorting` (sortBy/sortDir URL adapter, manual mode),
  and `AdminDataTable` (paginated DataTable + row→detail navigation +
  "Visar X–Y av Z" summary).
- Core addition (pre-freeze): `pagination` LoadMoreStrategy gained an
  optional `summary` string (left-aligned in the pagination row).
- The three local `SortableHeader` copies are deleted — all headers now
  use the shared `ui/sortable-header` through the manual-sorting adapter.
- Each table keeps only its own columns (with `meta.dt` labels + card
  slots), title/filters, and create-dialog. Sort fields, URL params, page
  props and detail routes unchanged — the admin pages were not touched.
- Free uplift: the admin lists now get the card renderer below 640px and
  the core's interactive-element guard.
