Laglig.se — the whole app on one page

Swedish legal-compliance SaaS: companies build laglistor from an 11K+ document legal corpus, track compliance per law, get a tool-using AI agent grounded in that corpus, monitor amendments, and run ISO-grade audit cycles. Diagrams are Mermaid — rendered live; the raw text is readable if rendering is unavailable. app-map.json is the machine twin.

02Request & auth flow

Middleware only checks JWT presence; the real tenancy/RBAC/billing enforcement is server-side in getWorkspaceContext() — Prisma can't run in Edge.

300/min prod"} -- ok --> PAGE["render"]
  RL -- exceeded --> E429["429 + X-RateLimit-*"]

  JWT -- no --> LOGIN["302 /login?callbackUrl"]

  CK -- "none, 0 ws" --> OB["302 /onboarding"]
  AUTO --> CTX
  CK -- set --> CTX["getWorkspaceContext()
Redis-cached 5min"]
  CTX --> MEM{"WorkspaceMember + role"}
  MEM -- no --> DENY["ACCESS_DENIED"]
  MEM -- yes --> GATES{"billing gates"}
  GATES -- past_due / trial_expired --> BILL["302 /settings?tab=billing"]
  GATES -- ok --> SHELL["WorkspaceShell + page

Session model: Supabase Auth stores credentials; NextAuth CredentialsProvider calls signInWithPassword, upserts the user into Prisma, and issues a 30-day JWT. Roles: OWNER · ADMIN · HR_MANAGER · MEMBER · AUDITOR (read-only, including agent tools).

08State machines

Styrdokument dual-version lifecycle (Epic 17)
stateDiagram-v2
  [*] --> DRAFT : createDocument / adopt template / agent draft approved
  DRAFT --> IN_REVIEW : submitDraftForReview
  IN_REVIEW --> DRAFT : back to draft
  IN_REVIEW --> APPROVED : promoteDraftToApproved
  DRAFT --> APPROVED : promoteDraftToApproved
  APPROVED --> DRAFT : new draft (dual pointers, approved stays live)
  APPROVED --> SUPERSEDED : replaced
  APPROVED --> ARCHIVED : retired
  note right of APPROVED : current_approved_version_id + current_draft_version_id coexist. Every save marks needs_reindex → sweep cron re-embeds.
PendingAgentAction — human-in-the-loop writes
stateDiagram-v2
  [*] --> PENDING : write tool called mid-stream
  PENDING --> APPROVED : approvePendingAction → executes for real
  PENDING --> REJECTED : rejectPendingAction
  PENDING --> IN_EDITOR : openDraftInEditor (draft_styrdokument)
  IN_EDITOR --> APPROVED : finalizeDraftFromEditor
  IN_EDITOR --> REJECTED : rejectDraftFromEditor
  PENDING --> EXPIRED : 7d TTL (expire-pending-actions cron)
  note right of PENDING : Params are an editable snapshot — user can adjust before approving. State fed back to the model each turn.

13Ops gotchas — read before touching prod

Supavisor pooler is the #1 fragility. Multi-MB insert statements × concurrency crash the pooler's tenant handler. Safe ingest profile: connection_limit=1, ≤50-row insert batches (enforced in sync-document-chunks.ts), breaker/watchdog. A wedged pool (ECHECKOUTTIMEOUT with idle DB) is fixed only by a dashboard project restart. Batch jobs must never use the prod transaction pooler.
Migrations are manual. Never apply DB migrations from the assistant/CI — hand the command to Alexander (prisma migrate deploy). DDL hangs = leaked idle-in-transaction connections; terminate them first. Never use the Supabase web editor for DDL.
PDF export is fragile by design. node-linker=hoisted + chromium-min + CHROMIUM_PACK_URL (must exist in the prod env scope) + forced function tracing.
"draft_styrdokument failed" may be a quota cap. Chat returns HTTP 402 at 2× the tier token cap — check WorkspaceUsage.tokens_used_this_period before debugging the skill.
Canonical UI patterns. Laglistor's DocumentListTable is the gold-standard table (don't migrate to shadcn). Safiro = titles/section labels only (font-medium, never faux-bold); GSF for content. Landing palette: warm ink/grays on cream, one sparing amber accent.
Search vs semantics. Full-text = Elasticsearch (PG ILIKE fallback). Semantic = ContentChunk pgvector only — legal_documents.embedding exists but is unused for search.
Cron hard-kills are silent. A job killed at maxDuration leaves a RUNNING zombie in CronJobRun and sends no failure email (the catch block never runs). Incident: discover-sfs-amendments hard-killed every run 2026-06-17→07-23 → amendments stale from 2026:1198. Fixed 2026-07-23: discovery/processing split into separate crons with budget guards, and sfs-pipeline-health (daily 07:30) sweeps zombies → FAILED and alerts on register-vs-DB staleness. Zombies in other jobs still surface via the same sweep.
